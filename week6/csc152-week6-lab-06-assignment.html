Create users fthomas and gmartin using PowerShell.

PowerShell Commands
# Create secure password $Password = Read-Host -AsSecureString "Enter password" # Create fthomas New-LocalUser -Name "fthomas" ` -Password $Password ` -FullName "Frank Thomas" ` -Description "IT Support" # Create gmartin New-LocalUser -Name "gmartin" ` -Password $Password ` -FullName "Grace Martin" ` -Description "Finance" # Add fthomas to Backup Operators Add-LocalGroupMember -Group "Backup Operators" -Member "fthomas" # Verify users Get-LocalUser | Where-Object {$_.Name -in @("egarcia","fthomas","gmartin")} | Select-Object Name, FullName, Description, Enabled

Part C: sudo Configuration

5 Points

Configure granular sudo access for users on your Rocky Linux server, implementing the principle of least privilege.

🚨 Critical: Always use visudo to edit sudoers files. Never edit /etc/sudoers directly with a text editor.

sudo Rules to Implement

User/Group Allowed Commands NOPASSWD
bwilson systemctl restart nginx, systemctl status nginx No
%engineering cat /var/log/nginx/*, tail /var/log/nginx/* No
svc_backup /usr/local/bin/backup.sh Yes

Create separate configuration files in /etc/sudoers.d/ for each rule.

Create webadmin sudoers file
# Create file for bwilson (web admin tasks) sudo visudo -f /etc/sudoers.d/webadmin # Add this content: # Web administrator - nginx management bwilson ALL=(root) /bin/systemctl restart nginx, /bin/systemctl status nginx
Create engineering sudoers file
# Create file for engineering group (log viewing) sudo visudo -f /etc/sudoers.d/engineering # Add this content: # Engineering team - view nginx logs %engineering ALL=(root) /bin/cat /var/log/nginx/*, /bin/tail /var/log/nginx/*
Create backup sudoers file
# Create file for backup service account sudo visudo -f /etc/sudoers.d/backup # Add this content: # Backup service account - run backup script without password svc_backup ALL=(root) NOPASSWD: /usr/local/bin/backup.sh
Verify Configuration
# Check syntax of all sudoers files sudo visudo -c # Test bwilson's sudo access sudo -l -U bwilson # Test engineering group access sudo -l -U dlee # List sudoers.d files ls -la /etc/sudoers.d/

Part D: Automation Scripts

4 Points

Create scripts to automate user management tasks, building on your Week 5 scripting skills.

Create a Bash script that creates a user with specified parameters.

create_user.sh
#!/bin/bash #=============================================== # Script: create_user.sh # Purpose: Create a new user with specified settings # Author: [Your Name] # Date: [Date] # Usage: ./create_user.sh username "Full Name" department #=============================================== # Check if running as root if [[ $EUID -ne 0 ]]; then echo "Error: This script must be run as root" exit 1 fi # Check arguments if [[ $# -lt 3 ]]; then echo "Usage: $0 username \"Full Name\" department" echo "Example: $0 jdoe \"John Doe\" engineering" exit 1 fi USERNAME="$1" FULLNAME="$2" DEPARTMENT="$3" # Check if user already exists if id "$USERNAME" &>/dev/null; then echo "Error: User $USERNAME already exists" exit 1 fi # Check if department group exists if ! getent group "$DEPARTMENT" &>/dev/null; then echo "Warning: Group $DEPARTMENT does not exist. Creating it..." groupadd "$DEPARTMENT" fi # Create user echo "Creating user: $USERNAME" useradd -m -c "$FULLNAME" -G "$DEPARTMENT,allstaff" "$USERNAME" if [[ $? -eq 0 ]]; then echo "User $USERNAME created successfully" echo "Setting temporary password..." echo "$USERNAME:TempPass123!" | chpasswd chage -d 0 "$USERNAME" # Force password change at next login echo "" echo "User Details:" echo "=============" id "$USERNAME" echo "" echo "Password aging:" chage -l "$USERNAME" | head -5 else echo "Error: Failed to create user $USERNAME" exit 1 fi exit 0
Test the Script
# Make script executable chmod +x create_user.sh # Test with a new user sudo ./create_user.sh testuser "Test User" engineering # Verify user was created id testuser # Clean up test user (optional) sudo userdel -r testuser

Create a PowerShell script that creates a user with specified parameters.

New-CompanyUser.ps1
#Requires -RunAsAdministrator <# .SYNOPSIS Creates a new local user with company standard settings. .DESCRIPTION This script creates a new local user account with specified full name, description, and group membership. .PARAMETER Username The username for the new account .PARAMETER FullName The full name of the user .PARAMETER Department The department (used for description and group) .EXAMPLE .\New-CompanyUser.ps1 -Username jdoe -FullName "John Doe" -Department "IT" .NOTES Author: [Your Name] Date: [Date] #> param( [Parameter(Mandatory=$true)] [string]$Username, [Parameter(Mandatory=$true)] [string]$FullName, [Parameter(Mandatory=$true)] [string]$Department ) # Check if user already exists if (Get-LocalUser -Name $Username -ErrorAction SilentlyContinue) { Write-Error "User $Username already exists" exit 1 } # Generate temporary password $TempPassword = ConvertTo-SecureString "TempPass123!" -AsPlainText -Force try { # Create user Write-Host "Creating user: $Username" -ForegroundColor Cyan New-LocalUser -Name $Username ` -Password $TempPassword ` -FullName $FullName ` -Description "$Department Department" ` -PasswordNeverExpires:$false # Check if department group exists, create if not $GroupName = "${Department}Team" if (-not (Get-LocalGroup -Name $GroupName -ErrorAction SilentlyContinue)) { Write-Host "Creating group: $GroupName" -ForegroundColor Yellow New-LocalGroup -Name $GroupName -Description "$Department Department" } # Add to groups Add-LocalGroupMember -Group $GroupName -Member $Username Add-LocalGroupMember -Group "Users" -Member $Username Write-Host "" Write-Host "User created successfully!" -ForegroundColor Green Write-Host "=========================" -ForegroundColor Green Get-LocalUser -Name $Username | Format-List Name, FullName, Description, Enabled Write-Host "Group Memberships:" -ForegroundColor Cyan Get-LocalGroup | Where-Object { (Get-LocalGroupMember -Group $_.Name -ErrorAction SilentlyContinue).Name -match $Username } | Select-Object Name } catch { Write-Error "Failed to create user: $_" exit 1 } Write-Host "" Write-Host "Note: User must change password at first login" -ForegroundColor Yellow
Test the Script
# Run the script .\New-CompanyUser.ps1 -Username testuser -FullName "Test User" -Department "IT" # Verify user was created Get-LocalUser -Name testuser | Format-List * # Clean up test user (optional) Remove-LocalUser -Name testuser

💡 Tips for Success

  • Take screenshots immediately after each command
  • Include the command and its output in screenshots
  • Use clear, descriptive captions for each screenshot
  • Organize your report by lab part (A, B, C, D)
  • Explain what each command does, not just show it
  • Forgetting -a with usermod -G: This replaces all groups instead of adding
  • Editing sudoers directly: Always use visudo
  • Wrong GIDs: Double-check the GID assignments in the tables
  • Missing screenshots: Document every step as you go
  • Script permissions: Remember chmod +x for Bash scripts
  • PowerShell execution policy: Ensure it's set to allow script execution (e.g., Set-ExecutionPolicy RemoteSigned)
  • User already exists: Use userdel username to remove and recreate
  • Group not found: Create groups before adding users to them
  • sudo syntax error: Run sudo visudo -c to check all files
  • PowerShell access denied: Run PowerShell as Administrator
  • Script won't run: Check execution policy with Get-ExecutionPolicy and script path (e.g., .\script.ps1)
  • User not in group: Log out and back in on Linux for new group memberships to take effect.
