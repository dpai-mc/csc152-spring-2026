Introduction

Now that you understand Windows user account concepts, SIDs, and built-in accounts, it's time to learn how to create and manage users and groups. Windows provides two primary methods for user management: the graphical Computer Management console and PowerShell cmdlets.

Both methods accomplish the same tasks, but each has advantages in different scenarios. A skilled Windows administrator should be proficient with both approaches.

Learning Objectives

  1. Create user accounts using the Computer Management GUI
  2. Create and manage users using PowerShell *-LocalUser cmdlets
  3. Configure password options and account settings
  4. Create and manage local groups using GUI and PowerShell
  5. Add and remove users from groups
  6. Choose the appropriate management method for different scenarios

Two Paths, Same Destination

üñ±Ô∏è GUI (Computer Management)
  • Best for: Single users, learning, verification
  • Pros: Visual, discoverable, immediate feedback
  • Cons: Not scriptable, slow for bulk operations

‚ö° PowerShell
  • Best for: Automation, bulk operations, remote management
  • Pros: Scriptable, repeatable, documentable

  • Module: Microsoft.PowerShell.LocalAccounts

User Properties Tabs

After creating a user, you can access additional settings by double-clicking the user or right-clicking ‚Üí Properties:

  • Account enabled/disabled status

Group memberships for the user:

  • View current group memberships
  • Add... - Add user to additional groups
  • Remove - Remove user from selected group
üí° Default: New users are automatically added to the "Users" group.

User environment settings:

  • Sessions: Session timeout and reconnection settings
  • Remote Control: Allow/configure remote shadowing

PowerShell User Management CSLO 1

LocalAccounts Module Cmdlets

PowerShell provides the Microsoft.PowerShell.LocalAccounts module for managing local users and groups. Here are the key cmdlets:

Cmdlet Purpose
New-LocalUser Create a new local user account
Get-LocalUser View local user account information
Set-LocalUser Modify user account properties
Enable-LocalUser Enable a disabled account
Disable-LocalUser Disable an account
Remove-LocalUser Delete a user account
Rename-LocalUser Change username

Creating Users with PowerShell

PowerShell: Create user without password
# Create a basic user with no password (account will be disabled) PS> New-LocalUser -Name "jsmith" -Description "John Smith - Sales" -NoPassword Name Enabled Description ---- ------- ----------- jsmith False John Smith - Sales
‚ö†Ô∏è Note: Users created with -NoPassword are disabled by default and cannot log in until a password is set.
PowerShell: Create user with interactive password prompt
# Prompt for password securely (recommended method) PS> $Password = Read-Host -AsSecureString "Enter password for new user" Enter password for new user: ******** # Create the user with the secure password PS> New-LocalUser -Name "jsmith" ` -Password $Password ` -FullName "John Smith" ` -Description "Sales Department" Name Enabled Description ---- ------- ----------- jsmith True Sales Department
‚úì Best Practice: Using Read-Host -AsSecureString prevents the password from being displayed or stored in command history.
PowerShell: Create user with inline secure string
# Convert plain text to secure string (use only in secure scripts) PS> $Password = ConvertTo-SecureString "TempPass123!" -AsPlainText -Force # Create user with all common options PS> New-LocalUser -Name "egarcia" ` -Password $Password ` -FullName "Elena Garcia" ` -Description "HR Department" ` -AccountNeverExpires ` -PasswordNeverExpires:$false ` -UserMayNotChangePassword:$false
üîí Security Warning: Avoid using ConvertTo-SecureString with plain text passwords in production scripts. Use credential files, Azure Key Vault, or interactive prompts instead.
PowerShell: Complete new employee setup
# Step 1: Get password securely PS> $Password = Read-Host -AsSecureString "Enter temporary password" # Step 2: Create the user PS> New-LocalUser -Name "fthomas" ` -Password $Password ` -FullName "Frank Thomas" ` -Description "IT Support" # Step 3: Force password change at next logon PS> Set-LocalUser -Name "fthomas" -PasswordNeverExpires $false PS> $User = Get-LocalUser -Name "fthomas" PS> $User | Set-LocalUser -PasswordNeverExpires $false # Step 4: Add to groups PS> Add-LocalGroupMember -Group "Remote Desktop Users" -Member "fthomas" PS> Add-LocalGroupMember -Group "Backup Operators" -Member "fthomas" # Step 5: Verify PS> Get-LocalUser -Name "fthomas" | Format-List *

Viewing User Information

PowerShell: Query user information
# List all local users PS> Get-LocalUser Name Enabled Description ---- ------- ----------- Administrator True Built-in account for administering the computer DefaultAccount False A user account managed by the system. Guest False Built-in account for guest access jsmith True Sales Department egarcia True HR Department # Get specific user details PS> Get-LocalUser -Name "jsmith" | Format-List * AccountExpires : Description : Sales Department Enabled : True FullName : John Smith PasswordChangeableDate : 2/23/2026 10:30:00 AM PasswordExpires : 5/24/2026 10:30:00 AM UserMayChangePassword : True PasswordRequired : True PasswordLastSet : 2/23/2026 10:30:00 AM LastLogon : Name : jsmith SID : S-1-5-21-xxx-xxx-xxx-1001 # Find disabled accounts PS> Get-LocalUser | Where-Object { $_.Enabled -eq $false }

Modifying Users

PowerShell: Modify user accounts
# Change description PS> Set-LocalUser -Name "jsmith" -Description "Sales Manager" # Set password to never expire (service accounts) PS> Set-LocalUser -Name "svc_backup" -PasswordNeverExpires $true # Set account expiration date PS> Set-LocalUser -Name "contractor1" -AccountExpires (Get-Date).AddDays(90) # Disable account PS> Disable-LocalUser -Name "jsmith" # Enable account PS> Enable-LocalUser -Name "jsmith" # Rename user PS> Rename-LocalUser -Name "jsmith" -NewName "jdoe" # Reset password PS> $NewPassword = Read-Host -AsSecureString "Enter new password" PS> Set-LocalUser -Name "jsmith" -Password $NewPassword

Deleting Users

PowerShell: Delete user accounts
# Delete a single user PS> Remove-LocalUser -Name "jsmith" # Delete with confirmation prompt PS> Remove-LocalUser -Name "jsmith" -Confirm # Delete multiple users PS> "user1", "user2", "user3" | ForEach-Object { Remove-LocalUser -Name $_ }
‚ö†Ô∏è No Recycle Bin: Unlike files, deleted user accounts cannot be recovered. The user's profile folder (C:\Users\username) remains but the account and SID are permanently deleted.

Group Management CSLO 1

Built-in Local Groups Review

Windows Server includes several built-in groups with predefined permissions:

Group Purpose Key Rights
Administrators Full system control All rights
Users Standard users Run apps, use printers
Remote Desktop Users RDP access Log on via RDP
Backup Operators Backup/restore files Bypass file permissions for backup
Power Users Legacy (limited use) Minimal in modern Windows

# Add a single user to a group PS> Add-LocalGroupMember -Group "Finance" -Member "gmartin" # Add multiple users to a group PS> Add-LocalGroupMember -Group "Remote Desktop Users" -Member "egarcia", "fthomas", "gmartin" # Remove user from a group PS> Remove-LocalGroupMember -Group "Finance" -Member "gmartin" # View group members PS> Get-LocalGroupMember -Group "Administrators" ObjectClass Name PrincipalSource ----------- ---- --------------- User SERVER1\Administrator Local # View all groups a user belongs to PS> Get-LocalGroup | ForEach-Object { $members = Get-LocalGroupMember -Group $_.Name -ErrorAction SilentlyContinue if ($members.Name -contains "SERVER1\gmartin") { $_.Name } }

Real-World Scenarios

Best Method: GUI

For a single user, the GUI is faster‚Äîjust fill in the form and click Create. No need to remember cmdlet syntax.

üñ±Ô∏è GUI Approach: Open Computer Management ‚Üí New User ‚Üí Fill form ‚Üí Create (30 seconds)

Best Method: PowerShell

Bulk operations demand automation. Creating 50 users via GUI would take hours; PowerShell does it in seconds.

PowerShell: Bulk user creation from CSV
# CSV format: Username,FullName,Description,Department PS> $Users = Import-Csv -Path "C:\NewUsers.csv" PS> $Password = ConvertTo-SecureString "TempPass123!" -AsPlainText -Force PS> foreach ($User in $Users) { New-LocalUser -Name $User.Username ` -Password $Password ` -FullName $User.FullName ` -Description $User.Description Add-LocalGroupMember -Group $User.Department -Member $User.Username }

Best Method: PowerShell

Generate consistent reports and save them for compliance documentation.

PowerShell: User audit report
# Generate user audit report PS> Get-LocalUser | Select-Object Name, Enabled, LastLogon, PasswordLastSet, PasswordExpires | Export-Csv -Path "C:\Reports\UserAudit_$(Get-Date -Format 'yyyyMMdd').csv" # Find accounts that haven't logged in for 90 days PS> $CutoffDate = (Get-Date).AddDays(-90) PS> Get-LocalUser | Where-Object { $_.LastLogon -lt $CutoffDate -and $_.Enabled }

Best Method: Either (GUI slightly faster for one-off checks)

Key Takeaways

üìù Remember These Points:
  • Windows provides GUI (Computer Management) and PowerShell for user management
  • Use New-LocalUser, Get-LocalUser, Set-LocalUser, Remove-LocalUser for user operations
