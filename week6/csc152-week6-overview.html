📖 Week Overview

Week 6 transitions from command-line fundamentals to one of the most critical aspects of server administration: managing users and groups. Building directly upon Week 5's scripting skills, you will learn to create, modify, and manage user accounts and groups on both Linux and Windows Server platforms.

This week introduces essential security concepts including password policies, account restrictions, and the principle of least privilege through sudo and role-based access control. User and group administration forms the foundation of access control in multi-user environments—every organization must manage who can access systems, what resources they can use, and what actions they can perform.

📚 Learning Modules

Week 6 content is organized into 7 comprehensive learning modules that progressively build user and group administration skills across both Linux and Windows Server environments.

6.1 Linux User Account Fundamentals

Understanding user accounts on Linux: UIDs, /etc/passwd, /etc/shadow, account types, and the user creation process.

⏱️ 90 minutes

6.2 Linux User Management Commands

Hands-on with useradd, usermod, userdel, and passwd commands. Creating users with specific options and managing existing accounts.

⏱️ 90 minutes

6.3 Linux Group Management

Creating and managing groups with groupadd, groupmod, groupdel, and gpasswd. Understanding primary vs. supplementary groups.

⏱️ 90 minutes

6.4 Windows User Account Fundamentals

Understanding Windows user accounts: SIDs, local vs. domain accounts, built-in accounts, and the Windows security model.

⏱️ 90 minutes

6.5 Windows User & Group Management

Managing users and groups via Computer Management GUI and PowerShell cmdlets (New-LocalUser, Add-LocalGroupMember).

⏱️ 90 minutes

6.6 Password Policies & Security

Configuring password aging with chage, /etc/login.defs, Local Security Policy, and account lockout policies on both platforms.

⏱️ 90 minutes

6.7 Privilege Escalation & Least Privilege

Understanding and configuring sudo, the sudoers file, Windows UAC, User Rights Assignment, and implementing least privilege.

⏱️ 90 minutes
💡 Module Sequence: Complete modules in order, as each builds upon concepts from previous modules. Modules 6.1-6.3 focus on Linux, 6.4-6.5 on Windows, and 6.6-6.7 cover cross-platform security concepts.

🔑 Key Concepts

File Purpose Key Fields
/etc/passwd User account information username:x:UID:GID:GECOS:home:shell
/etc/shadow Encrypted passwords & aging username:hash:lastchg:min:max:warn:inactive:expire
/etc/group Group definitions groupname:x:GID:members
/etc/gshadow Secure group information groupname:password:admins:members
Command Purpose
useradd Create new user accounts
usermod Modify existing user accounts
userdel Delete user accounts
passwd Change user passwords
groupadd Create new groups
gpasswd Administer group memberships
chage Change password aging settings
visudo Safely edit sudoers file
Cmdlet Purpose
New-LocalUser Create new local user
Get-LocalUser View local user information
Set-LocalUser Modify local user properties
Remove-LocalUser Delete local user
New-LocalGroup Create new local group
Add-LocalGroupMember Add user to group
Get-LocalGroupMember View group members

Definition: Users, processes, and systems should have only the minimum permissions necessary to perform their job functions—nothing more.

Implementation Strategies:
  • Default to no access - Grant permissions only when explicitly needed
  • Use groups - Assign permissions to groups, not individual users
  • Separate accounts - Admins should have regular and admin accounts
  • Time-limited access - sudo sessions expire; use JIT access where possible
  • Regular reviews - Audit and remove unnecessary permissions
Platform Implementation:
  • Linux: sudo with granular sudoers rules in /etc/sudoers.d/
  • Windows: User Account Control (UAC), User Rights Assignment

Basic Format: user host=(runas) commands

Component Description Example
user Username or %groupname jsmith, %webadmins
host Hostname or ALL ALL, webserver1
(runas) Target user:group (root), (ALL:ALL)
commands Allowed commands ALL, /bin/systemctl
⚠️ Always use visudo! Never edit /etc/sudoers directly. The visudo command validates syntax before saving, preventing lockouts.

💡 Tips for Success

  • Plan before creating: Design user naming conventions and group structures before implementation
  • Document decisions: Keep notes on why you made specific choices for future reference
  • Test in safe environment: Use your lab VMs to experiment before applying to production
  • Always use -a with usermod -G: Forgetting -a replaces all groups instead of adding!
  • Verify changes: Use id username and getent to confirm configurations
  • usermod -G without -a: This replaces all supplementary groups. Always use usermod -aG
  • Editing sudoers directly: Always use visudo to prevent syntax errors that lock you out
  • Wrong GIDs: Double-check GID assignments match the lab requirements
  • Forgetting screenshots: Document every step as you complete it
  • Script permissions: Remember chmod +x for Bash scripts
  • PowerShell execution policy: Set to RemoteSigned if scripts won't run
  • Complete modules in order: Each module builds on previous concepts
  • Practice commands: Type commands yourself rather than copy-pasting
  • Compare platforms: Note similarities and differences between Linux and Windows
  • Create cheat sheets: Summarize key commands and concepts for quick reference
  • Use discussion forums: Ask questions and learn from classmates
  • Attend office hours: Get help early if you're stuck
  • Review requirements: Re-read the project rubric to ensure all items are addressed
  • Test everything: Verify all configurations work as expected
  • Document thoroughly: Include clear screenshots and explanations
  • Check formatting: Ensure professional presentation of your report
  • Submit early: Don't wait until the last minute—technical issues happen
  • Ask for help: Reach out if you're stuck on any component
