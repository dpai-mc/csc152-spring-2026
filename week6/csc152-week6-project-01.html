Project Overview

🎯 Project Objective

Configure a complete multi-user server environment demonstrating mastery of server fundamentals, operating system installation, command-line administration, and user/group management concepts covered in Weeks 1−6.

Project Description

In this project, you will design and implement a multi-user server environment for a fictional small business. You will apply the skills learned throughout the first six weeks of the course to create a functional server infrastructure with proper user organization, group-based access control, and documented administrative procedures.

This project simulates a real-world scenario where a system administrator must set up a new server to support multiple departments within an organization. You will make decisions about server configuration, user organization, security policies, and administrative procedures—all essential skills for IT professionals.

You have been hired as the IT administrator for TechStart Solutions, a growing technology consulting firm with 15 employees across four departments.

Company Background

TechStart Solutions is a small technology consulting firm that provides IT services to local businesses. The company has recently outgrown their informal IT setup and needs a proper server infrastructure to manage employee access, file sharing, and administrative tasks.

Organizational Structure

Department Employees Access Requirements
Executive 2 (CEO, CFO) Full system access, all department files
Engineering 5 Development tools, project files, limited admin access
Sales 4 CRM data, client files, marketing materials
Operations 4 HR files, financial records, company policies

Your Assignment

As the new IT administrator, you must:

  1. Set up a server (Linux or Windows Server) to serve as the primary file and authentication server
  2. Create user accounts for all 15 employees with appropriate naming conventions
  3. Organize users into groups that reflect the organizational structure

  5. Configure administrative access following the principle of least privilege
  6. Create at least one automation script to assist with administrative tasks
  7. Document the entire configuration for future reference and onboarding

4. Password Policy Configuration

Implement password policies appropriate for a business environment:

Minimum Requirements

  • Minimum password length: 12 characters
  • Password complexity: Must include uppercase, lowercase, numbers, and special characters
  • Password expiration: 90 days
  • Password history: Remember last 5 passwords
  • Account lockout: After 5 failed attempts

Linux Implementation (PAM)

# /etc/security/pwquality.conf
minlen = 12
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1

Windows Implementation (Group Policy or secpol.msc)

# Local Security Policy Settings
Minimum password length: 12 characters
Password must meet complexity requirements: Enabled
Maximum password age: 90 days
Enforce password history: 5 passwords
Account lockout threshold: 5 invalid attempts

5. Access Control Implementation

Implement the principle of least privilege:

Linux (sudo configuration)

  • Only IT administrators should have full sudo access
  • Engineering users may need specific sudo commands for development tools
  • Document all sudo configurations in /etc/sudoers.d/
# /etc/sudoers.d/techstart

# IT Administrators - full access
%grp_itadmin    ALL=(ALL:ALL) ALL

# Engineering - limited commands
%grp_engineering    ALL=(ALL) /usr/bin/systemctl status *
%grp_engineering    ALL=(ALL) /usr/bin/journalctl

Windows (Local Groups/Policies)

  • Only IT administrators should be in the Administrators group
  • Use built-in groups (Remote Desktop Users, Backup Operators) appropriately
  • Document group membership and access rights

6. Administrative Automation Script

Create at least one script that automates an administrative task:

Script Options (choose one or more)

  • User Creation Script: Automate new user account creation with proper group membership
  • Account Audit Script: List all users with their group memberships and last login
  • Password Expiry Report: Identify accounts with passwords expiring soon
  • System Information Script: Gather and report system configuration details

Script Requirements

  • Include comments explaining the script's purpose and usage
  • Include error handling for common issues
  • Follow scripting best practices covered in Week 5
  • Test the script and document the results

Example: Linux User Audit Script

#!/bin/bash
# user_audit.sh - Generate user account audit report
# Author: [Your Name]
# Date: [Date]
# Usage: sudo ./user_audit.sh

echo "=========================================="
echo "TechStart Solutions - User Audit Report"
echo "Generated: $(date)"
echo "=========================================="
echo ""

# List all human users (UID >= 1000)
echo "=== User Accounts ==="
awk -F: '$3 >= 1000 && $3 < 65534 {print $1}' /etc/passwd | while read user; do
    groups=$(groups "$user" 2>/dev/null | cut -d: -f2)
    lastlog=$(lastlog -u "$user" 2>/dev/null | tail -1 | awk '{print $4, $5, $6, $7}')
    echo "User: $user"
    echo "  Groups: $groups"
    echo "  Last Login: $lastlog"
    echo ""
done

Example: PowerShell User Audit Script

# User_Audit.ps1 - Generate user account audit report
# Author: [Your Name]
# Date: [Date]
# Usage: .\User_Audit.ps1

Write-Host "=========================================="
Write-Host "TechStart Solutions - User Audit Report"
Write-Host "Generated: $(Get-Date)"
Write-Host "=========================================="
Write-Host ""

# Get all local users
$users = Get-LocalUser | Where-Object { $_.Enabled -eq $true }

foreach ($user in $users) {
    Write-Host "User: $($user.Name)"
    
    # Get group memberships
    $groups = Get-LocalGroup | Where-Object { 
        (Get-LocalGroupMember $_.Name -ErrorAction SilentlyContinue).Name -contains "$env:COMPUTERNAME\$($user.Name)"
    }
    Write-Host "  Groups: $($groups.Name -join ', ')"
    Write-Host "  Last Logon: $($user.LastLogon)"
    Write-Host ""
}
