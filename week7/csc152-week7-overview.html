👋 Welcome to Week 7

▼

Welcome to Week 7! This week we shift focus from who can log in (Week 6: User & Group Administration) to what they can access once they’re in. File permissions and Access Control Lists (ACLs) are the gatekeepers of your server infrastructure — they determine who can read, write, and execute every file and directory on your systems.

You’ll work across both Linux and Windows Server environments, learning the permission models native to each operating system. By the end of this week, you’ll be able to configure, analyze, and troubleshoot file-level access controls that enforce the principle of least privilege — a foundational concept in server security.

Why This Matters

Misconfigured permissions are one of the most common causes of security breaches and operational failures in IT environments. Whether it’s a developer who accidentally exposes sensitive data with chmod 777 or an NTFS inheritance rule that grants unintended access, permission errors can have serious consequences. The skills you build this week are essential for every system administrator.

Where Week 7 Fits

Week 7 concludes Phase 1: Awareness & Context of the Splunk integration plan. You’ll get a preview of how permission changes generate security events that can be monitored and audited — a topic you’ll explore in depth during Projects 3 and 4.

🎯 Learning Objectives

▼

By the end of Week 7, you will be able to:

  1. Explain the Linux file permission model including owner, group, and others tiers
  2. Configure Linux permissions using both symbolic (chmod u+rwx,g+rx,o-rwx) and numeric/octal (chmod 750) notation
  3. Identify and configure special permission bits (SUID, SGID, Sticky Bit)
  4. Implement POSIX Access Control Lists (ACLs) on Linux using setfacl and getfacl
  5. Configure NTFS permissions on Windows Server using both the GUI and PowerShell
  6. Explain NTFS permission inheritance and how to manage it
  7. Analyze effective permissions when multiple permission entries apply to a user
  8. Troubleshoot common access denied scenarios on both Linux and Windows
  9. Describe how permission changes generate security audit events that can be monitored with tools like Splunk
  10. Apply the principle of least privilege to file system access design

📖 Learning Modules

▼

Week 7 content is organized into four learning modules that progressively build your file permission and access control skills across both Linux and Windows Server environments.

Module 7.1: Linux File Permissions Fundamentals ⏱ ~2.5 hrs

The foundation for all Linux access control. Covers the three-tier permission model and both symbolic and numeric notation.

  • Owner (u), Group (g), and Others (o) permission tiers
  • Read (r), Write (w), and Execute (x) permission types
  • Symbolic notation: chmod u+rwx,g=rx,o-rwx
  • Numeric/octal notation: chmod 750
  • Changing ownership with chown and chgrp
  • Default permissions and umask
  • Special permission bits: SUID, SGID, Sticky Bit
Module 7.2: Linux POSIX ACLs ⏱ ~2 hrs

Extends the standard permission model to support fine-grained access for multiple users and groups on a single resource.

  • Why standard permissions are sometimes insufficient
  • POSIX ACL concepts: named users, named groups, mask
  • Setting ACLs with setfacl
  • Viewing ACLs with getfacl
  • Default ACLs and inheritance on directories
  • The ACL mask and effective permissions
  • Removing and backing up ACLs
Module 7.3: Windows NTFS Permissions & ACLs ⏱ ~2.5 hrs

Covers the Windows NTFS permission model including standard and advanced permissions, inheritance, and PowerShell management.

  • NTFS permission levels: Full Control, Modify, Read & Execute, Read, Write
  • Standard vs. Advanced (special) permissions
  • Permission inheritance and propagation
  • Explicit vs. inherited permissions
  • Allow vs. Deny entries and precedence rules
  • Managing permissions with PowerShell (Get-Acl, Set-Acl)
  • Using icacls for command-line management
Module 7.4: Troubleshooting & Security Monitoring Preview ⏱ ~2 hrs

Brings it all together with practical troubleshooting scenarios and a preview of how permissions connect to security monitoring.

  • Common permission troubleshooting methodology
  • Linux troubleshooting: diagnosing access denied errors
  • Windows troubleshooting: effective permissions analysis
  • Cross-platform permission comparison
  • Security audit events related to permissions (Event IDs 4670, 4663, 4656)
  • Splunk Phase 1 preview: why centralized log monitoring matters
  • Connection to Projects 3 and 4

🔑 Key Concepts Preview

▼

Here is a high-level preview of the core concepts you’ll encounter this week. Detailed explanations, examples, and practice exercises are provided in each learning module.

Linux Permission Model

Concept Description Module
Three-tier model Owner (u), Group (g), Others (o) — each with r, w, x 7.1
Symbolic notation chmod u+rwx,g=rx,o-rwx file 7.1
Octal notation chmod 750 file (rwxr-x---) 7.1
Special bits SUID (4), SGID (2), Sticky (1) — e.g., chmod 2750 7.1
POSIX ACLs setfacl -m u:charlie:r file for per-user access 7.2

Windows NTFS Permission Model

Concept Description Module
Standard permissions Full Control, Modify, Read & Execute, Read, Write 7.3
Inheritance Child objects inherit parent ACL entries by default 7.3
Allow vs. Deny Deny always overrides Allow at the same level 7.3
Effective permissions Cumulative result of all applicable ACL entries for a user 7.3
PowerShell management Get-Acl, Set-Acl, icacls 7.3

Cross-Platform Comparison

Feature Linux Windows NTFS
Basic model Owner / Group / Others ACL entries per user/group
Extended access POSIX ACLs (setfacl) Built-in ACL system
Inheritance Default ACLs on directories Automatic parent-to-child propagation
Deny capability No explicit deny (remove access instead) Explicit Deny entries supported
CLI tools chmod, chown, setfacl icacls, Get-Acl / Set-Acl

💡 Tips for Success

▼
  1. Practice in your lab environment. Permissions are a hands-on skill. Reading about chmod 750 is not the same as running it, verifying with ls -la, and testing access as different users. Use your Proxmox VMs to experiment.
  2. Master the octal conversion. Being able to quickly convert between symbolic (rwxr-x---) and octal (750) notation is a fundamental sysadmin skill. Practice until it becomes second nature.
  3. Compare Linux and Windows side by side. Understanding the similarities and differences between the two permission models will deepen your understanding of both. Module 7.4 provides a cross-platform comparison.
  4. Think “least privilege” first. When designing permissions, start with no access and add only what’s needed. It’s much easier to grant additional access than to discover you’ve been too permissive.
  5. Document as you go. Whether for Lab 7 or Project 1, take screenshots and record the commands you use. Good documentation saves time when troubleshooting later.
  6. Use Discussion 7 to learn from others. Permission mistakes are universal. Reading about your classmates’ troubleshooting experiences will expose you to scenarios you might not encounter on your own.
  7. Connect to Project 1. Every permission concept you learn this week can be immediately applied to your Project 1 server configuration. Work on both simultaneously for maximum efficiency.
