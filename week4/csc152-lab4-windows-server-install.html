Lab 4: Windows Server Installation & Server Manager

Course: CSC 152 - Network OS and Administration
Week: 4 of 17
Points: 27 points
Due Date: February 17, 2026, 11:59 PM
Estimated Time: 3-4 hours hands-on + 1-2 hours documentation

Lab Objectives

By completing this lab, you will be able to:

  • Install Windows Server in a virtual environment (Proxmox VM)
  • Perform comprehensive post-installation configuration tasks
  • Explore Server Manager and its key administrative features
  • Add common server roles and features to Windows Server
  • Configure basic remote management access (Remote Desktop Protocol and PowerShell Remoting)
  • Compare Windows Server administration with Linux server administration approaches

CSLO Alignment

CSLO 1: Manage and maintain servers, including operating system (OS) configuration, access control and virtualization.

Lab Overview

This lab guides you through the installation and initial configuration of Windows Server in your Proxmox virtual environment. You will then explore Server Manager, add essential server roles and features, and configure remote management. This hands-on experience will solidify your understanding of Windows Server's administrative interfaces and best practices, drawing comparisons to your Linux experience from Week 3.

Important Notes:

  • You must have a functional Proxmox VE environment set up from Week 1.
  • Refer to Modules 4.1-4.8 for conceptual background and detailed instructions.
  • Windows updates can be time-consuming. Plan your time accordingly.
  • Always obscure passwords in screenshots.

Part A: Windows Server Installation (8 points)

In this part, you will prepare your Proxmox VM, install Windows Server, and perform essential post-installation configuration.

1. Pre-Installation Planning and VM Configuration (2 points)

Document your planning decisions and VM setup in Proxmox.

  1. VM Resource Allocation:
    • CPU Cores: State the number of CPU cores allocated to your Windows Server VM. Justify your choice based on the principles learned in Week 2 (e.g., balancing performance and host resources).
    • RAM: State the amount of RAM allocated (minimum 4 GB). Explain why you chose this amount, considering server roles and performance.
    • Storage: State the allocated storage size (minimum 60 GB) and type (e.g., SCSI, SATA, VirtIO Block).
    • Network Configuration Plan: Outline the static IP address, subnet mask, default gateway, and DNS servers you plan to use for your server. Explain why static IP is preferred for servers.
  2. Screenshot of VM Configuration:
    • Include a screenshot of your Windows Server VM's hardware settings in Proxmox.
    • Caption: "Proxmox VM Hardware Configuration for Windows Server"
  3. Windows Server Edition Choice:
    • State the Windows Server edition you chose to install (e.g., Standard, Datacenter, Desktop Experience, Server Core).
    • Justification: Provide a 100-150 word justification for your edition choice, referencing concepts from Module 4.1 (e.g., virtualization rights, GUI vs. CLI, target environment).

2. Installation Process Documentation (4 points)

Document the key steps of the Windows Server installation process.

  1. Installation Steps:
    • Include screenshots of at least **four (4)** key steps during the Windows Server installation (e.g., language selection, license terms acceptance, custom install option, drive partitioning, Administrator password setup).
    • Ensure each screenshot has a clear caption.
  2. Initial Boot-up:
    • Include a screenshot of your Windows Server's desktop after the first successful boot and login.
    • Caption: "Windows Server Desktop After Initial Installation"

3. Post-Installation Configuration (2 points)

Perform and document the essential initial configuration tasks for your server.

  1. Computer Name and Domain:
    • Change the default computer name to follow the convention: CSC152-WS-YourName (e.g., CSC152-WS-DavidPai).
    • Confirm the server is in a Workgroup (for now).
    • Screenshot: Show the System Properties window with the new computer name.
    • Caption: "Windows Server Computer Name Changed"
  2. Network Configuration:
    • Configure your primary network adapter with the static IP address, subnet mask, default gateway, and DNS servers as planned in Part A.1.
    • Screenshot: Show the TCP/IPv4 Properties dialog box with your static IP configuration.
    • Caption: "Static IP Configuration for Primary Network Adapter"
  3. Windows Update:
    • Check for available updates.
    • Install all critical and security updates. This may require multiple restarts.
    • Screenshot: Show the Windows Update screen after updates have been checked and installed, indicating "You're up to date" or showing updates installed.
    • Caption: "Windows Update Status After Patching"
  4. Time Zone and Time Synchronization:
    • Set the correct time zone for your location.
    • Verify that time synchronization is configured (usually automatic).
    • Screenshot: Show the Date and Time settings with the correct time zone.
    • Caption: "Windows Server Time Zone Configuration"

Part C: Server Roles and Features (6 points)

In this part, you will install a common server role and a few features using both GUI and PowerShell, demonstrating your understanding from Module 4.4.

1. File Server Role Installation (3 points)

Install the File Server role on your Windows Server.

  1. Installation via Server Manager:
    • Use the "Add Roles and Features Wizard" in Server Manager to install the "File Server" role service (under File and Storage Services). Accept any required dependencies.
    • Screenshot: Show the "Confirmation" page of the wizard just before clicking "Install", clearly listing "File Server" as selected.
    • Caption: "Confirming File Server Role Installation"
  2. Verification:
    • After installation, verify the File Server role is installed and appears in Server Manager.
    • Screenshot: Show the Server Manager Dashboard with the "File and Storage Services" tile visible and healthy.
    • Caption: "File Server Role Verified on Dashboard"
  3. Basic File Share Creation (Optional, for extra practice):
    • Create a simple shared folder (e.g., C:\Share) and configure basic share permissions.
    • Screenshot: Show the share properties in File Explorer.
    • Caption: "Basic File Share Configuration"

2. Additional Features Installation and Testing (3 points)

Install two specific features using PowerShell and verify their functionality.

  1. Install Telnet Client Feature:
    • Use the PowerShell cmdlet Install-WindowsFeature to install the "Telnet Client" feature.
    • PowerShell Command & Output: Include the command used and its successful output.
  2. Test Telnet Client:
    • From your Windows Server, use the telnet command to test connectivity to a public web server (e.g., telnet google.com 80).
    • PowerShell Command & Output: Include the command used and its output (showing connection or failure).
    • Caption: "Testing Telnet Client Connectivity"
  3. Install Windows Server Backup Feature:
    • Use PowerShell to install the "Windows Server Backup" feature.
    • PowerShell Command & Output: Include the command used and its successful output.
  4. Verify Windows Server Backup:
    • Open the Windows Server Backup console (from Server Manager → Tools).
    • Screenshot: Show the main interface of the Windows Server Backup console.
    • Caption: "Windows Server Backup Console"

Part D: Remote Management (3 points)

Configure and test remote management access to your Windows Server, applying concepts from Module 4.5.

1. Remote Desktop Configuration and Testing (2 points)

  1. Enable Remote Desktop:
    • Enable Remote Desktop on your Windows Server via GUI (System Properties) or PowerShell. Ensure Network Level Authentication (NLA) is enabled.
    • Screenshot: Show the "Remote Desktop" section of the System Properties dialog, confirming it's enabled with NLA.
    • Caption: "Remote Desktop Enabled with NLA"
  2. Test RDP Connection:
    • From your host computer (or another VM), use the Remote Desktop Connection client (mstsc.exe) to connect to your Windows Server VM.
    • Screenshot: Show a screenshot of your host computer's desktop with the RDP session to your Windows Server actively displayed.
    • Caption: "Successful Remote Desktop Connection"

2. PowerShell Remoting Configuration (1 point)

Enable and test basic PowerShell Remoting.

  1. Enable PowerShell Remoting:
    • On your Windows Server, open PowerShell as Administrator and run the command to enable PowerShell Remoting.
    • PowerShell Command & Output: Include the command used and its successful output.
  2. Test PowerShell Remoting:
    • From your host computer (if PowerShell is installed) or another Windows VM, open PowerShell and use Enter-PSSession to connect to your Windows Server VM.
    • Run a simple command (e.g., Get-Service) within the remote session.
    • PowerShell Command & Output: Include the Enter-PSSession command, the simple command, and their outputs.
    • Caption: "Testing PowerShell Remoting"

Conclusion

After completing all parts of Lab 4, you should have a fully installed and initially configured Windows Server, ready for further administration tasks. You've practiced using both graphical and command-line tools, reinforcing the versatility required for modern server management.
