1. Windows Server Evolution and History

▼

Understanding the evolution of Windows Server helps you appreciate its current capabilities and design decisions. Windows Server has evolved from simple network operating systems to sophisticated platforms supporting modern cloud and hybrid infrastructures.

1.1 Windows Server Timeline

1993: Windows NT 3.1

First true 32-bit Windows server OS. Introduced domain-based networking and NTFS file system. Marked Microsoft's entry into enterprise server market.

2000: Windows 2000 Server

Introduced Active Directory, revolutionizing Windows network management. Added Group Policy for centralized management. Based on NT kernel with improved stability.

2003: Windows Server 2003

Enhanced Active Directory capabilities. Improved security features. Better performance and scalability. Widely adopted in enterprises.

2008: Windows Server 2008

Introduced Server Core installation option (no GUI). Hyper-V virtualization platform. PowerShell for automation. Server Manager redesigned.

2012: Windows Server 2012

Significant virtualization improvements. Storage Spaces for software-defined storage. Improved Hyper-V with live migration. Modern Server Manager interface.

2016: Windows Server 2016

Containers and Nano Server. Enhanced security features (Shielded VMs). Storage Replica for disaster recovery. Software-defined networking.

2019: Windows Server 2019

Hybrid cloud integration with Azure. Windows Admin Center. Kubernetes support. Enhanced security with Windows Defender ATP. Storage Migration Service.

2022: Windows Server 2022 (Current)

Secured-core server capabilities. Advanced multi-layer security. HTTPS and TLS 1.3 by default. SMB compression and encryption. Enhanced hybrid capabilities with Azure Arc.

1.2 Key Innovations Through Versions

Active Directory (2000) ▼

Revolutionary Impact: Active Directory transformed how organizations manage users, computers, and resources.

  • Before AD: Decentralized user databases, manual account management
  • After AD: Centralized directory, single sign-on, Group Policy
  • Legacy: Still the foundation of Windows networking (Week 11 topic)
Server Core (2008) ▼

Paradigm Shift: Minimal installation without GUI, similar to Linux server approach.

  • Benefits: Smaller attack surface, reduced patching, lower resource usage
  • Challenge: Required command-line proficiency
  • Impact: Influenced modern "infrastructure as code" practices
  • Today: Recommended for production environments
PowerShell (2008) ▼

Automation Revolution: Powerful scripting language for Windows administration.

  • Capability: Automate virtually any Windows administrative task
  • Design: Object-oriented, .NET-based, consistent verb-noun syntax
  • Remote Management: PowerShell Remoting for managing multiple servers
  • Adoption: Now essential skill for Windows administrators
  • Course Coverage: Week 5 focuses on PowerShell fundamentals
Hyper-V (2008) ▼

Virtualization Platform: Microsoft's Type 1 hypervisor competing with VMware.

  • Free with Windows Server: No separate licensing cost
  • Features: Live migration, failover clustering, replication
  • Comparison to Proxmox: Similar capabilities, different approach
    • Proxmox: Open-source, Linux-based (KVM)
    • Hyper-V: Proprietary, Windows-integrated
    • Both: Enterprise-grade virtualization
Hybrid Cloud Integration (2019-2022) ▼

Cloud-First Strategy: Deep integration with Microsoft Azure.

  • Azure Arc: Manage on-premises servers through Azure portal
  • Hybrid Identity: Synchronize on-premises AD with Azure AD
  • Azure Backup: Cloud-based backup solutions
  • Azure File Sync: Sync on-premises files to cloud
  • Trend: Blurring line between on-premises and cloud

1.3 Current Version: Windows Server 2022

Windows Server 2022, released in August 2021, is the current Long-Term Servicing Channel release and the focus for this course.

Key Features and Improvements:

  • Secured-Core Server: Hardware-based security features protecting firmware and boot process
  • Enhanced Security:
    • HTTPS and TLS 1.3 enabled by default
    • Improved Windows Defender integration
    • DNS-over-HTTPS support
    • SMB AES-256 encryption
  • Performance Improvements:
    • SMB compression for bandwidth optimization
    • Faster storage performance
    • Improved container support
  • Hybrid Cloud:
    • Deeper Azure integration
    • Azure Automanage for servers
    • Hotpatching (Azure Edition) for updates without reboot
  • Management:
    • Windows Admin Center improvements
    • Enhanced PowerShell capabilities
    • Better hardware compatibility

Why Windows Server 2022 for This Course

We focus on Windows Server 2022 (or 2019 if 2022 unavailable) because:

  • Current technology with long support lifecycle (2031 for 2022)
  • Represents modern server administration practices
  • Skills directly transferable to workplace environments
  • Evaluation versions freely available for education
  • Extensive documentation and community support

2. Windows Server Editions Explained

▼

Microsoft offers multiple editions of Windows Server to meet different organizational needs and budgets. Understanding the differences helps you choose the appropriate edition for specific scenarios.

2.1 Windows Server Standard Edition

Windows Server Standard
The cost-effective edition designed for physical or minimally virtualized environments. Provides core Windows Server functionality with limited virtualization rights.

Key Characteristics:

  • Installation Options: Both Desktop Experience (GUI) and Server Core available
  • Virtualization Rights: License covers 1 physical server + 2 virtual instances
  • Core Features: All standard roles and features included
  • Licensing Model: Per-core licensing (16-core minimum)
  • Target Market: Small to medium businesses, branch offices, specific-purpose servers
  • Cost: Lower than Datacenter edition

When to Choose Standard:

  • Running 2 or fewer VMs per physical host
  • Dedicated physical servers for specific roles
  • Budget-conscious deployments
  • Branch office servers with limited virtualization
  • Single-purpose servers (file server, print server, etc.)

What's NOT Included:

  • Unlimited virtualization (limited to 2 VMs)
  • Storage Spaces Direct (software-defined storage)
  • Storage Replica (storage replication)
  • Shielded Virtual Machines
  • Network Controller for SDN

2.2 Windows Server Datacenter Edition

Windows Server Datacenter
The premium edition designed for highly virtualized datacenters and software-defined infrastructure. Provides unlimited virtualization rights and advanced features.

Key Characteristics:

  • Installation Options: Both Desktop Experience and Server Core available
  • Virtualization Rights: Unlimited virtual instances on licensed physical server
  • Advanced Features: Includes all Standard features plus datacenter-specific capabilities
  • Licensing Model: Per-core licensing (16-core minimum)
  • Target Market: Large enterprises, cloud providers, highly virtualized environments
  • Cost: Significantly higher than Standard, but cost-effective for many VMs

When to Choose Datacenter:

  • Running more than 2 VMs per physical host
  • Hyper-V virtualization hosts
  • Software-defined datacenter deployments
  • Need for advanced features (Storage Spaces Direct, Shielded VMs)
  • Cloud service providers
  • Dense virtualization scenarios (10+ VMs per host)

Exclusive Datacenter Features:

  • Storage Spaces Direct: Software-defined storage using local disks
  • Storage Replica: Block-level replication between servers
  • Shielded Virtual Machines: Enhanced VM security with encryption
  • Software-Defined Networking: Network Controller for SDN
  • Host Guardian Service: Attestation for shielded VMs

2.3 Windows Server Essentials Edition

Windows Server Essentials
A simplified, small-business-focused edition with pre-configured roles and streamlined management. Limited to 25 users and 50 devices.

Key Characteristics:

  • User/Device Limits: Maximum 25 users and 50 devices
  • Simplified Management: Dashboard-based interface for non-technical users
  • Pre-Configured Roles: File sharing, backup, remote access ready out-of-box
  • No Virtualization Rights: Physical installation only
  • Single Domain: Cannot be domain controller in multi-domain forest
  • Cost: Lower entry price point for small businesses

When to Choose Essentials:

  • Very small businesses (under 25 employees)
  • Limited IT expertise on staff
  • Simple server needs (file sharing, backup, user management)
  • Budget-constrained organizations
  • First-time server deployment

Limitations to Consider:

  • Cannot exceed 25 user / 50 device limit
  • Cannot be additional domain controller
  • No virtualization rights
  • Limited scalability
  • Fewer advanced features

Essentials in This Course

We will not use Essentials edition in this course. We'll focus on Standard and Datacenter editions, which provide full server capabilities and better align with enterprise environments you'll encounter professionally.

3. Licensing Models and Compliance

▼

Windows Server licensing can be complex. Understanding licensing models helps organizations remain compliant and make cost-effective decisions.

3.1 Core-Based Licensing Model

Since Windows Server 2016, Microsoft uses a per-core licensing model for both Standard and Datacenter editions.

Core-Based Licensing
Licensing based on the number of physical processor cores in the server. Each license covers 2 cores, with a minimum of 16 cores (8 licenses) required per server.

How It Works:

  • Minimum Purchase: 16-core license pack (minimum per server)
  • Additional Cores: Purchase 2-core packs for servers with more than 16 cores
  • Per Physical Server: License applies to physical hardware, not VMs
  • All Physical Cores: Must license all cores in all processors

Licensing Calculation Examples:

Example 1: 16-Core Server ▼

Server Configuration: 2 processors × 8 cores each = 16 total cores

License Required: 1 × 16-core license pack (minimum)

Cost: Base license price

Virtualization Rights:

  • Standard: Can run 2 VMs
  • Datacenter: Can run unlimited VMs
Example 2: 24-Core Server ▼

Server Configuration: 2 processors × 12 cores each = 24 total cores

License Required: 1 × 16-core pack + 1 × 8-core pack = 24 cores licensed

Calculation:

  • 16-core minimum pack
  • Plus 8 additional cores (4 × 2-core packs)

Cost: Base license + additional core licenses

Example 3: Cost-Effectiveness Analysis ▼

Scenario: Hyper-V host with 32 cores running 10 VMs

Option A - Standard Edition:

  • Need 5 × Standard licenses (each covers 2 VMs)
  • Each Standard license: 32 cores = 16-core pack + 16-core pack
  • Total: 5 × (cost of 32-core Standard) = Very expensive

Option B - Datacenter Edition:

  • Need 1 × Datacenter license
  • 32 cores = 16-core pack + 16-core pack
  • Covers unlimited VMs
  • Total: 1 × (cost of 32-core Datacenter)

Break-Even Point: Datacenter becomes cost-effective at approximately 13+ VMs (varies by pricing)

3.2 Client Access Licenses (CALs)

In addition to server licenses, Windows Server requires Client Access Licenses for users or devices connecting to the server.

Client Access License (CAL)
A license that grants a user or device the right to access Windows Server services. Required in addition to the server license itself.

CAL Types:

  • User CAL: Assigned to a specific user who can access the server from any device
    • Best for: Users with multiple devices
    • Example: Employee with laptop, desktop, and phone
  • Device CAL: Assigned to a specific device; any user of that device can access server
    • Best for: Shared workstations, kiosks, shift workers
    • Example: Factory floor computer used by multiple shifts

CAL Requirements:

  • Required for accessing file shares, print services, Active Directory
  • NOT required for anonymous internet services (public website)
  • Some roles require additional CALs (Remote Desktop Services RDS CALs)
  • CAL version must match or exceed server version

CAL Compliance

Organizations must purchase appropriate number and type of CALs. Microsoft audits can result in significant penalties for non-compliance. Proper CAL management is a critical IT responsibility.

3.3 Evaluation Licenses for Learning

Microsoft provides evaluation versions of Windows Server for testing, learning, and proof-of-concept deployments.

Evaluation Version Characteristics:

  • Duration: 180 days (6 months) full functionality
  • Features: Identical to full licensed version
  • Conversion: Can convert to licensed version without reinstalling
  • Extensions: Can extend evaluation period (limited times)
  • Expiration: After 180 days, server reboots every hour until activated
  • No CALs Required: Evaluation doesn't require separate CALs
  • Download: Free from Microsoft Evaluation Center

4. Servicing Channels: LTSC vs. SAC

▼

Microsoft offers two servicing channels for Windows Server, each with different update cadences and support lifecycles.

4.1 Long-Term Servicing Channel (LTSC)

Long-Term Servicing Channel (LTSC)
Traditional Windows Server release model with major versions every 2-3 years and 10 years of support (5 years mainstream + 5 years extended).

LTSC Characteristics:

  • Release Cadence: New major version every 2-3 years
  • Support Duration: 10 years total (5 mainstream + 5 extended)
  • Updates: Monthly security and quality updates only
  • Feature Updates: No new features during lifecycle
  • Stability: Highly stable, predictable environment
  • Installation Options: Both Desktop Experience and Server Core
  • Editions: Standard, Datacenter, Essentials

LTSC Examples:

  • Windows Server 2022 (released August 2021, supported until 2031)
  • Windows Server 2019 (released October 2018, supported until 2029)
  • Windows Server 2016 (released September 2016, supported until 2027)

When to Choose LTSC:

  • Production servers requiring long-term stability
  • Environments with change control processes
  • Applications requiring specific server versions
  • Organizations preferring predictable update schedules
  • Most traditional server workloads
  • Recommendation: LTSC is appropriate for 95% of organizations

4.2 Semi-Annual Channel (SAC) - Deprecated

Semi-Annual Channel (SAC)
Rapid release model with new versions every 6 months and 18 months of support. Note: Microsoft discontinued SAC for Windows Server as of version 20H2.

SAC Characteristics (Historical):

  • Release Cadence: Every 6 months (spring and fall)
  • Support Duration: 18 months only
  • Installation Option: Server Core only (no Desktop Experience)
  • Features: Rapid access to new features and innovations
  • Target: Containers, microservices, modern application platforms

SAC Discontinuation

Microsoft discontinued the Semi-Annual Channel for Windows Server after version 20H2. Organizations should use:

  • LTSC: For traditional server workloads
  • Azure: For rapid innovation and cloud-native applications
  • Containers: For microservices and modern app platforms

This simplifies the Windows Server landscape - focus on LTSC for all server deployments.

5. Windows Server vs. Linux: Strategic Comparison

▼

Having learned about Linux servers in Week 3, you can now compare the two dominant server operating systems. Understanding when to choose each is a critical skill for IT professionals.

5.1 Comprehensive Comparison

Aspect Windows Server Linux Server
Cost Licensing fees (per-core + CALs) Free (most distributions), optional paid support
Interface GUI (Desktop Experience) or CLI (Server Core) Primarily CLI, GUI optional
Management Tools Server Manager, MMC snap-ins, PowerShell Command-line tools, text config files, optional web GUIs
Active Directory Native, full-featured AD DS Samba AD, FreeIPA, OpenLDAP (alternatives)

5.2 When to Choose Windows Server

Windows Server is the Better Choice When:

  • Microsoft Ecosystem: Organization uses Microsoft technologies
    • Active Directory for centralized user management
    • Microsoft SQL Server databases
    • Exchange Server for email
    • SharePoint for collaboration
    • .NET applications
  • GUI Preference: Staff prefers graphical management tools
  • Vendor Support: Organization requires commercial support contracts
  • Integration Needs: Deep integration with Windows desktops
  • Specific Applications: Applications only available for Windows
  • Existing Infrastructure: Already invested in Windows ecosystem
  • Compliance: Regulatory requirements for commercial support

5.3 When to Choose Linux Server

Linux is the Better Choice When:

  • Cost Sensitivity: Budget constraints or large-scale deployments
  • Web Services: Web servers, application servers, containers
  • Cloud Deployments: Cloud-native applications and microservices
  • Customization Needs: Require extensive customization
  • Open Source Philosophy: Organization prefers open-source solutions
  • Resource Efficiency: Need minimal resource footprint
  • Automation Focus: Heavy scripting and automation requirements
  • Specific Applications: Applications only available for Linux
  • Container Platforms: Docker, Kubernetes, containerized workloads

5.4 Hybrid Approaches

Many organizations use both Windows and Linux servers, choosing the best tool for each job:

Example Hybrid Infrastructure ▼

Typical Enterprise Deployment:

  • Windows Server:
    • Active Directory domain controllers
    • Exchange email servers
    • SQL Server databases (some)
    • File servers with Windows integration
    • SharePoint collaboration
  • Linux Server:
    • Web servers (Apache, Nginx)
    • Application servers (Java, Python, Node.js)
    • Container platforms (Docker, Kubernetes)
    • Database servers (MySQL, PostgreSQL)
    • Monitoring and logging infrastructure

Integration: Both environments can coexist and integrate (Samba for file sharing, LDAP authentication, etc.)

Skills Transferability ▼

Learning both Windows and Linux makes you more valuable:

  • Fundamental Concepts: User management, networking, storage, security principles are universal
  • Troubleshooting Skills: Methodologies transfer between platforms
  • Career Flexibility: Can work in diverse environments
  • Better Decisions: Choose the right tool for each job
  • Cross-Platform Integration: Understand how to make them work together

5.5 Cost Analysis

Total Cost of Ownership (TCO) Considerations:

Windows Server Costs ▼

Direct Costs:

  • Server license (Standard or Datacenter, per-core)
  • Client Access Licenses (User or Device CALs)
  • Additional role-specific CALs (RDS, etc.)
  • Software Assurance (optional, for upgrade rights)

Indirect Costs:

  • Higher hardware requirements (more RAM, storage)
  • Management tools (some third-party tools have costs)
  • Training for administrators

Benefits:

  • Commercial support from Microsoft
  • Integrated management tools
  • Potentially lower administration costs (GUI ease)
  • Extensive partner ecosystem
Linux Server Costs ▼

Direct Costs:

  • Operating system: Free (Ubuntu, Rocky Linux, Debian)
  • Optional: Commercial support subscriptions (Red Hat, SUSE)
  • No CALs required

Indirect Costs:

  • Administrator training (CLI proficiency required)
  • Potentially higher administration time initially
  • Third-party support if needed

Benefits:

  • No licensing costs for OS
  • Lower hardware requirements
  • Free updates and patches
  • Community support available
  • Highly customizable

Cost Analysis Conclusion

TCO isn't just about licensing costs:

  • Small Scale: Linux often cheaper overall
  • Large Scale: Cost difference narrows (administration costs matter more)
  • Microsoft Ecosystem: Windows Server TCO may be lower due to integration
  • Web/Cloud: Linux typically has lower TCO
  • Best Practice: Choose based on requirements, not just license cost
