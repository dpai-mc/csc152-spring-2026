Topic: Windows Server Administration

Week Overview

Week 4 transitions from Linux server administration to Windows Server environments, providing students with a comparative perspective on server operating systems. Students will install Windows Server, explore Server Manager as the central administrative interface, understand the distinction between server roles and features, and configure remote management capabilities. This week emphasizes the graphical and PowerShell-based administration approaches that distinguish Windows Server from Linux, while highlighting the common underlying server administration concepts that transcend operating system choice.

CSLO 1: Manage and maintain servers, including operating system (OS) configuration, access control and virtualization

Learning Objectives

By the end of Week 4, students will be able to:

1. Install and perform initial configuration of Windows Server
2. Navigate Server Manager and identify key administrative features
3. Distinguish between server roles and features and their purposes
4. Configure basic remote management access (Remote Desktop)
5. Compare and contrast Windows Server with Linux server administration
6. Understand Windows Server licensing models and editions
7. Perform post-installation configuration tasks
8. Document Windows Server configurations professionally

Key Concepts Summary

Windows Server Editions

  • Standard: Up to 2 VMs, general-purpose
  • Datacenter: Unlimited VMs, software-defined datacenter
  • Core-based licensing: Minimum 16 cores per server

Server Manager

  • Central management console
  • Role and feature installation
  • Multi-server management
  • Best Practices Analyzer

Roles vs. Features

  • Roles: Primary server functions (e.g., File Server, DNS)
  • Features: Supporting capabilities (e.g., .NET Framework, Telnet Client)

Remote Management

  • Remote Desktop (RDP): Port 3389
  • Windows Admin Center: Web-based
  • PowerShell Remoting: Command-line
  • MMC snap-ins: GUI tools
