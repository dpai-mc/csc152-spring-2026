1. Introduction to Server Manager

â–¼

Server Manager is the central administration console for Windows Server, providing a unified interface for managing server configuration, roles, features, and health. It automatically launches when you log in to Windows Server and serves as your primary starting point for administrative tasks.

1.1 What is Server Manager?

Server Manager
A management console that provides a centralized location for viewing server information, changing server roles and features, and performing management tasks. It consolidates administrative tools, wizards, and system information in a single interface.

Key Purposes of Server Manager:

  • Centralized Management: Single interface for multiple administrative tasks
  • Server Health Monitoring: Real-time view of server status, events, and performance
  • Role and Feature Management: Install, configure, and remove server roles and features
  • Quick Configuration Access: Fast access to common server settings
  • Multi-Server Management: Manage multiple servers from one console (advanced)
  • Best Practices Guidance: Built-in analyzer to identify configuration issues

1.2 Server Manager vs. Linux Administration

Windows Server Manager vs. Linux Administration Approaches
Aspect Windows Server Manager Linux Administration
Primary Interface Graphical User Interface (GUI) Command-Line Interface (CLI)
Centralized Console Server Manager provides single interface Multiple tools and commands (no single GUI)
Configuration Files Registry and GUI settings (some config files) Text-based configuration files in /etc/
Service Management Services console and PowerShell systemctl command
Package/Role Installation Server Manager wizard apt, dnf, yum commands
Remote Management Remote Desktop, Server Manager remote SSH (Secure Shell)
Learning Curve Easier for GUI-familiar users Steeper initial learning curve
Automation PowerShell scripting Bash scripting
Documentation Built-in help, tooltips, wizards Man pages, info, online docs

The Best of Both Worlds

Modern Windows Server administration combines GUI tools like Server Manager with powerful PowerShell command-line capabilities. This gives administrators flexibility:

  • Use Server Manager for initial setup and exploration
  • Use PowerShell for automation and remote management
  • Choose the best tool for each task

Similarly, Linux has evolved to include GUI management tools (Cockpit, Webmin) while maintaining its powerful CLI foundation.

1.3 Launching Server Manager

Server Manager automatically launches when you log in to Windows Server. If you close it, you can relaunch it several ways:

Method 1: Start Menu â–¼
  1. Click the Start button (Windows logo)
  2. Scroll to find "Server Manager" in the app list
  3. Click to launch

Tip: You can pin Server Manager to the taskbar for quick access (right-click â†’ Pin to taskbar).

Method 2: Windows Search â–¼
  1. Click the search icon in the taskbar (magnifying glass)
  2. Type "Server Manager"
  3. Press Enter or click the result
Method 3: Run Dialog â–¼
  1. Press Windows Key + R to open Run dialog
  2. Type: servermanager
  3. Press Enter

Fastest method for experienced administrators!

Method 4: PowerShell â–¼ 
# Launch Server Manager from PowerShell
Start-Process ServerManager

# Or simply:
servermanager

Useful when working in PowerShell and need to open Server Manager.

1.4 Controlling Automatic Launch

By default, Server Manager launches automatically at login. You can change this behavior:

  1. Open Server Manager
  2. Click the Manage menu (top right)
  3. Select Server Manager Properties
  4. Check or uncheck: "Do not start Server Manager automatically at logon"
  5. Click OK

Recommendation

For learning purposes, keep automatic launch enabled. It ensures you always have quick access to Server Manager and helps you become familiar with the interface. In production environments, experienced administrators often disable automatic launch to reduce resource usage.

3.3 Events Section

The Events section displays recent events from the Windows Event Log, filtered to show relevant administrative events.

Understanding Event Levels â–¼

Events are categorized by severity level:

  • Error (Red): Significant problem requiring attention (service failure, system error)
  • Warning (Yellow): Potential issue that may require attention (disk space low, service delay)
  • Information (Blue): Successful operation or status update (service started, update installed)

Viewing Event Details:

  1. Click on any event in the list
  2. Review the description and details
  3. Note the Event ID for researching solutions
  4. Click "Event Viewer" link to see full log
Filtering Events â–¼

You can filter which events are displayed:

  1. Click the "Events" dropdown in the section header
  2. Select time period (Last 24 hours, Last 7 days, etc.)
  3. Choose event levels to display (Errors, Warnings, Information)

Best Practice: Focus on Errors and Warnings initially. Information events are useful for detailed troubleshooting but can be overwhelming for daily monitoring.

3.4 Services Section

The Services section shows the status of Windows services running on the server.

Windows Service
A long-running background process that performs specific system functions. Services run without user interaction and can start automatically when the server boots. Examples include Windows Update, Windows Time, and DNS Server.

Service Status Indicators:

  • Running: Service is currently active and functioning
  • Stopped: Service is not running
  • Paused: Service is temporarily suspended (rare)

Service Startup Types:

  • Automatic: Starts when server boots (most server services)
  • Automatic (Delayed Start): Starts shortly after boot to reduce startup time
  • Manual: Starts only when explicitly requested
  • Disabled: Cannot be started

Service Management Caution

Be careful when stopping or disabling services:

  • Some services are critical for system operation
  • Stopping certain services can affect other dependent services
  • Always research a service before changing its status
  • Document any changes you make

3.5 Best Practices Analyzer (BPA)

The Best Practices Analyzer scans your server configuration and compares it against Microsoft's recommended best practices.

What BPA Checks:

  • Security configuration settings
  • Role-specific configurations
  • Performance optimization settings
  • Compliance with Microsoft guidelines

Running BPA:

  1. In Local Server view, locate the "Best Practices Analyzer" section
  2. Click "Tasks" â†’ "Start BPA Scan"
  3. Wait for scan to complete (usually 1-2 minutes)
  4. Review results categorized by severity

BPA Result Categories:

  • Error: Configuration that violates best practices and should be corrected
  • Warning: Configuration that may cause issues in certain scenarios
  • Information: Recommendations for optimization

BPA Recommendations

Run BPA:

  • After initial server installation and configuration
  • After installing new roles or features
  • Monthly as part of routine maintenance
  • Before deploying server to production

Not all BPA findings need immediate action - use judgment based on your environment and requirements.

3.6 Performance Section

The Performance section provides real-time monitoring of key server resources:

  • CPU Usage: Percentage of processor capacity in use
  • Memory Usage: Amount of RAM in use vs. total available

These are basic indicators. For detailed performance monitoring, use:

  • Task Manager: Real-time detailed resource usage
  • Resource Monitor: In-depth resource analysis
  • Performance Monitor (perfmon): Historical data and custom counters

4.2 Essential Administrative Tools

Here are the key tools you'll use frequently in server administration:

Computer Management â–¼

Purpose: Consolidated management console for system tools

Contains three main sections:

  • System Tools:
    • Task Scheduler - Schedule automated tasks
    • Event Viewer - View system logs
    • Shared Folders - Manage network shares
    • Local Users and Groups - Manage accounts (Week 6)
    • Performance - Performance monitoring
    • Device Manager - Hardware management
  • Storage:
    • Disk Management - Partition and format disks
  • Services and Applications:
    • Services - Manage Windows services
    • WMI Control - Windows Management Instrumentation

Common Uses:

  • Managing disk partitions
  • Viewing event logs
  • Managing services
  • Creating local user accounts
Event Viewer â–¼

Purpose: View and analyze system event logs

Key Log Categories:

  • Windows Logs:
    • Application - Application events
    • Security - Security auditing events
    • Setup - Installation events
    • System - System component events
  • Applications and Services Logs:
    • Role-specific logs
    • Feature-specific logs

Event Viewer Skills:

  • Filtering logs by date, level, or source
  • Creating custom views
  • Attaching tasks to events
  • Exporting logs for analysis

Troubleshooting Tip: When investigating issues, check Event Viewer first. Error messages often provide Event IDs that you can research online for solutions.

Services â–¼

Purpose: Manage Windows services

Service Management Tasks:

  • Start, stop, pause, or restart services
  • Change startup type (Automatic, Manual, Disabled)
  • Configure service recovery options
  • View service dependencies
  • Change service account credentials

Important Services to Know:

  • Windows Update (wuauserv): Manages Windows updates
  • Windows Time (W32Time): Time synchronization
  • DNS Client (Dnscache): DNS name resolution
  • Windows Firewall (mpssvc): Firewall service
  • Remote Desktop Services (TermService): RDP functionality

PowerShell Alternative:

# List all services
Get-Service

# Get specific service
Get-Service -Name wuauserv

# Start a service
Start-Service -Name wuauserv

# Stop a service
Stop-Service -Name wuauserv

# Restart a service
Restart-Service -Name wuauserv

# Set startup type
Set-Service -Name wuauserv -StartupType Automatic
Windows Firewall with Advanced Security â–¼

Purpose: Configure firewall rules and security policies

Key Features:

  • Inbound and outbound rules
  • Connection security rules (IPsec)
  • Monitoring active connections
  • Profile-based configuration (Domain, Private, Public)

Common Tasks:

  • Creating rules to allow specific applications
  • Opening ports for server services
  • Blocking unauthorized traffic
  • Monitoring firewall activity

Example: Allow Remote Desktop

  1. Open Windows Firewall with Advanced Security
  2. Click "Inbound Rules"
  3. Find "Remote Desktop" rules
  4. Enable the rules (green checkmark)

Security Note: Be cautious when creating firewall rules. Overly permissive rules can expose your server to attacks. Only open ports that are necessary for your services.

Task Scheduler â–¼

Purpose: Schedule automated tasks and scripts

Use Cases:

  • Running backups at scheduled times
  • Executing maintenance scripts
  • Automating report generation
  • Triggering actions based on events

Task Triggers:

  • Time-based (daily, weekly, monthly)
  • Event-based (when specific event occurs)
  • At startup or logon
  • When idle

Creating a Basic Task:

  1. Open Task Scheduler
  2. Click "Create Basic Task" in the Actions pane
  3. Name and describe the task
  4. Choose trigger (when to run)
  5. Choose action (what to run)
  6. Review and finish

Example: Schedule a PowerShell script to run daily at 2 AM for system cleanup tasks.

Performance Monitor (perfmon) â–¼

Purpose: Monitor and log system performance metrics

Key Features:

  • Real-time performance monitoring
  • Data collector sets for logging
  • Hundreds of performance counters
  • Custom reports and alerts

Important Counters:

  • Processor: % Processor Time
  • Memory: Available MBytes, Pages/sec
  • PhysicalDisk: % Disk Time, Avg. Disk Queue Length
  • Network Interface: Bytes Total/sec

Quick Start:

  1. Open Performance Monitor
  2. Click "Performance Monitor" in left pane
  3. Click green "+" to add counters
  4. Select counters from the list
  5. Click "Add" then "OK"
  6. Watch real-time graph

We'll explore Performance Monitor in detail in Week 16 (System Monitoring & Troubleshooting).

Resource Monitor â–¼

Purpose: Real-time detailed resource usage analysis

Tabs and Information:

  • Overview: Summary of CPU, disk, network, and memory
  • CPU: Per-process CPU usage and threads
  • Memory: Physical memory usage by process
  • Disk: Disk I/O by process and file
  • Network: Network activity by process and connection

When to Use:

  • Identifying which process is using high CPU
  • Finding which application is causing disk I/O
  • Tracking network connections and bandwidth usage
  • Investigating memory leaks

Advantage over Task Manager: Resource Monitor provides more detailed, real-time information about resource usage at the process and file level.

Local Security Policy â–¼

Purpose: Configure local security settings and policies

Key Policy Categories:

  • Account Policies:
    • Password Policy (complexity, length, age)
    • Account Lockout Policy (failed login attempts)
  • Local Policies:
    • Audit Policy (what to log)
    • User Rights Assignment (who can do what)
    • Security Options (various security settings)
  • Windows Defender Firewall: Firewall policies
  • Advanced Audit Policy: Detailed auditing configuration

Example: Configure Password Policy

  1. Open Local Security Policy
  2. Expand "Account Policies" â†’ "Password Policy"
  3. Double-click "Minimum password length"
  4. Set desired length (e.g., 12 characters)
  5. Click OK

Domain vs. Local: In domain environments (Week 11), Group Policy Objects (GPOs) from Active Directory override local security policies. Local Security Policy is primarily for standalone or workgroup servers.

One of Server Manager's primary functions is installing and managing server roles and features. This section introduces these concepts; we'll explore them in depth in Module 4.4.

5.1 Roles vs. Features: Quick Overview

Server Role
A primary function or service that a server provides to clients or other servers on a network. Examples: File Server, Web Server (IIS), DNS Server, DHCP Server, Active Directory Domain Services.
Feature
Supporting software that adds functionality to roles or provides standalone capabilities. Features enhance or support roles but aren't primary server functions. Examples: .NET Framework, PowerShell, Windows Backup, Telnet Client.

Key Differences:

Roles vs. Features Comparison
Aspect Roles Features
Purpose Primary server function Supporting functionality
Examples File Server, DNS, DHCP, Web Server .NET Framework, PowerShell, Backup
Typical Quantity Usually one primary role per server Multiple features per server
Management Tools Includes role-specific management tools May or may not include management tools
Dependencies May require specific features Usually independent

5.2 Accessing Role and Feature Management

Server Manager provides the Add Roles and Features Wizard for installation:

To Launch the Wizard:

  • Method 1: Server Manager â†’ Manage menu â†’ Add Roles and Features
  • Method 2: Dashboard â†’ Quick Start tile â†’ Add roles and features

5.3 Viewing Installed Roles and Features

To see what's currently installed:

GUI Method:

  1. Server Manager â†’ Manage â†’ Add Roles and Features
  2. Click through to "Server Roles" or "Features" pages
  3. Checked items are installed
  4. Can also view in Dashboard (each role gets a tile)

PowerShell Method:

# View all installed roles and features
Get-WindowsFeature | Where-Object {$_.Installed -eq $True}

# View only installed roles
Get-WindowsFeature | Where-Object {$_.Installed -eq $True -and $_.FeatureType -eq "Role"}

# View only installed features
Get-WindowsFeature | Where-Object {$_.Installed -eq $True -and $_.FeatureType -eq "Feature"}
