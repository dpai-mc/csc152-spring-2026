§1 — IPv4 Address Structure

Every device on an IP network needs a unique address — just like every house on a street needs a unique number. An IPv4 address is a 32-bit number that identifies a device (called a host) on a network.

Dotted-Decimal Notation

Humans don't work well with 32-bit binary strings, so IPv4 addresses are written as four octets (groups of 8 bits) separated by dots. Each octet can represent a decimal value from 0 to 255.

Binary Representation

Underneath the dotted-decimal format, each octet is an 8-bit binary number. Understanding binary is essential for subnetting. Each bit position represents a power of 2:

Bit Position Values in One Octet
Bit Position 7 6 5 4 3 2 1 0
Value 128 64 32 16 8 4 2 1

To convert a decimal octet to binary, determine which powers of 2 sum to that value.

Start with 192 and subtract the largest power of 2 that fits:

  1. 192 ≥ 128? Yes → bit 7 = 1, remainder = 192 − 128 = 64
  2. 64 ≥ 64? Yes → bit 6 = 1, remainder = 64 − 64 = 0
  3. All remaining bits = 0

Result: 11000000

Verification: 128 + 64 = 192 ✓

  1. 168 ≥ 128? Yes → bit 7 = 1, remainder = 40
  2. 40 ≥ 64? No → bit 6 = 0
  3. 40 ≥ 32? Yes → bit 5 = 1, remainder = 8
  4. 8 ≥ 16? No → bit 4 = 0
  5. 8 ≥ 8? Yes → bit 3 = 1, remainder = 0
  6. Remaining bits = 0

Result: 10101000

Verification: 128 + 32 + 8 = 168 ✓

The full binary representation of 192.168.1.10 is:

192     . 168     . 1       . 10
11000000. 10101000. 00000001. 00001010

§2 — Subnet Masks & Network Boundaries

An IP address alone doesn't tell you which part identifies the network and which part identifies the host. That's the job of the subnet mask.

How Subnet Masks Work

A subnet mask is also a 32-bit number. It uses a contiguous string of 1-bits (the network portion) followed by 0-bits (the host portion). The mask is applied to an IP address using a bitwise AND operation to determine the network ID.

Common Subnet Masks

Frequently Used Subnet Masks
CIDR Dotted-Decimal Mask Network Bits Host Bits Usable Hosts
/8255.0.0.082416,777,214
/16255.255.0.0161665,534
/24255.255.255.0248254
/25255.255.255.128257126
/26255.255.255.19226662
/27255.255.255.22427530
/28255.255.255.24028414
/30255.255.255.2523022
💡 Key Formula

Usable Hosts = 2h − 2, where h is the number of host bits. The two subtracted addresses are the network ID (all host bits = 0) and the broadcast address (all host bits = 1).

Bitwise AND: Finding the Network ID

To find the network ID, perform a bitwise AND between the IP address and the subnet mask. Any bit position where the mask has a 1 keeps the IP address bit; where the mask has a 0, the result is 0.

IP Address:    11000000.10101000.00000001.00001010  (192.168.1.10)
Subnet Mask:   11111111.11111111.11111111.00000000  (255.255.255.0)
               ──────────────────────────────────
Network ID:    11000000.10101000.00000001.00000000  (192.168.1.0)

The last octet is zeroed out because the mask has 0s there. The network ID is 192.168.1.0/24.

§3 — CIDR Notation

CIDR (Classless Inter-Domain Routing, pronounced "cider") notation is the modern way to express an IP address together with its subnet mask. Instead of writing the full dotted-decimal mask, you append a slash and the number of network bits to the IP address.

Why CIDR Replaced Classful Addressing

Early IP addressing used rigid "classes" (A, B, C) that wasted huge numbers of addresses. CIDR allows subnets of any size by using variable-length prefix lengths. You should be aware of the legacy classes for exam purposes, but CIDR is the standard in modern networking.

Class First Octet Range Default Mask CIDR Equivalent Networks Hosts per Network
A1–126255.0.0.0/8126~16.7 million
B128–191255.255.0.0/1616,38465,534
C192–223255.255.255.0/24~2.1 million254

Problem: A company needing 500 addresses had to get a Class B (/16) with 65,534 addresses — wasting over 65,000 addresses. CIDR solves this by allowing a /23 (510 usable hosts) instead.

Reading CIDR Notation

When you see 10.0.50.0/27, you know:

  • Network address: 10.0.50.0
  • Prefix length: 27 bits for the network portion
  • Host bits: 32 − 27 = 5
  • Usable hosts: 25 − 2 = 30
  • Subnet mask: 255.255.255.224

§4 — Subnetting Calculations

Subnetting is the process of dividing a larger network into smaller, more manageable pieces. This is a critical skill for Project 2, where you will design three separate subnets for Management, Servers, and Clients.

The Subnetting Process

Given a network and a prefix length, you can calculate three key values:

  1. Network ID — the first address in the subnet (all host bits = 0)
  2. Broadcast Address — the last address in the subnet (all host bits = 1)
  3. Usable Host Range — Network ID + 1 through Broadcast − 1

Step-by-Step Method

Step 1: Determine Host Bits

Subtract the prefix length from 32:

Host bits = 32 − prefix length

Example: For /27 → 32 − 27 = 5 host bits

This tells you: usable hosts = 25 − 2 = 30

Step 2: Calculate Block Size

The block size (also called the "increment") is 2host bits. This is how many total addresses are in each subnet.

Block size = 2h

Example: For /27 → 25 = 32

Subnets start at multiples of the block size in the relevant octet.

Step 3: Find Subnet Boundaries

Starting from 0 in the "interesting" octet (the octet where subnetting occurs), count by the block size:

For 192.168.1.0/27 (block size 32, interesting octet is the 4th):

Subnet # Network ID First Usable Last Usable Broadcast
1192.168.1.0192.168.1.1192.168.1.30192.168.1.31
2192.168.1.32192.168.1.33192.168.1.62192.168.1.63
3192.168.1.64192.168.1.65192.168.1.94192.168.1.95
4192.168.1.96192.168.1.97192.168.1.126192.168.1.127
…continues in increments of 32…

Step 4: Document the Summary

For each subnet, record:

  • Network ID / CIDR: e.g., 192.168.1.0/27
  • Subnet mask: 255.255.255.224
  • Usable range: 192.168.1.1 – 192.168.1.30
  • Broadcast: 192.168.1.31
  • Usable hosts: 30
  • Purpose: e.g., "Management subnet"

This documentation format is exactly what Project 2 Part A requires.

§5 — Private vs. Public Address Ranges

RFC 1918 defines three address blocks reserved for private use — they are not routable on the public internet. Every organization uses these ranges internally, and a router or firewall performs Network Address Translation (NAT) to map private addresses to a public address for internet access.

RFC 1918 Private Address Ranges
CIDR Block Address Range Number of Addresses Common Name
10.0.0.0/8 10.0.0.0 – 10.255.255.255 16,777,216 Class A private
172.16.0.0/12 172.16.0.0 – 172.31.255.255 1,048,576 Class B private
192.168.0.0/16 192.168.0.0 – 192.168.255.255 65,536 Class C private

Special Address Ranges

Beyond RFC 1918, you should recognize these reserved ranges:

Other Notable Reserved Ranges
Range Purpose
127.0.0.0/8Loopback (localhost) — used for testing the local TCP/IP stack
169.254.0.0/16Link-local / APIPA — auto-assigned when DHCP fails
0.0.0.0/0Default route — "all networks" (used in routing tables)
255.255.255.255Limited broadcast — sent to all hosts on the local segment
⚠ Troubleshooting Tip

If you see a 169.254.x.x address on a server, it means the DHCP client did not receive a response. This is one of the first things to check when troubleshooting connectivity — a topic covered in detail in Module 8.4.

§6 — IPv6 Addressing Fundamentals

IPv4's 32-bit address space provides about 4.3 billion addresses — not enough for the modern internet. IPv6 uses 128-bit addresses, providing approximately 3.4 × 1038 unique addresses. While most enterprise LANs still run IPv4 (and your Project 2 lab will use IPv4), understanding IPv6 basics is essential for any system administrator.

IPv6 Address Format

An IPv6 address is written as eight groups of four hexadecimal digits, separated by colons:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Abbreviation Rules

IPv6 addresses are long, so two abbreviation rules make them more manageable:

You can remove leading zeros in each 4-digit group (but at least one digit must remain):

2001:0db8:85a3:0000:0000:8a2e:0370:7334
                    ↓
2001: db8:85a3:   0:   0:8a2e: 370:7334

One (and only one) consecutive sequence of all-zero groups can be replaced with ::

2001:db8:85a3:0:0:8a2e:370:7334
                ↓
2001:db8:85a3::8a2e:370:7334

Important: You can only use :: once per address. Using it twice would make the address ambiguous.

Link-Local Addresses

Every IPv6-enabled interface automatically generates a link-local address in the fe80::/10 range. These addresses are used for communication on the local network segment only (not routable). You will see them when running ip addr on Linux or ipconfig on Windows.

# Example link-local address on a Linux interface
fe80::215:5dff:fe01:a3b2%eth0

In this course and Project 2, you will primarily work with IPv4. However, Quiz 8 will include IPv6 questions, and understanding IPv6 basics prepares you for modern network environments and certification exams (CompTIA Server+, Network+).

§7 — Physical Segments vs. Logical Subnets

A key concept in network administration is the distinction between physical and logical network elements — and how they relate to each other.

Physical Network Elements

  • NICs (Network Interface Cards) — the hardware that connects a device to the network
  • Cables (Ethernet, fiber) — the physical medium carrying signals
  • Switches — devices that connect multiple hosts on the same physical segment
  • Routers / Firewalls — devices that connect different network segments

Logical Network Elements

  • IP addresses — logical identifiers assigned to interfaces
  • Subnets — logical groupings of IP addresses
  • VLANs (Virtual LANs) — logical segmentation on a single physical switch
  • Routing tables — logical rules for forwarding traffic between subnets

How They Map Together

🔗 Week 6–7 Connection

In Weeks 6–7, you organized users into groups (departments) and assigned permissions based on those groups. Network subnets work similarly: you organize devices into subnets (segments) and assign access rules (firewall rules) based on those segments. The organizational thinking is the same — the scope changes from users and files to devices and traffic.

Why Subnetting Matters for Server Administrators

  • Security: Placing servers on a separate subnet from clients allows firewall rules to control which traffic reaches the servers.
  • Performance: Smaller subnets reduce broadcast traffic, improving network efficiency.
  • Management: Logical grouping makes it easier to apply policies, monitor traffic, and troubleshoot issues.
  • Scalability: A well-designed IP scheme accommodates future growth without re-addressing.

Module Summary & Next Steps

Key Takeaways

  1. An IPv4 address is a 32-bit number written as four dotted-decimal octets (0–255 each).
  2. The subnet mask determines which bits are the network portion and which are the host portion.
  3. CIDR notation (/24, /27, etc.) is the modern way to express the prefix length.
  4. Usable hosts = 2h − 2 (subtracting network ID and broadcast address).
  5. RFC 1918 defines private address ranges (10.x, 172.16–31.x, 192.168.x) used in internal networks.
  6. IPv6 uses 128-bit addresses in hexadecimal colon notation with two abbreviation rules.
  7. Physical NICs and cables provide connectivity; IP addresses and subnets provide logical organization.
