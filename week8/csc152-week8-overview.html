Overview & Learning Objectives

Week 8 marks a major transition in the course. After five weeks focused on OS installation, command-line skills, user management, and file permissions (CSLOs 1 & 2), you now shift to network configuration — the domain of CSLO 3: Configure systems for network connectivity, including physical and logical elements.

This week builds the networking foundation you need for the rest of the semester. You will learn how IP addressing and subnetting work, configure network interfaces on both Linux and Windows Server, understand the difference between static and dynamic addressing, and practice using essential network diagnostic tools. Everything this week feeds directly into Project 2 (multi-segment network infrastructure) and prepares you for Week 9 (DHCP and DNS server configuration).

🎯 CSLO 3 — Primary Alignment

Configure systems for network connectivity, including physical and logical elements.

All four modules this week, along with Lab 8 and Quiz 8, directly assess CSLO 3. This is the first week in the course dedicated entirely to networking.

Learning Modules

Week 8 consists of four self-study modules. Complete them in order — each builds on the previous one. The first three modules are pre-class preparation; Module 8.4 reinforces what you practice in the Zoom lab session.

Module 8.1: IP Addressing & Subnetting Foundations

⏱ ~1.5 hours 📖 Self-Study (Reading + Practice)

Covers IPv4 address structure (dotted-decimal and binary), subnet masks and the bitwise AND operation, CIDR notation, subnetting calculations (network ID, broadcast, usable range), private vs. public address ranges (RFC 1918), IPv6 addressing fundamentals, and the relationship between physical segments and logical subnets. Includes interactive binary conversion and subnetting exercises with immediate feedback.

Module 8.2: Network Interface Configuration

⏱ ~2 hours 📖 Self-Study + 🖥 Zoom Lab Core

Puts Module 8.1 theory into practice. Covers configuring static IP addresses on Linux (using Netplan YAML files and nmcli) and Windows Server (using the GUI and PowerShell New-NetIPAddress). Explains persistent vs. runtime configuration changes, and walks through verifying settings with ip addr and ipconfig. This module forms the core of the in-class Zoom lab session.

Module 8.3: DNS Client Configuration & Hostname Resolution

⏱ ~1 hour 📖 Self-Study (Reading + Practice)

Explains how DNS client resolution works, including /etc/resolv.conf on Linux and DNS server settings in Windows network adapter properties. Covers hostname configuration, the role of /etc/hosts and C:\Windows\System32\drivers\etc\hosts for local name resolution, and how DNS client settings integrate with the static IP configuration from Module 8.2. Prepares you for DNS server setup in Week 9.

Module 8.4: Network Testing & Verification Tools

⏱ ~1.5 hours 📖 Self-Study (Reading + Practice)

Covers the essential network diagnostic toolkit: ping (connectivity testing), traceroute/tracert (path tracing), nslookup/dig (DNS queries), ip addr/ipconfig/ifconfig (interface status), and ss/netstat (listening ports). Introduces a structured troubleshooting methodology (bottom-up, from physical layer to application) and common failure patterns such as APIPA addresses (169.254.x.x) indicating DHCP failure.

Quiz 8: Network Configuration Fundamentals Required

📝 Online Quiz ⏱ ~30–45 minutes 📊 25 points 📅 Due: Sunday, March 15, 2026 at 11:59 PM

Quiz 8 is required and covers all four modules: IPv4/IPv6 addressing and subnetting calculations (Module 8.1), network interface configuration concepts for Linux and Windows (Module 8.2), DNS client configuration (Module 8.3), and network diagnostic tools and troubleshooting methodology (Module 8.4). Looking Ahead: Week 9 Preview

Week 9: Network Services & Infrastructure builds directly on this week's client-side networking skills by moving to the server side. You will:

  • Configure a DHCP server — automate the IP address assignment process you performed manually this week
  • Set up basic DNS zones — build the name resolution infrastructure your clients rely on
  • Learn common server ports and protocols — understand what services listen on which ports
  • Configure basic firewall rules — control traffic between the network segments you designed this week using iptables and Windows Firewall
  • Introduction to centralized log collection — begin configuring pfSense syslog forwarding to a shared Splunk instance (Phase 2 of the Splunk integration plan)
➡️ Week 9 CSLO Alignment

Week 9 aligns with CSLO 2 (Apply physical and network data security techniques) and CSLO 3 (Configure systems for network connectivity). The firewall and Splunk components introduce the security monitoring thread that continues through the rest of the semester.

A solid understanding of IP addressing, interface configuration, and diagnostic tools from this week is essential for Week 9. If any Week 8 concepts feel unclear, revisit the relevant module and practice problems before moving on.
