3.1 Understanding Command Structure

Basic Command Syntax:

command [options] [arguments]

Components:

  • Command: The program to execute (ls, cd, cat)
  • Options: Modify command behavior (usually start with - or --)
    • Short options: -l, -a, -h
    • Long options: --help, --version
    • Combined: -lah (same as -l -a -h)
  • Arguments: What the command acts upon (files, directories, etc.)
# Command only
pwd

# Command with argument
cd /home

# Command with option and argument
ls -l /etc

# Command with multiple options and argument
ls -lah /var/log

# Command with long option
ls --all --human-readable /home

3.2 Essential Navigation Commands

pwd

Purpose: Print Working Directory - shows your current location

pwd
# Show current directory
pwd
# Output: /home/alice

# After changing directories
cd /var/log
pwd
# Output: /var/log

Use Cases:

  • Confirm your location before running commands
  • Verify you're in the correct directory
  • Debugging scripts that aren't working as expected

ls

Purpose: List directory contents

ls [options] [directory]

Common Options:

-l Long format -a Show hidden files -h Human-readable sizes -t Sort by time -r Reverse order -R Recursive
# Basic listing
ls

# Long format with human-readable sizes
ls -lh

# Show all files including hidden
ls -la

# Sort by modification time (newest first)
ls -lt

# Combine multiple options
ls -lah

cd

Purpose: Change Directory - navigate to different directories

cd [directory]

Special Shortcuts:

  • cd or cd ~ - Go to home directory
  • cd .. - Go up one directory level
  • cd - - Go to previous directory
  • cd / - Go to root directory
  • cd ../.. - Go up two levels
# Go to specific directory
cd /var/log

# Go home
cd ~

# Go up one level
cd ..

# Go to previous directory
cd -
Understanding ls -l Output:

3.3 File and Directory Operations

mkdir - Make Directory

Purpose: Create new directories

mkdir [options] directory_name

Common Options:

-p Create parent directories as needed -m Set permissions -v Verbose (show what's being created)
# Create single directory
mkdir projects

# Create multiple directories
mkdir docs photos videos

# Create nested directories (parents created automatically)
mkdir -p projects/website/html
mkdir -p projects/website/css
mkdir -p projects/website/js

# Create with specific permissions
mkdir -m 755 public_html

# Verbose mode
mkdir -pv documents/2024/reports
# Output:
# mkdir: created directory 'documents'
# mkdir: created directory 'documents/2024'
# mkdir: created directory 'documents/2024/reports'

Real-World Example:

# Set up project structure
mkdir -p ~/myproject/{src,tests,docs,config}
# Creates:
# ~/myproject/
# ‚îú‚îÄ‚îÄ src/
# ‚îú‚îÄ‚îÄ tests/
# ‚îú‚îÄ‚îÄ docs/
# ‚îî‚îÄ‚îÄ config/

touch - Create Empty File

Purpose: Create empty files or update modification time

touch filename
# Create single file
touch readme.txt

# Create multiple files
touch index.html style.css script.js

# Create file with path
touch ~/documents/notes.txt

# Update modification time of existing file
touch existing_file.txt

Why Use Touch:

  • Quickly create placeholder files
  • Update file timestamps
  • Test file creation permissions
  • Create files for scripts to process

cp - Copy Files and Directories

Purpose: Copy files and directories

cp [options] source destination

Common Options:

-r Recursive (copy directories) -i Interactive (prompt before overwrite) -v Verbose -p Preserve attributes
# Copy file
cp file1.txt file2.txt

# Copy file to directory
cp report.pdf ~/documents/

# Copy multiple files to directory
cp file1.txt file2.txt file3.txt ~/backup/

# Copy directory (requires -r)
cp -r projects/ projects_backup/

# Interactive copy (prompts before overwriting)
cp -i important.txt important_backup.txt

# Verbose recursive copy
cp -rv website/ website_backup/

# Copy with wildcard
cp *.txt ~/documents/
cp directory1 directory2

Forgot -r flag for directory copying

cp -r directory1 directory2

Correctly copies directory and all contents

mv - Move/Rename Files

Purpose: Move files to new location or rename them

mv [options] source destination

Common Options:

-i Interactive (prompt before overwrite) -v Verbose -n No clobber (don't overwrite)
# Rename file
mv oldname.txt newname.txt

# Move file to directory
mv report.pdf ~/documents/

# Move multiple files
mv file1.txt file2.txt file3.txt ~/archive/

# Move and rename
mv ~/downloads/document.pdf ~/documents/report_2024.pdf

# Move directory
mv old_project/ ~/archive/old_project/

# Interactive move
mv -i important.txt ~/backup/
Key Difference from cp: The mv command doesn't require -r for directories because it's moving/renaming, not copying. It works the same for files and directories.

rm - Remove Files

Purpose: Delete files and directories

rm [options] file

Common Options:

-r Recursive (delete directories) -i Interactive (prompt for each file) -f Force (no prompts) -v Verbose
‚ö†Ô∏è WARNING: rm is permanent! There's no recycle bin. Deleted files are gone forever. Always double-check before deleting, especially with wildcards or the -r flag.
# Remove single file
rm unwanted.txt

# Remove multiple files
rm file1.txt file2.txt file3.txt

# Remove with wildcard
rm *.tmp

# Interactive removal (safer)
rm -i important.txt

# Remove directory (requires -r)
rm -r old_project/

# Force removal (dangerous!)
rm -rf directory/

# Verbose removal
rm -v *.log
‚ö†Ô∏è DANGEROUS COMMANDS - NEVER RUN:
sudo rm -rf / # Deletes everything
sudo rm -rf /* # Deletes everything
rm -rf ~/* # Deletes your entire home
Safety Tips:
  • Use -i for important files
  • List files before deleting: ls *.txt then rm *.txt
  • Double-check wildcards
  • Consider using trash-put instead of rm

3.4 Viewing File Contents

Command Purpose When to Use Example
cat Display entire file Short files that fit on one screen cat readme.txt
less Page through file Large files, need to scroll/search less /var/log/syslog
head Show first lines Check file format, quick preview head -n 20 file.txt
tail Show last lines Check recent log entries tail -f /var/log/syslog
file Determine file type Identify unknown files file mystery.bin
wc Count lines/words Count entries in files wc -l /etc/passwd

less Navigation Keys:

Space or f - Forward one page
b - Backward one page
/pattern - Search forward
n - Next search result
g - Go to beginning
G - Go to end
q - Quit
# Display entire file
cat readme.txt

# Page through large file
less /var/log/syslog

# First 10 lines (default)
head logfile.txt

# First 20 lines
head -n 20 logfile.txt

# Last 10 lines (default)
tail logfile.txt

# Follow log file (watch in real-time)
tail -f /var/log/syslog

# Count lines in file
wc -l /etc/passwd

# Determine file type
file document.pdf

3.5 Command-Line Efficiency Tips

Tab Completion

How It Works:

  • Press Tab once: Complete if unique
  • Press Tab twice: Show all possibilities
# Type partial command and press Tab
cat do[Tab]
# Completes to: cat document.txt (if unique)

# See all options
ls /et[Tab][Tab]
# Shows: etc/ ethtool

# Complete directory names
cd /var/lo[Tab]
# Completes to: cd /var/log/

Command History

Navigation:

  • Up Arrow - Previous command
  • Down Arrow - Next command
  • Ctrl+R - Search history
  • !! - Repeat last command
  • !n - Repeat command number n
  • !string - Repeat last command starting with string
# View history
history

# Repeat last command
!!

# Repeat with sudo
sudo !!

# Repeat specific command
!100 # Runs command #100 from history

# Repeat last command starting with 'cd'
!cd

# Clear history
history -c

üìù Module Summary ‚ñº

Key Takeaways

  • Command structure: command [options] [arguments]
  • pwd shows current directory, cd changes directory, ls lists contents
  • mkdir creates directories, -p flag creates parent directories
  • touch creates empty files or updates timestamps
  • cp copies files/directories (use -r for directories)
  • mv moves or renames files and directories
  • rm deletes files (PERMANENT - no undo!)
  • cat displays entire file, less pages through large files
  • head shows first lines, tail shows last lines (tail -f follows)
  • Tab completion and command history improve efficiency
  • Options can be combined: -lah is same as -l -a -h
  • Always double-check before using rm, especially with -r or -f
