4.1 Understanding Package Management

What is a Package?

Definition: A package is a compressed archive containing:

  • Compiled software (binaries)
  • Configuration files
  • Documentation
  • Metadata (version, dependencies, description)
  • Installation/removal scripts
Think of a package like a meal kit delivery. It contains all the ingredients (software files), instructions (installation scripts), and information about what else you need (dependencies), all bundled together for easy installation.

Package Formats:

  • DEB: Used by Debian, Ubuntu (.deb files)
  • RPM: Used by Red Hat, Rocky Linux, Fedora (.rpm files)

What is a Package Manager?

Definition: A package manager is a tool that automates the process of installing, updating, configuring, and removing software packages.

Key Functions:

Download and install software automatically

üîÑ Updates

Understanding Repositories

Definition: A repository (repo) is a centralized storage location containing software packages.

Repository Structure:

How Repositories Work:
Package manager maintains list of available packages
When you request software, it checks the repository
Downloads package and dependencies
Verifies package integrity (checksums, signatures)

Understanding Dependencies

Definition: A dependency is software that another program requires to function.

You want to install: Web Server (nginx) nginx depends on: ‚îú‚îÄ‚îÄ libssl (encryption library) ‚îú‚îÄ‚îÄ libpcre (regular expressions) ‚îî‚îÄ‚îÄ zlib (compression library) ‚îî‚îÄ‚îÄ libc (standard C library)

Dependency Resolution:

Package managers automatically:

  • Identify all required dependencies
  • Check which are already installed
  • Download and install missing dependencies
  • Handle dependency conflicts
# You run: sudo apt install nginx # Package manager does: Reading package lists... Done Building dependency tree... Done The following additional packages will be installed: nginx-common nginx-core libssl1.1 libpcre3 The following NEW packages will be installed: nginx nginx-common nginx-core libssl1.1 libpcre3 # Automatically installs nginx + 4 dependencies

4.2 APT Package Management (Ubuntu/Debian)

APT Overview: Advanced Package Tool (APT) is the package manager for Debian-based distributions like Ubuntu. It uses .deb package format and provides simple, powerful commands for software management.

Essential APT Commands:

Command Purpose When to Use
sudo apt update Update package lists from repositories Before installing software or upgrading
sudo apt upgrade Upgrade all installed packages Regular maintenance, security updates
sudo apt install pkg Install a package Adding new software
sudo apt remove pkg Remove package (keep config) Uninstalling software
sudo apt purge pkg Remove package and config files Complete removal
apt search keyword Search for packages Finding software
apt show pkg Display package details Learning about software before installing
sudo apt autoremove Remove unused dependencies Cleaning up after removals

APT Command Details:

apt update - Update Package Lists

Purpose: Downloads the latest package information from repositories

# Update package lists sudo apt update # Output: Hit:1 http://archive.ubuntu.com/ubuntu jammy InRelease Get:2 http://archive.ubuntu.com/ubuntu jammy-updates InRelease [119 kB] Get:3 http://security.ubuntu.com/ubuntu jammy-security InRelease [110 kB] Fetched 336 kB in 2s (168 kB/s) Reading package lists... Done Building dependency tree... Done All packages are up to date.
Important: apt update does NOT install or upgrade anything. It only updates the list of available packages. Think of it like refreshing your web browser before shopping online.

apt upgrade - Upgrade Installed Packages

Purpose: Upgrades all installed packages to their latest versions

# Upgrade all packages sudo apt upgrade # Output: Reading package lists... Done Building dependency tree... Done Calculating upgrade... Done The following packages will be upgraded: openssh-server openssh-client libssl1.1 3 upgraded, 0 newly installed, 0 to remove Need to get 1,234 kB of archives. After this operation, 45.6 kB additional disk space will be used. Do you want to continue? [Y/n] Y

Safe Upgrade Process:

# 1. Update package lists sudo apt update # 2. See what will be upgraded apt list --upgradable # 3. Perform upgrade sudo apt upgrade # 4. Reboot if kernel was updated sudo reboot

apt install - Install Packages

Purpose: Install new software packages

# Install single package sudo apt install nginx # Install multiple packages sudo apt install nginx mysql-server php # Install without prompting sudo apt install -y htop # Reinstall package sudo apt install --reinstall nginx

apt remove vs apt purge

Key Difference:

sudo apt remove nginx

Removes package but keeps configuration files in /etc/nginx/

Use when: You might reinstall later

sudo apt purge nginx

Removes package AND all configuration files

Use when: Complete removal needed

4.3 DNF/YUM Package Management (Rocky Linux/RHEL)

DNF Overview: Dandified YUM (DNF) is the modern replacement for YUM on Red Hat family distributions. It uses .rpm package format and provides similar functionality to APT with some key differences.

APT vs DNF Command Comparison:

Task APT (Ubuntu) DNF (Rocky Linux)
Update package lists sudo apt update (Automatic with dnf commands)
Upgrade packages sudo apt upgrade sudo dnf update
Install package sudo apt install pkg sudo dnf install pkg
Remove package sudo apt remove pkg sudo dnf remove pkg
Search packages apt search keyword dnf search keyword
Package info apt show pkg dnf info pkg
List installed apt list --installed dnf list installed
Clean cache sudo apt clean sudo dnf clean all
Remove unused sudo apt autoremove sudo dnf autoremove
Key Difference: DNF doesn't have a separate "update package lists" command like APT's apt update. DNF automatically refreshes metadata when you run other commands. The command dnf update actually upgrades packages (equivalent to apt upgrade).

4.4 Common Package Management Workflows

Workflow: Installing New Software

Ubuntu (APT)

Update package lists sudo apt update
Search for package apt search nginx
View package details apt show nginx
Install package sudo apt install nginx
Verify installation which nginx nginx -v

Rocky Linux (DNF)

(Metadata updates automatically) # No separate update needed
Search for package dnf search nginx
View package details dnf info nginx
Install package sudo dnf install nginx
Verify installation which nginx nginx -v

Workflow: System Updates

Ubuntu (APT)

Update package lists sudo apt update
See what will update apt list --upgradable
Install updates sudo apt upgrade
Remove unused packages sudo apt autoremove
Reboot if kernel updated sudo reboot

Rocky Linux (DNF)

Check for updates dnf check-update
(Updates shown automatically) # Built into check-update
Install updates sudo dnf update
Remove unused packages sudo dnf autoremove
Reboot if kernel updated sudo reboot
‚ö†Ô∏è Before Major Updates:
  • Take a VM snapshot (if using virtualization)
  • Backup important data
  • Review what will be updated
  • Plan for potential downtime
  • Have a rollback plan

Workflow: System Cleanup

Ubuntu (APT)

Remove package + configs sudo apt purge old-package
Remove unused dependencies sudo apt autoremove
Clear package cache sudo apt clean
Fix broken dependencies sudo apt --fix-broken install

Rocky Linux (DNF)

Remove package sudo dnf remove old-package
Remove unused dependencies sudo dnf autoremove
Clear all caches sudo dnf clean all
Check for problems sudo dnf check

4.5 Package Management Best Practices

Security Best Practices:

üîÑ Regular Updates
# Ubuntu sudo apt update && sudo apt upgrade -y # Rocky Linux sudo dnf update -y

Update weekly or enable automatic security updates

‚úì Verify Sources
# Check package repository apt policy nginx # Verify signatures apt-key list

Only use trusted repositories

üìã Document Changes
# List installed packages dpkg -l > installed.txt # Or for RPM rpm -qa > installed.txt

Keep records of what's installed

System Maintenance Schedule:

üìÖ Weekly Maintenance
# Ubuntu sudo apt update sudo apt upgrade sudo apt autoremove sudo apt autoclean # Rocky Linux sudo dnf update sudo dnf autoremove sudo dnf clean all
Pro Tip: Create a simple maintenance script to automate regular updates. This ensures you don't forget critical security patches. However, always review what will be updated before applying changes to production servers.

üí° Remember:
  • APT: Separate update and upgrade commands
  • DNF: Single update command does both
  • Both use install, remove, search with same syntax
  • Always use sudo for installation/removal operations
  • Search and info commands don't require sudo

üìù Module Summary ‚ñº

Key Takeaways

  • Package managers automate software installation, updates, and removal
  • Packages contain binaries, configs, docs, and dependency information
  • Repositories are centralized storage locations for packages
  • Dependencies are automatically resolved by package managers
  • APT (Ubuntu) uses .deb packages, DNF (Rocky) uses .rpm packages
  • apt update refreshes package lists, apt upgrade installs updates
  • dnf update does both refresh and upgrade in one command
  • apt remove keeps configs, apt purge deletes everything
  • Regular updates are essential for security
  • Always verify package sources and use official repositories
  • Document installed packages for troubleshooting and replication
  • Autoremove cleans up unused dependencies after removal
