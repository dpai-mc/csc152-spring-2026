👋 Welcome to Week 13

This week marks a pivotal shift in the course. Over the first twelve weeks you installed, configured, and secured individual Linux and Windows servers. Now we zoom out to the abstraction layer that makes modern data centers possible: virtualization. You will learn how hypervisors decouple operating systems from hardware, how virtual machines are created and managed, and how virtual networking and storage work — concepts you have been using in our Proxmox lab all semester but will now understand at a deeper level.

We also extend virtualization into the cloud, exploring IaaS, PaaS, and SaaS service models. On the practical side, you will deploy Splunk Universal Forwarders across your lab servers — building directly on the Splunk Enterprise instance you installed in Week 12 — and Project 3: Splunk Security Monitoring is officially assigned this week.

🎯 Week 13 Learning Goals

By the end of this week you will be able to:

  1. Explain virtualization concepts and the benefits of server virtualization in enterprise environments.
  2. Compare and contrast Type 1 (bare-metal) and Type 2 (hosted) hypervisors and identify appropriate use cases for each.
  3. Create and configure virtual machines, including CPU, memory, storage, and network adapter settings.
  4. Manage virtual networks and virtual storage within a hypervisor environment.
  5. Describe cloud computing models (IaaS, PaaS, SaaS) and explain how they relate to virtualization.
  6. Install and configure Splunk Universal Forwarders on Linux and Windows servers to send logs to a centralized Splunk Enterprise instance.
  7. Verify data ingestion in Splunk and create at least one simple dashboard panel.

💡 Key Concepts to Watch For

  • Type 1 vs. Type 2 hypervisors — bare-metal (ESXi, Hyper-V, KVM) vs. hosted (VirtualBox, VMware Workstation) and when to use each.
  • Virtual network modes — NAT, bridged, internal, and host-only and how they affect VM connectivity.
  • VM lifecycle management — snapshots, clones (full vs. linked), and OVA/OVF export/import.
  • Virtual storage — virtual disk formats (VDI, VMDK, VHD/VHDX), dynamic vs. fixed allocation, and virtual storage controllers.
  • Cloud service models — IaaS, PaaS, SaaS and the NIST essential characteristics of cloud computing.
  • Shared responsibility model — who secures what in each cloud service model.
  • Splunk forwarder architecture — Universal Forwarder → outputs.conf → Splunk Enterprise receiver; inputs.conf for data sources.
  • Containers vs. VMs — awareness-level comparison of OS-level vs. hardware-level virtualization.
