Network Services & Infrastructure

You configured network clients last week. This week you become the administrator who runs the services those clients depend on — deploying DHCP and DNS servers, standing up a pfSense 2.7.2 virtual firewall in Proxmox, and using firewall rules and network segmentation to enforce security boundaries. Everything you build this week feeds directly into Project 2.

Week 9 of 17 CSLO 2 · CSLO 3 Required: 26.5 pts  |  Optional EC: 30 pts Project 2 assigned this week (due Week 11)

🎯 Week 9 Learning Goals

By the end of Week 9 you will be able to:

1 Configure a DHCP server to provide dynamic IP addressing to client systems on Linux and Windows Server.
2 Explain the DNS server hierarchy and configure basic forward and reverse lookup zones.
3 Identify common server protocols and their associated port numbers (SSH, DNS, DHCP, HTTP/S, SMB, RDP, and more).
4 Configure basic firewall rules to control network traffic using iptables/nftables, Windows Firewall, or pfSense.
5 Describe how centralized log collection supports network security monitoring.
6 Forward pfSense syslog events to Splunk and run basic SPL searches on firewall and network events.

Week 9 addresses two Course Student Learning Outcomes: CSLO 2 — Apply physical and network data security techniques (firewall rules, segmentation, log monitoring) and CSLO 3 — Configure systems for network connectivity, including physical and logical elements (DHCP, DNS, pfSense, subnets).

💡 Tips for Success This Week

  1. Plan your network first. Before touching Proxmox or pfSense, sketch your topology on paper and document the IP addressing scheme. A clear plan prevents hours of troubleshooting later.
  2. Take VM snapshots. Snapshot your pfSense VM after a successful installation and again after each major configuration step. Snapshots make it easy to roll back if something breaks.
  3. Test incrementally. Verify DHCP is working before moving to DNS. Verify DNS before configuring firewall rules. Chasing multiple simultaneous failures is much harder than isolating one at a time.
  4. Understand firewall rule order. pfSense processes rules top to bottom — the first matching rule wins. Place more specific rules above general ones, and remember that an implicit deny-all sits at the bottom of every ruleset.
  5. Post Discussion 5 by Wednesday. The initial post is due Wednesday March 18 at 11:59 PM. Don't let it slip to the weekend — peer responses are also required by Sunday.
  6. Use the pfSense documentation. The Netgate documentation is comprehensive and searchable. When you're stuck, check the docs before spending an hour guessing.

Why These Concepts Matter Together

DHCP and DNS are the invisible infrastructure that makes every network usable — clients need addresses (DHCP) and need to translate names to addresses (DNS) before they can do anything else. Firewall rules and network segmentation determine who can reach those services and what they can do once they get there. pfSense ties all of these together in a single manageable appliance. And Splunk gives you visibility into what is actually happening across all of it — which is how you move from just configuring a network to administering one.

🔗 How This Week Connects

Week 8 gave you the client-side foundation: you configured static IP addresses and DHCP client mode on Linux and Windows Server, set up DNS client resolution, and practiced structured network troubleshooting with tools like ping, nslookup, and traceroute. Week 9 flips the perspective: instead of consuming network services, you are now providing them. The DHCP server you deploy this week is the service your Week 8 clients were requesting addresses from. The DNS server you configure is the resolver your clients were pointed at. The firewall rules you write in pfSense are what makes all of that communication secure and controlled. This week is also the on-ramp for Project 2 — every module maps directly to a deliverable in the project, and the lab activities mirror the project tasks closely enough that completing Lab 9 (EC) is the fastest way to get a head start. The Splunk introduction in Module 9.5 begins a skill progression that will grow through Weeks 12–13 and become a central component of Project 3.
