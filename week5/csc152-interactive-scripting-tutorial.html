üì¶ Lesson 1: Variables in Scripts

What are Variables?

Variables are containers that store data values. They allow scripts to be dynamic and reusable by storing information that can change.

Bash Variables

In Bash, variables are assigned without spaces around the equals sign:

#!/bin/bash
# Variable assignment (no spaces!)
SERVER_NAME="webserver01"
PORT=8080
LOG_PATH="/var/log/myapp"

# Using variables with $ prefix
echo "Server: $SERVER_NAME"
echo "Port: $PORT"

PowerShell Variables

In PowerShell, variables always start with $ and can include spaces around the equals sign:

# PowerShell variable assignment
$ServerName = "webserver01"
$Port = 8080
$LogPath = "C:\Logs\myapp"

# Using variables
Write-Host "Server: $ServerName"
Write-Host "Port: $Port"
üí° Pro Tip: Use descriptive variable names! $srv is harder to understand than $ServerName. Clear variable names make your scripts easier to maintain.

üéØ Knowledge Check

In Bash, which is the CORRECT way to assign a variable?

Correct! Bash variables must be assigned without spaces around the equals sign. The $ is only used when referencing (reading) the variable, not when assigning.

üîÄ Lesson 2: Conditional Statements (If/Else)

Making Decisions in Scripts

Conditional statements allow scripts to make decisions and execute different code based on whether conditions are true or false.

Bash If/Else Syntax

#!/bin/bash
DISK_USAGE=85

if [ $DISK_USAGE -gt 80 ]; then
    echo "WARNING: Disk usage is high!"
elif [ $DISK_USAGE -gt 60 ]; then
    echo "Disk usage is moderate."
else
    echo "Disk usage is normal."
fi

PowerShell If/Else Syntax

# PowerShell conditional
$DiskUsage = 85

if ($DiskUsage -gt 80) {
    Write-Host "WARNING: Disk usage is high!"
}
elseif ($DiskUsage -gt 60) {
    Write-Host "Disk usage is moderate."
}
else {
    Write-Host "Disk usage is normal."
}

üéØ Knowledge Check

What keyword ends an if statement in Bash?

Correct! Bash uses 'fi' (if spelled backwards) to end if statements. PowerShell uses curly braces { }.

üîÑ Lesson 3: Loops (For and While)

Repeating Actions

Loops allow you to execute a block of code multiple times. This is essential for processing multiple files, users, or servers.

Bash For Loop

#!/bin/bash
# Loop through a list of servers
for SERVER in web01 web02 web03 db01; do
    echo "Checking $SERVER..."
    ping -c 1 $SERVER
done

# Loop through files
for FILE in /var/log/*.log; do
    echo "Processing: $FILE"
done

PowerShell ForEach Loop

# Loop through servers
$Servers = @("web01", "web02", "web03", "db01")

foreach ($Server in $Servers) {
    Write-Host "Checking $Server..."
    Test-Connection -ComputerName $Server -Count 1
}

# Loop through files using pipeline
Get-ChildItem -Path "C:\Logs\*.log" | ForEach-Object {
    Write-Host "Processing: $($_.Name)"
}

While Loops

# Bash while loop
COUNT=1
while [ $COUNT -le 5 ]; do
    echo "Attempt $COUNT"
    ((COUNT++))
done
# PowerShell while loop
$Count = 1
while ($Count -le 5) {
    Write-Host "Attempt $Count"
    $Count++
}
üí° Use Case: Loops are perfect for checking multiple servers, processing log files, creating multiple user accounts, or performing batch operations.

üéØ Knowledge Check

What keyword ends a for loop in Bash?

Correct! Bash for loops end with 'done'. PowerShell uses closing curly braces { }.

‚å®Ô∏è Lesson 4: Script Parameters and Input

Making Scripts Flexible

Parameters allow users to pass information to scripts when they run them, making scripts reusable for different situations.

Bash Script Parameters

#!/bin/bash
# Script: backup.sh
# Usage: ./backup.sh /source/path /dest/path

# $0 = script name, $1 = first arg, $2 = second arg
SOURCE=$1
DESTINATION=$2

# Check if parameters provided
if [ -z "$SOURCE" ] || [ -z "$DESTINATION" ]; then
    echo "Usage: $0 <source> <destination>"
    exit 1
fi

echo "Backing up $SOURCE to $DESTINATION"
cp -r $SOURCE $DESTINATION

PowerShell Parameters

# Script: Backup.ps1
# Usage: .\Backup.ps1 -Source "C:\Data" -Destination "D:\Backup"

param(
    [Parameter(Mandatory=$true)]
    [string]$Source,
    
    [Parameter(Mandatory=$true)]
    [string]$Destination
)

Write-Host "Backing up $Source to $Destination"
Copy-Item -Path $Source -Destination $Destination -Recurse

üéØ Knowledge Check

In a Bash script, what does $1 represent?

Correct! $1 represents the first argument passed to the script. $0 is the script name, and $# is the total number of arguments.

üõ†Ô∏è Lesson 5: Practical Script Examples

Real-World Administrative Scripts

Let's combine what we've learned into practical scripts that system administrators use daily.

Bash: System Health Check Script

#!/bin/bash
# health_check.sh - System health monitoring script

echo "===== System Health Check ====="
echo "Date: $(date)"
echo

# Check disk usage
echo "--- Disk Usage ---"
DISK_USAGE=$(df -h / | awk 'NR==2 {print $5}' | tr -d '%')

if [ $DISK_USAGE -gt 80 ]; then
    echo "WARNING: Disk usage is ${DISK_USAGE}%"
else
    echo "OK: Disk usage is ${DISK_USAGE}%"
fi

# Check memory
echo
echo "--- Memory Usage ---"
free -h | head -2

# Check critical services
echo
echo "--- Critical Services ---"
for SERVICE in sshd nginx mysql; do
    if systemctl is-active --quiet $SERVICE 2>/dev/null; then
        echo "$SERVICE: RUNNING"
    else
        echo "$SERVICE: NOT RUNNING"
    fi
done

PowerShell: Service Monitor Script

# ServiceMonitor.ps1 - Monitor critical Windows services

param(
    [string[]]$Services = @("W32Time", "Spooler", "WinRM")
)

Write-Host "===== Service Status Report ====="
Write-Host "Date: $(Get-Date)"
Write-Host

foreach ($ServiceName in $Services) {
    $Svc = Get-Service -Name $ServiceName -ErrorAction SilentlyContinue
    
    if ($Svc) {
        if ($Svc.Status -eq 'Running') {
            Write-Host "[OK] $ServiceName is running" -ForegroundColor Green
        }
        else {
            Write-Host "[WARN] $ServiceName is $($Svc.Status)" -ForegroundColor Yellow
        }
    }
    else {
        Write-Host "[ERROR] $ServiceName not found" -ForegroundColor Red
    }
}
üí° Best Practice: Always include comments, error handling, and usage instructions in your scripts. Future you (and your colleagues) will thank you!
‚ö†Ô∏è Remember: Test scripts in a safe environment first. Use virtual machines for testing before running scripts on production servers.

üéØ Knowledge Check

What is the purpose of the shebang (#!/bin/bash) at the start of a Bash script?

Correct! The shebang (#!) tells the system which interpreter to use to execute the script. #!/bin/bash means 'run this script with the Bash shell.'
