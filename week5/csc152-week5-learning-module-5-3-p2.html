Working with Objects

The most fundamental difference between PowerShell and traditional command-line interfaces is that PowerShell works with objects rather than text. Understanding objects is the key to PowerShell mastery.

Understanding PowerShell Objects

üìñ What is an Object?

An object is a structured data entity that contains:

  • Properties: Characteristics or attributes (like variables)
  • Methods: Actions the object can perform (like functions)
  • Type: The kind of object (Process, Service, File, etc.)

Analogy: Think of an object like a car:

  • Properties: Color, Make, Model, Year, Speed
  • Methods: Start(), Stop(), Accelerate(), Brake()
  • Type: Automobile

üêß Bash (Text-Based)

# List processes (outputs text)
ps aux

# Output is text strings:
USER  PID %CPU %MEM COMMAND
root  1   0.0  0.1  /sbin/init

# Must parse text to extract data
ps aux | awk '{print $2, $11}'

Problem: Need to parse text, know column positions, handle formatting variations

‚ö° PowerShell (Object-Based)

# Get processes (outputs objects)
Get-Process

# Output is objects with properties:
# Each process has Id, Name, CPU, etc.

# Access properties directly
Get-Process | Select-Object Id, ProcessName

Advantage: Direct property access, no parsing needed, strongly typed data

Exploring Object Properties

Using Get-Member to Explore Objects

Get-Member reveals everything about an object: its type, properties, and methods.

# Get information about process objects
Get-Process | Get-Member

# Output shows:
# TypeName: System.Diagnostics.Process
# 
# Name              MemberType     Definition
# ----              ----------     ----------
# Handles           Property       int Handles {get;}
# Id                Property       int Id {get;}
# ProcessName       Property       string ProcessName {get;}
# CPU               Property       double CPU {get;}
# WorkingSet        Property       long WorkingSet {get;}
# Kill              Method         void Kill()
# Start             Method         bool Start()
# ...and many more

# Get only properties
Get-Process | Get-Member -MemberType Property

# Get only methods
Get-Process | Get-Member -MemberType Method

Accessing Object Properties

Use dot notation to access properties:

# Get a single process
$process = Get-Process -Name "notepad"

# Access properties using dot notation
$process.Id              # Process ID
$process.ProcessName     # Process name
$process.CPU             # CPU time
$process.WorkingSet      # Memory usage (bytes)
$process.StartTime       # When process started

# Display specific properties
Write-Host "Process ID: $($process.Id)"
Write-Host "Process Name: $($process.ProcessName)"
Write-Host "Memory: $([math]::Round($process.WorkingSet / 1MB, 2)) MB"

# Access properties in pipeline
Get-Process | ForEach-Object {
    Write-Host "$($_.ProcessName) is using $($_.WorkingSet) bytes"
}

Selecting Properties

üìñ Select-Object Cmdlet

Select-Object chooses which properties to display or work with. It's one of the most frequently used cmdlets in PowerShell.

Select-Object Examples

# Select specific properties
Get-Process | Select-Object ProcessName, Id, CPU

# Select and rename properties
Get-Process | Select-Object @{Name="Name";Expression={$_.ProcessName}}, 
                            @{Name="Memory(MB)";Expression={$_.WorkingSet / 1MB}}

# Select first 10 objects
Get-Process | Select-Object -First 10

# Select last 5 objects
Get-Process | Select-Object -Last 5

# Select unique values
Get-Process | Select-Object ProcessName -Unique

# Select and skip
Get-Process | Select-Object -Skip 5 -First 10

# Select all properties (useful for exploring)
Get-Process | Select-Object -Property *

# Create custom objects
Get-Process | Select-Object ProcessName, 
    @{Name="MemoryMB";Expression={[math]::Round($_.WorkingSet/1MB,2)}},
    @{Name="CPUSeconds";Expression={[math]::Round($_.CPU,2)}}

Object Methods

üìñ What are Methods?

Methods are actions that objects can perform. They're like functions that belong to the object.

Using Object Methods

# Get a process object
$process = Get-Process -Name "notepad"

# View available methods
$process | Get-Member -MemberType Method

# Call methods using dot notation
$process.Kill()          # Terminate the process
$process.Refresh()       # Update process information
$process.WaitForExit()   # Wait for process to exit

# String methods
$text = "Hello, PowerShell!"
$text.ToUpper()          # HELLO, POWERSHELL!
$text.ToLower()          # hello, powershell!
$text.Length             # 18 (property, not method)
$text.Substring(0, 5)    # Hello
$text.Replace("Hello", "Hi")  # Hi, PowerShell!
$text.Contains("Power")  # True
$text.Split(",")         # Split into array

# DateTime methods
$date = Get-Date
$date.AddDays(7)         # Add 7 days
$date.AddMonths(-1)      # Subtract 1 month
$date.ToString("yyyy-MM-dd")  # Format date

# Array methods
$services = Get-Service
$services.Count          # Number of services
$services.GetType()      # Object type information

Object Types

Understanding Object Types

# Get object type
$process = Get-Process -Name "notepad"
$process.GetType()

# Output:
# IsPublic IsSerial Name            BaseType
# -------- -------- ----            --------
# True     False    Process         System.ComponentModel.Component

# Different cmdlets return different types
(Get-Process).GetType().FullName
# System.Diagnostics.Process

(Get-Service).GetType().FullName
# System.ServiceProcess.ServiceController

(Get-ChildItem).GetType().FullName
# System.IO.FileInfo or System.IO.DirectoryInfo

# Check if object is of specific type
$process -is [System.Diagnostics.Process]  # True
$process -is [String]                      # False

# Type conversion
[int]"123"              # Convert string to integer
[string]123             # Convert integer to string
[datetime]"2026-02-17"  # Convert string to DateTime

Process Object Structure

TypeName: System.Diagnostics.Process

Properties:
  • Id (int) - Process identifier
  • ProcessName (string) - Name of process
  • CPU (double) - CPU time used
  • WorkingSet (long) - Memory in use
  • StartTime (DateTime) - When started
  • Handles (int) - Number of handles

Methods:
  • Kill() - Terminate process
  • Refresh() - Update information
  • Start() - Start process
  • WaitForExit() - Wait for completion

The PowerShell Pipeline

The PowerShell pipeline is similar to the Linux/Unix pipe, but instead of passing text, it passes objects. This makes it significantly more powerful and easier to work with.

Pipeline Basics

üìñ What is the Pipeline?

The pipeline (|) connects cmdlets together, passing the output of one cmdlet as input to the next. In PowerShell, entire objects flow through the pipeline, not just text.

# Basic pipeline syntax
Cmdlet1 | Cmdlet2 | Cmdlet3

# Example: Get processes, filter, and sort
Get-Process | Where-Object {$_.CPU -gt 10} | Sort-Object CPU -Descending

üêß Bash Pipeline (Text)

# Get processes using > 10% CPU
ps aux | \
  awk '$3 > 10' | \
  sort -k3 -rn

# Text flows through pipeline
# Must parse columns
# Column positions matter

‚ö° PowerShell Pipeline (Objects)

# Get processes using > 10 CPU seconds
Get-Process | 
  Where-Object {$_.CPU -gt 10} |
  Sort-Object CPU -Descending

# Objects flow through pipeline
# Access properties by name
# Type-safe operations

Filtering with Where-Object

üìñ Where-Object Cmdlet

Where-Object (alias: where or ?) filters objects based on property values. It's like an if statement in the pipeline.

Where-Object Examples

# Basic filtering
Get-Service | Where-Object {$_.Status -eq "Running"}

# Multiple conditions (AND)
Get-Process | Where-Object {$_.CPU -gt 10 -and $_.WorkingSet -gt 100MB}

# Multiple conditions (OR)
Get-Service | Where-Object {$_.Status -eq "Running" -or $_.Status -eq "Paused"}

# Using comparison operators
Get-Process | Where-Object {$_.ProcessName -like "chrome*"}
Get-ChildItem | Where-Object {$_.Length -gt 1MB}
Get-Service | Where-Object {$_.StartType -eq "Automatic"}

# Simplified syntax (PowerShell 3.0+)
Get-Service | Where Status -eq "Running"
Get-Process | Where CPU -gt 10
Get-ChildItem | Where Length -gt 1MB

# Using the $_ variable
# $_ represents the current object in the pipeline
Get-Process | Where-Object {
    $_.ProcessName -like "s*" -and 
    $_.WorkingSet -gt 50MB
}

# Negation (NOT)
Get-Service | Where-Object {$_.Status -ne "Running"}
Get-Process | Where-Object {-not ($_.ProcessName -like "system*")}

Sorting with Sort-Object

Sort-Object Examples

# Sort by single property
Get-Process | Sort-Object ProcessName
Get-Service | Sort-Object Status

# Sort descending
Get-Process | Sort-Object CPU -Descending
Get-ChildItem | Sort-Object Length -Descending

# Sort by multiple properties
Get-Process | Sort-Object CPU, WorkingSet -Descending

# Sort and select top items
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

# Sort with custom expression
Get-ChildItem | Sort-Object @{Expression={$_.Length}; Descending=$true}

# Stable sort (maintains order for equal values)
Get-Service | Sort-Object Status -Stable

Formatting Output

üìñ Format Cmdlets

Format cmdlets control how objects are displayed. Important: Format cmdlets should always be the last cmdlets in a pipeline because they convert objects to formatted text.

Cmdlet Purpose Best For
Format-Table Display as table Multiple objects, few properties
Format-List Display as list Single object, many properties
Format-Wide Display in columns Single property, many objects
Format-Custom Custom formatting Complex formatting needs

Format-Table Examples

# Default table format
Get-Process | Format-Table

# Select specific properties
Get-Process | Format-Table ProcessName, Id, CPU

# Auto-size columns
Get-Process | Format-Table -AutoSize

# Wrap text instead of truncating
Get-Process | Format-Table -Wrap

# Group by property
Get-Service | Format-Table -GroupBy Status

# Custom column widths
Get-Process | Format-Table ProcessName, 
    @{Label="Memory(MB)";Expression={$_.WorkingSet/1MB};FormatString="N2"}

# Aliases
Get-Process | ft ProcessName, CPU  # ft = Format-Table

Format-List Examples

# Display all properties
Get-Service wuauserv | Format-List *

# Display specific properties
Get-Process notepad | Format-List ProcessName, Id, StartTime, CPU

# Multiple objects as lists
Get-Service | Select-Object -First 3 | Format-List Name, Status, StartType

# Aliases
Get-Service wuauserv | fl *  # fl = Format-List

Format-Wide Examples

# Display process names in columns
Get-Process | Format-Wide ProcessName

# Specify number of columns
Get-Process | Format-Wide ProcessName -Column 4

# Auto-size columns
Get-Process | Format-Wide ProcessName -AutoSize

# Aliases
Get-Process | fw ProcessName  # fw = Format-Wide

Measuring and Grouping

Measure-Object Examples

# Count objects

# Calculate statistics

# Multiple properties
Get-ChildItem | Measure-Object -Property Length -Sum -Average

Group-Object Examples

# Group services by status

# Group processes by name

# Group files by extension

Get-Process | Group-Object ProcessName | 
    Select-Object Name, Count | 
    Sort-Object Count -Descending

Complex Pipeline Examples

Real-World Pipeline Scenarios

# Find top 10 memory-consuming processes
Get-Process | 
    Sort-Object WorkingSet -Descending | 
    Select-Object -First 10 ProcessName, 
        @{Name="Memory(MB)";Expression={[math]::Round($_.WorkingSet/1MB,2)}}

# Find services set to Automatic but not running
Get-Service | 
    Where-Object {$_.StartType -eq "Automatic" -and $_.Status -ne "Running"} |
    Select-Object Name, Status, StartType

# Find largest files in directory
Get-ChildItem -Recurse -File | 
    Sort-Object Length -Descending | 
    Select-Object -First 20 FullName, 
        @{Name="Size(MB)";Expression={[math]::Round($_.Length/1MB,2)}}

# Count file types in directory
Get-ChildItem -Recurse -File | 
    Group-Object Extension | 
    Select-Object Name, Count | 
    Sort-Object Count -Descending

# System health check pipeline
$report = @{
    TotalProcesses = (Get-Process | Measure-Object).Count
    RunningServices = (Get-Service | Where Status -eq "Running" | Measure-Object).Count
    StoppedServices = (Get-Service | Where Status -eq "Stopped" | Measure-Object).Count
    MemoryUsedGB = [math]::Round((Get-Process | Measure-Object WorkingSet -Sum).Sum / 1GB, 2)
    TopCPUProcess = (Get-Process | Sort-Object CPU -Descending | Select-Object -First 1).ProcessName
}
$report

‚ö†Ô∏è Pipeline Best Practices

  • Filter early: Use Where-Object early in pipeline to reduce data
  • Format last: Format cmdlets should always be last
  • Select wisely: Use Select-Object to choose only needed properties
  • Use aliases carefully: Full cmdlet names in scripts, aliases for interactive use
  • Break long pipelines: Use line continuation (backtick `) for readability

Discovery and Help System

PowerShell's built-in discovery and help system is one of its greatest strengths. You can learn PowerShell using PowerShell itself!

Finding Cmdlets with Get-Command

üìñ Get-Command

Get-Command helps you discover available cmdlets, functions, and aliases. It's your first stop when you need to find a command.

Get-Command Examples

PowerShell Providers

PowerShell providers allow you to access different data stores using the same cmdlets you use for the file system. This unified approach makes PowerShell incredibly powerful.

Understanding Providers

üìñ What is a Provider?

A provider is a PowerShell adapter that makes a data store look like a file system drive. You can navigate and manipulate different types of data stores using familiar cmdlets like Get-ChildItem, Set-Location, and New-Item.

Viewing Available Providers

# List all providers
Get-PSProvider

# Output shows:
# Name         Capabilities                      Drives
# ----         ------------                      ------
# Registry     ShouldProcess                     {HKLM, HKCU}
# Alias        ShouldProcess                     {Alias}
# Environment  ShouldProcess                     {Env}
# FileSystem   Filter, ShouldProcess, Credentials {C, D, ...}
# Function     ShouldProcess                     {Function}
# Variable     ShouldProcess                     {Variable}
# Certificate  ShouldProcess                     {Cert}

# Get drives for all providers
Get-PSDrive

FileSystem Provider

Working with the File System

# Navigate file system
Set-Location C:\Windows
Get-ChildItem

# These are equivalent
Get-ChildItem C:\Windows
Get-ChildItem -Path C:\Windows
Get-Item C:\Windows

# Create items
New-Item -Path "C:\Temp\TestFolder" -ItemType Directory
New-Item -Path "C:\Temp\test.txt" -ItemType File

# Copy, move, remove
Copy-Item -Path "C:\Temp\test.txt" -Destination "C:\Temp\backup.txt"
Move-Item -Path "C:\Temp\test.txt" -Destination "C:\Temp\NewFolder\"
Remove-Item -Path "C:\Temp\backup.txt"

Registry Provider

Accessing the Windows Registry

# Navigate to registry
Set-Location HKLM:\Software

# List registry keys (like directories)
Get-ChildItem HKLM:\Software\Microsoft

# Get registry values (like files)
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion

# Create registry key
New-Item -Path "HKCU:\Software\MyApp"

# Set registry value
Set-ItemProperty -Path "HKCU:\Software\MyApp" -Name "Setting1" -Value "Value1"

# Get specific registry value
Get-ItemProperty -Path "HKCU:\Software\MyApp" -Name "Setting1"

# Remove registry key
Remove-Item -Path "HKCU:\Software\MyApp" -Recurse

# Common registry locations
Set-Location HKLM:\  # Local Machine
Set-Location HKCU:\  # Current User

# Example: Check Windows version from registry
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion" | 
    Select-Object ProductName, CurrentBuild

Certificate Provider

Managing Certificates

# Navigate to certificate store
Set-Location Cert:\

# List certificate stores
Get-ChildItem Cert:\

# View certificates in store
Get-ChildItem Cert:\LocalMachine\My
Get-ChildItem Cert:\CurrentUser\My

# Find specific certificate
Get-ChildItem Cert:\LocalMachine\My | 
    Where-Object {$_.Subject -like "*example.com*"}

# Get certificate details
Get-ChildItem Cert:\LocalMachine\My | 
    Select-Object Subject, Issuer, NotAfter

# Find expiring certificates
Get-ChildItem Cert:\LocalMachine\My | 
    Where-Object {$_.NotAfter -lt (Get-Date).AddDays(30)}

Environment Provider

Working with Environment Variables

# Navigate to environment variables
Set-Location Env:\

# List all environment variables
Get-ChildItem Env:

# Get specific environment variable
$env:Path
$env:ComputerName
$env:UserName
$env:TEMP

# Set environment variable (current session)
$env:MyVariable = "MyValue"

# Create permanent environment variable
[Environment]::SetEnvironmentVariable("MyVar", "MyValue", "User")

# Get environment variable
Get-Item Env:Path
Get-ChildItem Env:TEMP

# Common environment variables
$env:COMPUTERNAME    # Computer name
$env:USERNAME        # Current user
$env:USERPROFILE     # User profile path
$env:TEMP            # Temp directory
$env:PROGRAMFILES    # Program Files path
$env:WINDIR          # Windows directory

Other Providers

Alias, Function, and Variable Providers

# Alias provider
Set-Location Alias:\

Exercise 1: System Information Report

Task: Create a pipeline that generates a system information report.

Requirements:

  • Computer name and OS version
  • Total and available memory
  • Number of running processes and services
  • Top 5 processes by CPU usage
  • Disk space information for all drives

Solution Approach:

# System Information Report
Write-Host "=== System Information Report ===" -ForegroundColor Cyan
Write-Host ""

# Computer Info
$computerInfo = Get-ComputerInfo
Write-Host "Computer Name: $($computerInfo.CsName)"
Write-Host "OS: $($computerInfo.WindowsProductName)"
Write-Host "OS Version: $($computerInfo.WindowsVersion)"
Write-Host ""

# Memory Info
$memory = Get-CimInstance Win32_OperatingSystem
$totalMemoryGB = [math]::Round($memory.TotalVisibleMemorySize / 1MB, 2)
$freeMemoryGB = [math]::Round($memory.FreePhysicalMemory / 1MB, 2)
Write-Host "Total Memory: $totalMemoryGB GB"
Write-Host "Free Memory: $freeMemoryGB GB"
Write-Host ""

# Process and Service Counts
$processCount = (Get-Process | Measure-Object).Count
$runningServices = (Get-Service | Where-Object Status -eq "Running" | Measure-Object).Count
Write-Host "Running Processes: $processCount"
Write-Host "Running Services: $runningServices"
Write-Host ""

# Top 5 CPU Processes
Write-Host "Top 5 Processes by CPU:"
Get-Process | Sort-Object CPU -Descending | Select-Object -First 5 ProcessName, CPU | Format-Table

# Disk Space
Write-Host "Disk Space Information:"
Get-PSDrive -PSProvider FileSystem | 
    Where-Object {$_.Used -ne $null} |
    Select-Object Name, 
        @{Name="Used(GB)";Expression={[math]::Round($_.Used/1GB,2)}},
        @{Name="Free(GB)";Expression={[math]::Round($_.Free/1GB,2)}} |
    Format-Table

Exercise 2: Service Management

Task: Analyze Windows services and identify potential issues.

Requirements:

  • Find services set to Automatic but not running
  • Count services by status
  • List services that can be stopped
  • Export results to CSV

Solution:

# Service Analysis

# Services set to Automatic but not running
Write-Host "Services set to Automatic but not Running:" -ForegroundColor Yellow
$autoNotRunning = Get-Service | 
    Where-Object {$_.StartType -eq "Automatic" -and $_.Status -ne "Running"}
$autoNotRunning | Format-Table Name, Status, StartType

# Count services by status
Write-Host "`nService Count by Status:" -ForegroundColor Cyan
Get-Service | Group-Object Status | 
    Select-Object Name, Count | 
    Format-Table

# Services that can be stopped
Write-Host "Services that can be stopped:" -ForegroundColor Green
Get-Service | 
    Where-Object {$_.CanStop -eq $true} | 
    Select-Object -First 10 Name, Status | 
    Format-Table

# Export to CSV
$autoNotRunning | Export-Csv -Path "AutoNotRunning.csv" -NoTypeInformation
Write-Host "`nResults exported to AutoNotRunning.csv"

Exercise 3: File System Analysis

Task: Analyze files in C:\Windows\System32

Requirements:

  • Count files by extension
  • Find largest 10 files
  • Calculate total size
  • Find files modified in last 7 days

Solution:

# File System Analysis

$path = "C:\Windows\System32"

# Count files by extension
Write-Host "File Count by Extension:" -ForegroundColor Cyan
Get-ChildItem -Path $path -File | 
    Group-Object Extension | 
    Sort-Object Count -Descending | 
    Select-Object -First 10 Name, Count | 
    Format-Table

# Largest 10 files
Write-Host "`nTop 10 Largest Files:" -ForegroundColor Yellow
Get-ChildItem -Path $path -File | 
    Sort-Object Length -Descending | 
    Select-Object -First 10 Name, 
        @{Name="Size(MB)";Expression={[math]::Round($_.Length/1MB,2)}} |
    Format-Table

# Total size
$totalSize = (Get-ChildItem -Path $path -File | 
    Measure-Object -Property Length -Sum).Sum
$totalSizeGB = [math]::Round($totalSize / 1GB, 2)
Write-Host "Total Size: $totalSizeGB GB"

# Files modified in last 7 days
Write-Host "`nFiles Modified in Last 7 Days:" -ForegroundColor Green
Get-ChildItem -Path $path -File | 
    Where-Object {$_.LastWriteTime -gt (Get-Date).AddDays(-7)} | 
    Select-Object Name, LastWriteTime | 
    Format-Table

üéØ Key Concepts Mastered

Working with Objects

  • PowerShell works with .NET objects, not text
  • Objects have properties (data) and methods (actions)
  • Use Get-Member to explore objects
  • Access properties with dot notation
  • Select-Object chooses which properties to work with

The PowerShell Pipeline

  • Pipeline passes objects between cmdlets
  • Where-Object filters objects by property values
  • Sort-Object orders objects
  • Format cmdlets control display (always use last)
  • Export cmdlets save data to various formats

Discovery and Help

  • Get-Command finds available cmdlets
  • Get-Help provides comprehensive documentation
  • About topics explain PowerShell concepts
  • Show-Command provides GUI for building commands
  • PowerShell is self-documenting

PowerShell Providers

  • Providers make data stores accessible like file systems
  • FileSystem, Registry, Certificate, Environment providers
  • Use same cmdlets for different data stores
  • Navigate providers like directories

PowerShell vs Bash Comparison

Similarities

  • Pipeline concept (| operator)
  • Variables ($variable)
  • Loops and conditionals
  • Functions
  • Scripting capabilities
  • Remote execution

Key Differences

  • Objects vs text in pipeline
  • Verb-Noun vs short commands
  • Built-in help system
  • Discoverable syntax
  • Case-insensitive
  • .NET integration
