Introduction to Advanced Bash Commands

Welcome to Module 5.1! This module builds upon your foundational Linux command-line skills from Week 3 by introducing advanced Bash commands, text processing utilities, and command chaining techniques that are essential for efficient server administration.

üìñ What is Advanced Command-Line Administration?

Advanced command-line administration refers to the use of sophisticated command combinations, text processing tools, and automation techniques to perform complex system administration tasks efficiently. Unlike basic commands that perform simple operations, advanced techniques leverage the power of command chaining, regular expressions, and stream processing to handle large-scale administrative tasks.

Module Learning Objectives

By the end of this module, you will be able to:

  • Execute advanced file and text manipulation commands
  • Use pipes and redirection to chain commands effectively
  • Apply text processing tools (grep, sed, awk) for system administration tasks
  • Utilize command history and shortcuts for efficiency
  • Search and filter system information using command-line tools

Why Advanced Bash Commands Matter

As a system administrator, you'll frequently need to:

  • Process log files: Search through thousands of lines to find specific errors or patterns
  • Manage files at scale: Find, modify, or delete files across complex directory structures
  • Automate repetitive tasks: Combine commands to create powerful one-liners
  • Monitor system resources: Track processes, memory, disk usage, and network connections
  • Troubleshoot issues: Quickly identify problems using command-line diagnostic tools

üí° Real-World Scenario

Imagine you're managing a web server that's experiencing performance issues. Using advanced Bash commands, you can:

  1. Search Apache logs for all 500 errors in the last hour using grep
  2. Count how many times each error occurred using sort and uniq
  3. Identify the top IP addresses generating errors using awk
  4. Find processes consuming excessive memory using ps and sort

All of this can be accomplished with a few well-crafted command pipelines!

  • Basic Linux navigation (cd, ls, pwd)
  • File operations (cp, mv, rm, mkdir)
  • Viewing file contents (cat, less, head, tail)

Advanced File Operations

While basic file commands like cp and mv handle simple file management, advanced file operations allow you to search, compare, compress, and manipulate files at scale.

Finding Files: The find Command

üìñ The find Command

The find command searches for files and directories in a directory hierarchy based on various criteria such as name, type, size, modification time, and permissions. It's one of the most powerful and versatile commands for file management.

Basic find Syntax

find [path] [options] [expression]

Common find Examples

Example 1: Find Files by Name

# Find all .log files in /var/log find /var/log -name "*.log" # Find files with exact name (case-sensitive) find /home -name "config.txt" # Find files with name (case-insensitive) find /etc -iname "*.CONF"

Explanation: The -name option searches for files matching the pattern. Use -iname for case-insensitive searches. Wildcards (* and ?) work as in shell globbing.

Example 2: Find Files by Type

# Find all directories find /home/user -type d # Find all regular files find /tmp -type f # Find all symbolic links find /usr/bin -type l

Explanation: The -type option filters by file type: f (regular file), d (directory), l (symbolic link), b (block device), c (character device).

Example 3: Find Files by Size

# Find files larger than 100MB find /var -type f -size +100M # Find files smaller than 1KB find /tmp -type f -size -1k # Find files exactly 50MB find /home -type f -size 50M

Explanation: The -size option uses: + (greater than), - (less than), no prefix (exactly). Units: k (kilobytes), M (megabytes), G (gigabytes).

Example 4: Find Files by Modification Time

# Find files modified in the last 7 days find /var/log -type f -mtime -7 # Find files modified more than 30 days ago find /tmp -type f -mtime +30 # Find files modified exactly 10 days ago find /home -type f -mtime 10

Explanation: -mtime measures in days. -atime (access time) and -ctime (change time) work similarly. For minutes, use -mmin, -amin, -cmin.

Example 5: Find and Execute Commands

# Find and delete all .tmp files find /tmp -name "*.tmp" -type f -delete # Find and display detailed info find /var/log -name "*.log" -exec ls -lh {} \; # Find and move files to archive find /data -name "*.old" -exec mv {} /archive/ \; # Find and change permissions find /var/www -type f -exec chmod 644 {} \;

Explanation: -exec runs a command on each found file. {} represents the current file. \; terminates the command. Use -delete for simple deletions.

‚ö†Ô∏è Warning: Be Careful with -delete

The -delete option permanently removes files without confirmation. Always test your find command without -delete first to verify it finds the correct files!

# SAFE: Test first find /tmp -name "*.tmp" -type f # DANGEROUS: Only run after verifying find /tmp -name "*.tmp" -type f -delete

Archive and Compression

üìñ Archives and Compression

Archives combine multiple files into a single file, while compression reduces file size. The tar command handles both archiving and compression, making it essential for backups and file distribution.

The tar Command

# Create a tar archive tar -cvf archive.tar /path/to/directory # Create a compressed tar.gz archive tar -czvf archive.tar.gz /path/to/directory # Create a compressed tar.bz2 archive (better compression) tar -cjvf archive.tar.bz2 /path/to/directory # Extract a tar archive tar -xvf archive.tar # Extract a tar.gz archive tar -xzvf archive.tar.gz # List contents without extracting tar -tvf archive.tar # Extract to specific directory tar -xzvf archive.tar.gz -C /destination/path

üí° Remembering tar Options

Common tar options:

  • c - Create an archive
  • x - eXtract an archive
  • v - Verbose (show files being processed)
  • f - File (specify archive filename)
  • z - gZip compression
  • j - bZip2 compression
  • t - lisT contents

Other Compression Tools

Command Extension Compression Speed
gzip .gz Good Fast
bzip2 .bz2 Better Slower

Compression Examples

Text Processing and Manipulation

Text processing is one of the most powerful capabilities of the Linux command line. As a system administrator, you'll spend significant time analyzing log files, configuration files, and command output. Mastering text processing tools is essential for efficient troubleshooting and automation.

Pattern Searching with grep

üìñ What is grep?

grep (Global Regular Expression Print) searches text for patterns and prints matching lines. It's one of the most frequently used commands in system administration for log analysis, configuration file searches, and filtering command output.

Basic grep Syntax

grep [options] pattern [file...]

Basic grep Examples

# Search for a pattern in a file grep "error" /var/log/syslog # Case-insensitive search grep -i "error" /var/log/syslog # Search recursively in directories grep -r "TODO" /home/user/projects/ # Show line numbers grep -n "error" /var/log/syslog # Count matching lines grep -c "error" /var/log/syslog # Show only filenames with matches grep -l "error" /var/log/*.log # Invert match (show non-matching lines) grep -v "debug" /var/log/application.log

Advanced grep with Regular Expressions

# Match lines starting with "Error" grep "^Error" /var/log/syslog # Match lines ending with "failed" grep "failed$" /var/log/syslog # Match any single character (.) grep "err.r" /var/log/syslog # Matches "error", "err0r", etc. # Match IP addresses (basic pattern) grep "[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}" access.log # Extended regex with -E (or egrep) grep -E "error|warning|critical" /var/log/syslog # Match word boundaries grep -w "error" /var/log/syslog # Won't match "errors" or "terrored" # Show context around matches grep -A 3 "error" /var/log/syslog # Show 3 lines After grep -B 2 "error" /var/log/syslog # Show 2 lines Before grep -C 2 "error" /var/log/syslog # Show 2 lines of Context

üí° Common grep Use Cases for Sysadmins

# Find all failed SSH login attempts grep "Failed password" /var/log/auth.log # Find all HTTP 500 errors in Apache logs grep " 500 " /var/log/apache2/access.log # Find processes using port 80 sudo netstat -tulpn | grep ":80" # Search for configuration errors grep -i "error" /etc/apache2/apache2.conf # Find all TODO comments in code grep -rn "TODO" /var/www/html/

Stream Editing with sed

üìñ What is sed?

sed (Stream Editor) performs text transformations on an input stream (file or pipeline). It's particularly useful for find-and-replace operations, deleting lines, and modifying text in automated scripts.

Basic sed Operations

Substitution (Find and Replace)

# Replace first occurrence on each line sed 's/old/new/' file.txt # Replace all occurrences (global) sed 's/old/new/g' file.txt # Replace and save to new file sed 's/old/new/g' file.txt > newfile.txt # Replace in-place (modify original file) sed -i 's/old/new/g' file.txt # Replace with backup sed -i.bak 's/old/new/g' file.txt # Case-insensitive replacement sed 's/old/new/gi' file.txt

Syntax: s/pattern/replacement/flags

  • s = substitute command
  • g = global (all occurrences)
  • i = case-insensitive

Deleting Lines

# Delete line 5 sed '5d' file.txt # Delete lines 5-10 sed '5,10d' file.txt # Delete last line sed '$d' file.txt # Delete lines containing pattern sed '/pattern/d' file.txt # Delete empty lines sed '/^$/d' file.txt # Delete lines starting with # sed '/^#/d' file.txt

Practical sed Examples

# Change IP address in configuration sed -i 's/192.168.1.100/192.168.1.200/g' /etc/network/interfaces # Remove comments and empty lines from config sed '/^#/d; /^$/d' /etc/ssh/sshd_config # Add text at beginning of each line sed 's/^/# /' file.txt # Add text at end of each line sed 's/$/ - processed/' file.txt # Print only lines 10-20 sed -n '10,20p' file.txt # Replace multiple spaces with single space sed 's/ */ /g' file.txt

Text Processing with awk

üìñ What is awk?

awk is a powerful text processing language designed for data extraction and reporting. It excels at processing structured text (columns and rows) and is particularly useful for log analysis and generating reports from command output.

Basic awk Concepts

awk processes text line by line and splits each line into fields (columns). By default, fields are separated by whitespace.

# Basic syntax awk 'pattern { action }' file # Print specific fields (columns) # $1 = first field, $2 = second field, $0 = entire line

Basic awk Examples

# Print first column awk '{print $1}' file.txt # Print first and third columns awk '{print $1, $3}' file.txt # Print last column awk '{print $NF}' file.txt # Print with custom separator awk '{print $1 " - " $2}' file.txt # Use different field separator (e.g., colon for /etc/passwd) awk -F: '{print $1}' /etc/passwd # Print lines longer than 80 characters awk 'length > 80' file.txt # Print line numbers with content awk '{print NR, $0}' file.txt

awk for System Administration

# Print usernames from /etc/passwd awk -F: '{print $1}' /etc/passwd # Show users with UID >= 1000 (regular users) awk -F: '$3 >= 1000 {print $1}' /etc/passwd # Calculate total disk usage df -h | awk 'NR>1 {sum+=$3} END {print "Total:", sum}' # Print processes using more than 10% CPU ps aux | awk '$3 > 10 {print $2, $3, $11}' # Extract IP addresses from log awk '{print $1}' /var/log/apache2/access.log | sort | uniq -c | sort -rn # Print 5th column of CSV file awk -F, '{print $5}' data.csv # Sum values in second column awk '{sum += $2} END {print "Total:", sum}' numbers.txt

Advanced awk: Pattern Matching

# Print lines containing "error" awk '/error/ {print}' /var/log/syslog # Print lines NOT containing "debug" awk '!/debug/ {print}' /var/log/application.log # Print lines where 3rd field equals "ESTABLISHED" awk '$3 == "ESTABLISHED" {print}' netstat_output.txt # Print if 4th field is greater than 1000 awk '$4 > 1000 {print $1, $4}' data.txt # Multiple conditions awk '$3 > 10 && $4 < 100 {print}' data.txt

Command Chaining and Redirection

One of the most powerful features of the Linux command line is the ability to chain commands together, creating complex operations from simple building blocks. This is the essence of the Unix philosophy: "Do one thing and do it well."

Understanding Standard Streams

üìñ The Three Standard Streams

Every Linux process has three standard streams:

  • stdin (0): Standard Input - where the command reads input (usually keyboard)
  • stdout (1): Standard Output - where the command writes normal output (usually screen)
  • stderr (2): Standard Error - where the command writes error messages (usually screen)

Output Redirection

Operator Description Example
> Redirect stdout to file (overwrite) ls > filelist.txt
>> Redirect stdout to file (append) echo "new line" >> log.txt
2> Redirect stderr to file command 2> errors.txt
&> Redirect both stdout and stderr command &> output.txt
2>&1 Redirect stderr to stdout command > out.txt 2>&1

Redirection Examples

# Save command output to file ls -l > directory_listing.txt # Append to existing file date >> log.txt # Discard output (send to /dev/null) command > /dev/null # Save errors to separate file find / -name "*.conf" 2> errors.txt # Save both output and errors command > output.txt 2>&1 # Separate output and errors command > output.txt 2> errors.txt # Discard errors, keep output command 2> /dev/null # Discard everything command &> /dev/null

Input Redirection

Operator Description Example
< Read input from file sort < unsorted.txt
<< Here document (multi-line input) cat << EOF
<<< Here string (single line input) grep "pattern" <<< "$variable"

Input Redirection Examples

# Sort contents of file sort < unsorted.txt > sorted.txt # Count lines in file wc -l < file.txt # Here document example cat << EOF > newfile.txt Line 1 Line 2 Line 3 EOF # Here string example grep "error" <<< "This is an error message"

Pipes: Connecting Commands

üìñ What is a Pipe?

A pipe (|) connects the stdout of one command to the stdin of another command. This allows you to chain commands together, with each command processing the output of the previous one.

Basic Pipe Examples

# List files and count them ls | wc -l # Search and sort results grep "error" /var/log/syslog | sort # Process listing sorted by memory usage ps aux | sort -k4 -rn | head -10 # Find largest files du -h /home | sort -rh | head -10 # Count unique IP addresses in log cat access.log | awk '{print $1}' | sort | uniq | wc -l

Complex Pipeline Examples

# Find top 10 IP addresses accessing your server cat /var/log/apache2/access.log | \ awk '{print $1}' | \ sort | \ uniq -c | \ sort -rn | \ head -10 # Find processes using most CPU ps aux | \ awk '{print $3, $11}' | \ sort -rn | \ head -10 # Analyze disk usage by directory du -h /var | \ sort -rh | \ head -20 # Find and list largest files find /home -type f -exec du -h {} \; | \ sort -rh | \ head -20 # Count error types in log grep "ERROR" /var/log/application.log | \ awk '{print $5}' | \ sort | \ uniq -c | \ sort -rn

Command Substitution

üìñ Command Substitution

Command substitution allows you to use the output of one command as an argument to another command. There are two syntaxes: $(command) (preferred) and `command` (backticks, legacy).

Command Substitution Examples

# Store command output in variable current_date=$(date +%Y-%m-%d) echo "Today is $current_date" # Use command output as argument echo "There are $(ls | wc -l) files in this directory" # Nested command substitution echo "The user $(whoami) is logged in from $(hostname)" # Create backup with timestamp cp important.conf important.conf.$(date +%Y%m%d) # Find and process files for file in $(find /tmp -name "*.tmp"); do rm "$file" done # Check disk usage of specific directory disk_usage=$(du -sh /var/log | awk '{print $1}') echo "Log directory size: $disk_usage"

Combining Commands

Operator Description Example
; Run commands sequentially cd /tmp; ls
&& Run second command only if first succeeds mkdir test && cd test
|| Run second command only if first fails ping -c1 server || echo "Server down"
& Run command in background long_process &

Command Combination Examples

# Sequential execution (both run regardless of success) command1; command2; command3 # Conditional execution (AND) cd /var/www/html && git pull && sudo systemctl restart apache2 # Conditional execution (OR) ping -c1 google.com || echo "No internet connection" # Combined conditional command1 && command2 || command3 # If command1 succeeds, run command2; otherwise run command3 # Background execution ./long_running_script.sh & # Multiple conditions mkdir backup && cp *.txt backup/ && echo "Backup complete" || echo "Backup failed"

‚ö†Ô∏è Common Pitfall: Understanding && and ||

Be careful with the order of operations when combining && and ||:

# This might not work as expected: command1 && command2 || command3 # If command2 fails, command3 will run! # Better approach: if command1; then command2 else command3 fi

Process Management

Understanding and managing processes is crucial for system administration. Processes are running instances of programs, and you'll need to monitor, control, and troubleshoot them regularly.

Viewing Processes

üìñ What is a Process?

A process is an instance of a running program. Each process has a unique Process ID (PID), consumes system resources (CPU, memory), and has a parent process (except for init/systemd, PID 1).

The ps Command

Command Description Use Case
ps Show processes for current user Quick check of your processes
ps aux Show all processes (BSD style) System-wide process view
ps -ef Show all processes (Unix style) Alternative to ps aux
ps -u username Show processes for specific user User-specific troubleshooting

ps Command Examples

# Show all processes ps aux # Show processes in tree format ps auxf # Show processes for specific user ps -u www-data # Find specific process ps aux | grep apache2 # Show processes sorted by CPU usage ps aux --sort=-%cpu | head -10 # Show processes sorted by memory usage ps aux --sort=-%mem | head -10 # Custom output format ps -eo pid,ppid,cmd,%cpu,%mem --sort=-%cpu | head -10

üí° Understanding ps aux Output

USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND root 1 0.0 0.1 169604 11896 ? Ss Jan15 0:03 /sbin/init www-data 1234 2.5 1.5 234567 98765 ? S 10:23 1:45 /usr/sbin/apache2
  • USER: Process owner
  • PID: Process ID
  • %CPU: CPU usage percentage
  • %MEM: Memory usage percentage
  • VSZ: Virtual memory size (KB)
  • RSS: Resident set size (physical memory, KB)
  • STAT: Process state (R=running, S=sleeping, Z=zombie, etc.)
  • START: Start time
  • TIME: CPU time consumed
  • COMMAND: Command that started the process

The top Command

üìñ What is top?

top provides a real-time, dynamic view of running processes. It updates continuously and shows system summary information plus a list of processes sorted by resource usage.

Using top

# Start top top # Common top commands (while running): # q - quit # k - kill process (prompts for PID) # M - sort by memory usage # P - sort by CPU usage # 1 - show individual CPU cores # h - help # u - filter by user # Run top in batch mode (non-interactive) top -b -n 1 > top_snapshot.txt # Show processes for specific user top -u www-data # Update every 5 seconds instead of default 3 top -d 5

Alternative: htop

üí° htop - An Improved top

htop is an enhanced version of top with a more user-friendly interface, color coding, and mouse support. It may need to be installed separately:

# Install htop sudo apt install htop # Debian/Ubuntu sudo yum install htop # RHEL/CentOS # Run htop htop

Features of htop:

  • Color-coded display
  • Horizontal and vertical scrolling
  • Mouse support
  • Tree view of processes
  • Easier to kill processes

Process Control

Killing Processes

Command Description Use Case
kill PID Send TERM signal (graceful shutdown) Standard way to stop a process
kill -9 PID Send KILL signal (force kill) When process won't respond to TERM
killall name Kill all processes with name Stop all instances of a program
pkill pattern Kill processes matching pattern Pattern-based process termination

Process Termination Examples

# Find and kill a process ps aux | grep firefox kill 1234 # Replace 1234 with actual PID # Force kill if needed kill -9 1234 # Kill all Firefox processes killall firefox # Kill processes by pattern pkill -f "python.*script.py" # Kill processes for specific user pkill -u username # Send different signals kill -SIGHUP 1234 # Reload configuration kill -SIGTERM 1234 # Graceful shutdown (default) kill -SIGKILL 1234 # Force kill (same as -9)

‚ö†Ô∏è Warning: Use kill -9 Carefully

The kill -9 (SIGKILL) command forcefully terminates a process without allowing it to clean up. This can lead to:

  • Corrupted data files
  • Orphaned child processes
  • Unreleased resources
  • Lock files left behind

Always try kill (SIGTERM) first, and only use kill -9 as a last resort!

Background and Foreground Processes

üìñ Foreground vs Background

  • Foreground process: Runs in the current terminal and receives keyboard input. You must wait for it to complete before running another command.
  • Background process: Runs independently of the terminal. You can continue using the terminal while it runs.
Command Description
command & Start command in background
Ctrl+Z Suspend current foreground process
bg Resume suspended process in background
fg Bring background process to foreground
jobs List background jobs

Background Process Examples

# Start process in background ./long_running_script.sh & # List background jobs jobs # Output: [1]+ Running ./long_running_script.sh & # Suspend current process (Ctrl+Z), then resume in background ./script.sh # Press Ctrl+Z bg # Bring to foreground fg %1 # %1 refers to job number 1 # Start multiple background jobs ./job1.sh & ./job2.sh & ./job3.sh & jobs # Bring specific job to foreground fg %2 # Kill background job kill %1

Process Priority

üìñ Nice Values

Process priority is controlled by "nice" values ranging from -20 (highest priority) to 19 (lowest priority). Default is 0. Only root can set negative nice values.

Priority Management

# Start process with lower priority (nice value 10) nice -n 10 ./cpu_intensive_task.sh # Start with higher priority (requires root) sudo nice -n -10 ./important_task.sh # Change priority of running process renice -n 5 -p 1234 # Set PID 1234 to nice value 5 # Change priority for all processes of a user sudo renice -n 10 -u username # View nice values ps -eo pid,ni,cmd | grep script.sh

Memory Usage

üìñ Understanding Memory Usage

Linux memory management is complex. The system uses "available" memory for caching to improve performance. This cached memory is automatically released when applications need it, so high memory usage doesn't necessarily indicate a problem.

Memory Monitoring Commands

# Show memory usage (human-readable) free -h # Show memory in MB free -m # Continuous updates every 2 seconds free -h -s 2 # Show memory usage from /proc/meminfo cat /proc/meminfo # Show memory-intensive processes ps aux --sort=-%mem | head -10 # Detailed memory statistics vmstat 1 5 # Update every 1 second, 5 times

üí° Reading free Output

$ free -h total used free shared buff/cache available Mem: 7.7G 2.1G 3.2G 156M 2.4G 5.2G Swap: 2.0G 0B 2.0G
  • total: Total installed RAM
  • used: Memory used by processes
  • free: Completely unused memory
  • shared: Memory used by tmpfs
  • buff/cache: Memory used for caching (can be freed if needed)
  • available: Memory available for new applications (most important number!)
