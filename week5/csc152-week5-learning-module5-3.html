Introduction to PowerShell

Welcome to Module 5.3! Building on your introduction to PowerShell from Week 4 (Module 4.6) and your Bash scripting experience from Modules 5.1 and 5.2, this module provides comprehensive coverage of PowerShell cmdlets, the PowerShell pipeline, and object-oriented command-line administration in Windows Server environments.

üìñ What is PowerShell?

PowerShell is Microsoft's task automation and configuration management framework, consisting of a command-line shell and scripting language built on the .NET Framework. Unlike traditional command-line interfaces that work with text, PowerShell works with objects, making it more powerful and flexible for system administration.

Key characteristics:

  • Object-based: Commands pass .NET objects, not text
  • Consistent syntax: Verb-Noun naming convention
  • Discoverable: Built-in help and command discovery
  • Extensible: Modules and custom cmdlets
  • Cross-platform: PowerShell Core runs on Windows, Linux, and macOS

PowerShell vs. Bash: Key Differences

üêß Bash (Linux/Unix)

  • Text-based: Commands output text strings
  • Parsing required: Use grep, awk, sed to extract data
  • Case-sensitive: Commands and variables
  • Compact syntax: Short commands (ls, cd, grep)
  • Pipeline: Passes text between commands
  • Philosophy: Do one thing well, chain tools

‚ö° PowerShell (Windows)

  • Object-based: Commands output .NET objects
  • Direct access: Access properties without parsing
  • Case-insensitive: Commands and variables
  • Verbose syntax: Descriptive cmdlet names
  • Pipeline: Passes objects between commands
  • Philosophy: Consistency, discoverability, power

Comparison Example: Listing Processes

Bash approach (text-based):

#!/bin/bash
# Get processes using more than 10% CPU
ps aux | awk '$3 > 10 {print $2, $3, $11}'

# Output is text that must be parsed
# 1234 15.2 /usr/bin/firefox
# 5678 12.8 /usr/bin/chrome

PowerShell approach (object-based):

# Get processes using more than 10% CPU
Get-Process | Where-Object {$_.CPU -gt 10} | 
    Select-Object Id, CPU, ProcessName

# Output is objects with properties
# Id  CPU      ProcessName
# --  ---      -----------
# 1234 15.2    firefox
# 5678 12.8    chrome

Key difference: In PowerShell, you work directly with process objects and their properties. In Bash, you work with text and must parse it.

Cmdlet Fundamentals

Cmdlets (pronounced "command-lets") are the building blocks of PowerShell. Understanding their structure and naming convention is essential for PowerShell proficiency.

The Verb-Noun Naming Convention

üìñ Cmdlet Naming Convention

All PowerShell cmdlets follow a consistent Verb-Noun naming pattern:

Verb-Noun

Example: Get-Process

  • Verb = Action to perform (Get, Set, New, Remove, Start, Stop, etc.)
  • Noun = Resource to act upon (Process, Service, Item, Content, etc.)

Why this convention?

  • Consistency: All cmdlets follow the same pattern
  • Predictability: If you know one cmdlet, you can guess others
  • Discoverability: Easy to find cmdlets for specific tasks
  • Readability: Self-documenting code

Common PowerShell Verbs

Verb Action Examples
Get Retrieve information Get-Process, Get-Service, Get-ChildItem
Set Modify existing resource Set-Service, Set-Location, Set-Content
New Create new resource New-Item, New-Service, New-ADUser
Remove Delete resource Remove-Item, Remove-Service, Remove-ADUser
Start Begin operation Start-Service, Start-Process, Start-Job
Stop End operation Stop-Service, Stop-Process, Stop-Computer
Restart Stop and start Restart-Service, Restart-Computer
Test Verify condition Test-Connection, Test-Path, Test-NetConnection
Enable Activate feature Enable-PSRemoting, Enable-WindowsFeature
Disable Deactivate feature Disable-PSRemoting, Disable-WindowsFeature

Verb-Noun Pattern Examples

# Get cmdlets follow predictable pattern
Get-Process      # Get running processes
Get-Service      # Get Windows services
Get-EventLog     # Get event log entries
Get-ChildItem    # Get files and directories
Get-Content      # Get file contents
Get-ComputerInfo # Get computer information

# Set cmdlets modify existing resources
Set-Service      # Modify service configuration
Set-Location     # Change current directory
Set-Content      # Write content to file
Set-ExecutionPolicy  # Change script execution policy

# New cmdlets create resources
New-Item         # Create new file or directory
New-Service      # Create new Windows service
New-LocalUser    # Create new local user account

# Remove cmdlets delete resources
Remove-Item      # Delete file or directory
Remove-Service   # Delete Windows service
Remove-LocalUser # Delete local user account

# Start/Stop cmdlets control operations
Start-Service    # Start a Windows service
Stop-Process     # Stop a running process
Restart-Computer # Restart the computer

Cmdlet Parameters

üìñ Cmdlet Parameters

Parameters modify how a cmdlet operates. They provide additional information or options to the cmdlet.

Parameter syntax:

Cmdlet-Name -ParameterName ParameterValue

# Examples:
Get-Process -Name "chrome"
Get-Service -ComputerName "Server01"
Get-ChildItem -Path "C:\Windows" -Recurse

Parameter types:

  • Named parameters: Explicitly specify parameter name
  • Positional parameters: Order determines which parameter
  • Switch parameters: Boolean flags (present = true, absent = false)
  • Required parameters: Must be provided
  • Optional parameters: Have default values

Parameter Examples

# Named parameters (explicit)
Get-Process -Name "notepad"
Get-ChildItem -Path "C:\Windows" -Filter "*.exe"

# Positional parameters (order matters)
Get-Process "notepad"           # Name is first positional parameter
Get-ChildItem "C:\Windows"      # Path is first positional parameter

# Switch parameters (boolean flags)
Get-ChildItem -Recurse          # Include subdirectories
Get-ChildItem -Force            # Include hidden files
Get-Service -DependentServices  # Show dependent services

# Multiple parameters
Get-ChildItem -Path "C:\Windows" -Filter "*.log" -Recurse -Force

# Parameter shortcuts (aliases)
Get-ChildItem -Path "C:\Windows"   # Full parameter name
Get-ChildItem -P "C:\Windows"      # Abbreviated (minimum unique)
gci "C:\Windows"                   # Alias for cmdlet

# Common parameters (available on all cmdlets)
Get-Process -Verbose               # Show detailed information
Get-Service -ErrorAction SilentlyContinue  # Suppress errors
Get-ChildItem -WhatIf              # Preview without executing

Cmdlet Aliases

üìñ PowerShell Aliases

Aliases are shortcut names for cmdlets. PowerShell includes many built-in aliases for common cmdlets, including aliases that match Linux/Unix commands.

Cmdlet Aliases Purpose
Get-ChildItem gci, ls, dir List files and directories
Set-Location sl, cd, chdir Change directory
Copy-Item cpi, cp, copy Copy files
Remove-Item ri, rm, del Delete files
Move-Item mi, mv, move Move files
Get-Content gc, cat, type Read file contents
Write-Output echo, write Display output
Select-Object select Select properties
Where-Object where, ? Filter objects
ForEach-Object foreach, % Process each object

Using Aliases

# These are equivalent:
Get-ChildItem
gci
ls
dir

# These are equivalent:
Set-Location C:\Windows
sl C:\Windows
cd C:\Windows
chdir C:\Windows

# View all aliases
Get-Alias

# Find aliases for a cmdlet
Get-Alias -Definition Get-ChildItem

# Find cmdlet for an alias
Get-Alias ls

# Create your own alias
New-Alias -Name list -Value Get-ChildItem

# Remove an alias
Remove-Alias -Name list

üí° Best Practice: When to Use Aliases

  • Interactive use: Aliases are great for typing commands quickly
  • Scripts: Use full cmdlet names for clarity and maintainability
  • Documentation: Always use full cmdlet names
  • Shared code: Full names prevent confusion
# Good for interactive use:
gci | where {$_.Length -gt 1MB}

# Better for scripts:
Get-ChildItem | Where-Object {$_.Length -gt 1MB}

Essential Administrative Cmdlets

These cmdlets form the foundation of PowerShell server administration. Mastering them enables you to perform most common administrative tasks from the command line.

File System Management

Get-ChildItem
List files and directories (like ls in Linux or dir in CMD)
# Basic usage
Get-ChildItem

# List specific directory
Get-ChildItem C:\Windows

# List with filter
Get-ChildItem -Path C:\Windows -Filter "*.exe"

# Recursive listing
Get-ChildItem -Path C:\Windows -Recurse

# Show hidden files
Get-ChildItem -Force

# List only directories
Get-ChildItem -Directory

# List only files
Get-ChildItem -File

# Combine parameters
Get-ChildItem -Path C:\Windows\System32 -Filter "*.dll" -Recurse -Force

Output:

    Directory: C:\Windows


Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d-----        12/7/2023   3:14 PM                Program Files
d-----        12/7/2023   3:14 PM                Windows
-a----        12/7/2023   3:14 PM          12345 test.txt
Set-Location
Change current directory (like cd in Linux/CMD)
# Change to specific directory
Set-Location C:\Windows

# Change to parent directory
Set-Location ..

# Change to user's home directory
Set-Location ~

# Return to previous location
Set-Location -

# Using alias
cd C:\Windows
sl C:\Windows
Copy-Item
Copy files and directories
# Copy file
Copy-Item -Path "C:\source\file.txt" -Destination "C:\destination\"

# Copy and rename
Copy-Item -Path "C:\source\file.txt" -Destination "C:\destination\newname.txt"

# Copy directory recursively
Copy-Item -Path "C:\source\folder" -Destination "C:\destination\" -Recurse

# Copy with confirmation
Copy-Item -Path "C:\source\*" -Destination "C:\destination\" -Confirm

# Using alias
cp source.txt destination.txt
Remove-Item
Delete files and directories
# Delete file
Remove-Item -Path "C:\temp\file.txt"

# Delete directory and contents
Remove-Item -Path "C:\temp\folder" -Recurse

# Delete with confirmation
Remove-Item -Path "C:\temp\*" -Confirm

# Force delete (including read-only)
Remove-Item -Path "C:\temp\file.txt" -Force

# Preview deletion without executing
Remove-Item -Path "C:\temp\*" -WhatIf

# Using alias
rm file.txt
del file.txt
New-Item
Create new files and directories
# Create directory
New-Item -Path "C:\temp\newfolder" -ItemType Directory

# Create file
New-Item -Path "C:\temp\newfile.txt" -ItemType File

# Create file with content
New-Item -Path "C:\temp\newfile.txt" -ItemType File -Value "Hello, World!"

# Create multiple directories
New-Item -Path "C:\temp\dir1", "C:\temp\dir2", "C:\temp\dir3" -ItemType Directory

# Using alias
mkdir newfolder

Service Management

Get-Service
Retrieve Windows service information
# List all services
Get-Service

# Get specific service
Get-Service -Name "wuauserv"

# Get services by display name pattern
Get-Service -DisplayName "*Windows Update*"

# Get only running services
Get-Service | Where-Object {$_.Status -eq "Running"}

# Get stopped services
Get-Service | Where-Object {$_.Status -eq "Stopped"}

# Get services that can be stopped
Get-Service | Where-Object {$_.CanStop -eq $true}

# Get services on remote computer
Get-Service -ComputerName "Server01"

Output:

Status   Name               DisplayName
------   ----               -----------
Running  wuauserv           Windows Update
Stopped  BITS               Background Intelligent Transfer Ser...
Running  Spooler            Print Spooler
Start-Service / Stop-Service / Restart-Service
Control Windows services
# Start a service
Start-Service -Name "wuauserv"

# Stop a service
Stop-Service -Name "wuauserv"

# Restart a service
Restart-Service -Name "wuauserv"

# Start with confirmation
Start-Service -Name "wuauserv" -Confirm

# Start and wait for it to complete
Start-Service -Name "wuauserv" -PassThru

# Stop multiple services
Stop-Service -Name "service1", "service2", "service3"

# Restart all services matching pattern
Get-Service -Name "App*" | Restart-Service

Process Management

Get-Process
Get information about running processes
# List all processes
Get-Process

# Get specific process
Get-Process -Name "notepad"

# Get processes using wildcard
Get-Process -Name "chrome*"

# Get process by ID
Get-Process -Id 1234

# Get processes sorted by CPU usage
Get-Process | Sort-Object CPU -Descending

# Get processes using more than 100MB memory
Get-Process | Where-Object {$_.WorkingSet -gt 100MB}

# Get top 10 memory consumers
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10

# Get processes on remote computer
Get-Process -ComputerName "Server01"

Stop-Process
Terminate running processes
# Stop process by name
Stop-Process -Name "notepad"

# Stop process by ID
Stop-Process -Id 1234

# Force stop process
Stop-Process -Name "chrome" -Force

# Stop with confirmation
Stop-Process -Name "notepad" -Confirm

# Stop all processes matching pattern
Get-Process -Name "chrome*" | Stop-Process

# Stop processes using more than 500MB
Get-Process | Where-Object {$_.WorkingSet -gt 500MB} | Stop-Process

System Information

Get-ComputerInfo
Get comprehensive computer information
# Get all computer information
Get-ComputerInfo

# Get specific properties
Get-ComputerInfo | Select-Object WindowsProductName, WindowsVersion, OsArchitecture

# Get OS information
Get-ComputerInfo | Select-Object Os*

# Get BIOS information
Get-ComputerInfo | Select-Object Bios*

# Get network information
Get-ComputerInfo | Select-Object *Network*

# Get time zone
Get-ComputerInfo | Select-Object TimeZone
Get-EventLog
Read Windows event logs
# List available event logs
Get-EventLog -List

# Get latest 20 entries from System log
Get-EventLog -LogName System -Newest 20

# Get error entries
Get-EventLog -LogName System -EntryType Error -Newest 50

# Get entries from specific source
Get-EventLog -LogName Application -Source "Application Error"

# Get entries from last 24 hours
Get-EventLog -LogName System -After (Get-Date).AddDays(-1)

# Get entries with specific Event ID
Get-EventLog -LogName System -InstanceId 1074

Network Commands

Test-Connection
Test network connectivity (like ping)
# Ping a computer
Test-Connection -ComputerName "google.com"

# Ping with specific count
Test-Connection -ComputerName "google.com" -Count 4

# Quiet mode (returns boolean)
Test-Connection -ComputerName "google.com" -Quiet

# Ping multiple computers
Test-Connection -ComputerName "server1", "server2", "server3"

# Continuous ping
Test-Connection -ComputerName "google.com" -Count ([int]::MaxValue)
Get-NetIPAddress
Get IP address configuration
# Get all IP addresses
Get-NetIPAddress

# Get IPv4 addresses only
Get-NetIPAddress -AddressFamily IPv4

# Get IP addresses for specific adapter
Get-NetIPAddress -InterfaceAlias "Ethernet"

# Get IP configuration (like ipconfig)
Get-NetIPConfiguration
