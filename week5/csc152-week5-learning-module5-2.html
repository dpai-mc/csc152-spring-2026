Introduction to Bash Scripting

Welcome to Module 5.2! Building on the advanced Bash commands you learned in Module 5.1, this module introduces you to Bash scripting‚Äîthe art of combining commands into automated, reusable tools. Scripting transforms you from a command executor into a system automation expert.

üìñ What is a Bash Script?

A Bash script is a text file containing a series of Bash commands that are executed sequentially. Scripts automate repetitive tasks, combine multiple commands into workflows, and create custom administrative tools. Think of a script as a recipe‚Äîa set of instructions that can be followed repeatedly to achieve consistent results.

Why Learn Bash Scripting?

Manual Command Execution

  • Type commands one at a time
  • Easy to make typos or mistakes
  • Time-consuming for repetitive tasks
  • Hard to remember complex sequences
  • Not easily shareable
  • No consistency guarantee

Automated Script Execution

  • Execute hundreds of commands instantly
  • Consistent, error-free execution
  • Saves hours on repetitive tasks
  • Self-documenting (with comments)
  • Easily shared with team
  • Guaranteed consistency

Real-World Scripting Scenarios

üí° System Administration Use Cases

1. User Account Creation

Instead of manually creating users one at a time, a script can:

  • Read user data from a CSV file
  • Create user accounts with proper settings
  • Set up home directories
  • Configure group memberships
  • Send welcome emails
  • Log all actions for auditing

2. System Backup

A backup script can:

  • Identify files that changed today
  • Create compressed archives
  • Upload to remote storage
  • Verify backup integrity
  • Send success/failure notifications
  • Clean up old backups automatically

3. Log File Analysis

An analysis script can:

  • Parse Apache/Nginx logs
  • Identify error patterns
  • Count requests by IP address
  • Generate HTML reports
  • Email alerts for critical issues

4. System Health Monitoring

A monitoring script can:

  • Check disk space usage
  • Monitor CPU and memory
  • Verify critical services are running
  • Test network connectivity
  • Alert administrators if thresholds exceeded

üí° Learning Approach

Scripting is a hands-on skill. The most effective way to learn is by:

  1. Start Simple: Begin with basic scripts and gradually add complexity
  2. Type, Don't Copy: Type scripts yourself to build muscle memory
  3. Make Mistakes: Errors are learning opportunities‚Äîembrace them!
  4. Test Frequently: Run your script after each addition
  5. Read Others' Scripts: Study well-written scripts to learn techniques
  6. Comment Everything: Explain your code‚Äîit helps you understand it better

‚ö†Ô∏è Safety First

Scripts can be powerful‚Äîand dangerous if misused. Always:

  • Test scripts in a safe environment (VM or test directory)
  • Use echo to preview destructive commands before executing
  • Back up important data before running scripts that modify files
  • Be especially careful with commands like rm, mv, and chmod
  • Validate user input to prevent unexpected behavior

Script Basics

Let's create your first Bash script and understand the fundamental components that make scripts work.

Creating Your First Script

Step-by-Step: Hello World Script

Step 1: Create the Script File

# Create a new file called hello.sh nano hello.sh # Or use vim if you prefer vim hello.sh

Step 2: Add the Shebang and Content

#!/bin/bash # My first Bash script # Author: Your Name # Date: February 17, 2026 echo "Hello, World!" echo "Welcome to Bash scripting!" echo "Today is $(date)"

Step 3: Save and Exit

  • In nano: Press Ctrl+X, then Y, then Enter
  • In vim: Press Esc, type :wq, press Enter

Step 4: Make the Script Executable

# Add execute permission chmod +x hello.sh # Verify permissions ls -l hello.sh # Output: -rwxr-xr-x 1 user user 123 Feb 17 10:30 hello.sh

Step 5: Execute the Script

# Method 1: Direct execution (requires execute permission) ./hello.sh # Method 2: Explicit bash invocation (no execute permission needed) bash hello.sh # Method 3: Source the script (runs in current shell) source hello.sh # or . hello.sh

Expected Output:

Hello, World! Welcome to Bash scripting! Today is Mon Feb 17 10:30:45 PST 2026

Understanding the Shebang

üìñ What is a Shebang?

The shebang (also called "hashbang") is the #!/bin/bash line at the beginning of a script. It tells the system which interpreter to use to execute the script.

Syntax: #! followed by the absolute path to the interpreter

Common shebangs:

  • #!/bin/bash - Bash shell (most common)
  • #!/bin/sh - POSIX shell (more portable)
  • #!/usr/bin/env bash - Finds bash in PATH (portable)
  • #!/usr/bin/python3 - Python 3 script
  • #!/usr/bin/perl - Perl script

üí° Shebang Best Practice

Use #!/usr/bin/env bash for maximum portability:

#!/usr/bin/env bash # This works even if bash is installed in a non-standard location

This searches for bash in the user's PATH, making the script more portable across different systems.

Comments in Scripts

üìñ Script Comments

Comments are lines that the shell ignores. They document your code for humans (including future you!).

Syntax: Lines beginning with # (except the shebang)

Comment Examples and Best Practices

#!/bin/bash # ============================================ # Script: backup_system.sh # Author: John Doe # Date: February 17, 2026 # Description: Creates a compressed backup of # important system directories # Usage: ./backup_system.sh [destination] # ============================================ # Single-line comment explaining the next command echo "Starting backup process..." # Multi-line comment for complex logic # This section checks if the backup directory exists # If it doesn't, the script creates it with proper permissions # The -p flag creates parent directories if needed if [ ! -d "/backup" ]; then mkdir -p /backup # Create backup directory fi # Inline comment explaining a specific parameter tar -czf backup.tar.gz /etc /home # -c=create -z=gzip -f=file # TODO: Add email notification feature # FIXME: Handle case when disk is full # NOTE: This requires root privileges

Script Structure and Organization

Well-Structured Script Template

#!/bin/bash # ============================================ # Script Header (Metadata) # ============================================ # Script Name: template.sh # Author: Your Name # Date: 2026-02-17 # Version: 1.0 # Description: Brief description of what the script does # Usage: ./template.sh [arguments] # ============================================ # ============================================ # Configuration Section # ============================================ # Set script-wide variables and constants BACKUP_DIR="/backup" LOG_FILE="/var/log/script.log" MAX_RETRIES=3 # ============================================ # Function Definitions # ============================================ # Define functions before main script logic function log_message() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE" } function check_prerequisites() { # Check if required commands are available command -v tar >/dev/null 2>&1 || { echo "tar is required"; exit 1; } } # ============================================ # Main Script Logic # ============================================ # Check prerequisites check_prerequisites # Main operations log_message "Script started" # Your main script code here log_message "Script completed successfully" # ============================================ # Exit with appropriate status # ============================================ exit 0

Script Execution Methods

Method Command Requirements Behavior
Direct execution ./script.sh Execute permission, shebang Runs in new subshell
Bash invocation bash script.sh None Runs in new bash subshell
Source source script.sh None Runs in current shell
Source (short) . script.sh None Runs in current shell

‚ö†Ô∏è Subshell vs. Current Shell

Subshell (./script.sh or bash script.sh):

  • Creates a new shell process
  • Variables set in script don't affect parent shell
  • Use for most scripts

Current shell (source script.sh or . script.sh):

  • Runs in your current shell
  • Variables set in script affect current shell
  • Use for environment setup scripts
# Example demonstrating the difference echo 'export MY_VAR="hello"' > test.sh # Method 1: Subshell (variable not available after) bash test.sh echo $MY_VAR # Output: (empty) # Method 2: Source (variable available after) source test.sh echo $MY_VAR # Output: hello

Exit Codes and Status

üìñ Exit Codes

Every command and script returns an exit code (also called exit status or return code) when it finishes:

  • 0 = Success
  • 1-255 = Various error conditions

The special variable $? contains the exit code of the last executed command.

Working with Exit Codes

#!/bin/bash # Check if a file exists ls /etc/passwd echo "Exit code: $?" # Output: 0 (success) ls /nonexistent/file echo "Exit code: $?" # Output: 2 (file not found) # Use exit codes in scripts if ls /etc/passwd > /dev/null 2>&1; then echo "File exists" else echo "File does not exist" fi # Set custom exit codes function divide() { if [ "$2" -eq 0 ]; then echo "Error: Division by zero" return 1 # Return error code fi echo $(( $1 / $2 )) return 0 # Return success } divide 10 2 echo "Exit code: $?" # Output: 0 divide 10 0 echo "Exit code: $?" # Output: 1

Common Exit Codes

Exit Code Meaning Example
0 Success Command completed successfully
1 General error Catchall for general errors
2 Misuse of shell builtin Missing keyword or command
126 Command cannot execute Permission problem or not executable
127 Command not found Possible problem with $PATH
128+n Fatal error signal "n" 130 = terminated by Ctrl+C (128+2)

Variables and Data Types

Variables are containers that store data. They're essential for creating dynamic, flexible scripts that can work with different inputs and conditions.

Variable Declaration and Assignment

üìñ Bash Variables

In Bash, variables are created when you assign a value to them. There's no need to declare them beforehand or specify a type.

Syntax:

VARIABLE_NAME="value" # No spaces around =

Important rules:

  • No spaces around the = sign
  • Variable names are case-sensitive
  • By convention, use UPPERCASE for environment/global variables
  • Use lowercase for local variables
  • Variable names can contain letters, numbers, and underscores
  • Cannot start with a number

Variable Assignment Examples

#!/bin/bash # String variables name="John Doe" server="web01.example.com" message='Hello, World!' # Single quotes: literal string # Numeric variables (stored as strings but can be used in arithmetic) age=25 port=8080 count=0 # Variables with command output current_date=$(date +%Y-%m-%d) user_home=$(echo $HOME) file_count=$(ls -1 | wc -l) # Empty variables empty_var="" unset_var= # Multiple assignments first="John" last="Doe" age=30 # Using variables echo "Name: $name" echo "Age: $age" echo "Date: $current_date"

‚ö†Ô∏è Common Variable Mistakes

# WRONG: Spaces around = name = "John" # Error: command not found # WRONG: $ in assignment $name="John" # Error: command not found # RIGHT: No spaces name="John" # Correct # WRONG: Starting with number 1name="John" # Error: invalid variable name # RIGHT: Letter or underscore first name1="John" # Correct _name="John" # Correct

Using Variables

Variable Expansion

#!/bin/bash name="Alice" # Basic expansion echo $name # Output: Alice echo "Hello, $name" # Output: Hello, Alice # Brace expansion (recommended) echo ${name} # Output: Alice echo "Hello, ${name}!" # Output: Hello, Alice! # Why braces matter echo "$nameTest" # Output: (empty - looks for variable nameTest) echo "${name}Test" # Output: AliceTest # Concatenation full_name="${name} Smith" echo $full_name # Output: Alice Smith # In commands mkdir "${name}_backup" cp file.txt "${name}_file.txt"

Special Variables

Variable Description Example Value
$0 Script name ./myscript.sh
$1, $2, $3... Positional parameters (arguments) First, second, third argument
$# Number of arguments 3
$@ All arguments as separate words "arg1" "arg2" "arg3"
$* All arguments as single word "arg1 arg2 arg3"
$? Exit status of last command 0 (success) or 1-255 (error)
$$ Process ID of current script 12345
$! PID of last background command 12346

Command-Line Arguments

#!/bin/bash # Script: greet.sh # Usage: ./greet.sh John 25 echo "Script name: $0" echo "First argument: $1" echo "Second argument: $2" echo "Number of arguments: $#" echo "All arguments: $@" # Using arguments name=$1 age=$2 if [ $# -lt 2 ]; then echo "Usage: $0 " exit 1 fi echo "Hello, $name! You are $age years old."

Running the script:

$ ./greet.sh John 25 Script name: ./greet.sh First argument: John Second argument: 25 Number of arguments: 2 All arguments: John 25 Hello, John! You are $age years old.

Reading User Input

Interactive Input with read

#!/bin/bash # Basic input echo "What is your name?" read name echo "Hello, $name!" # Prompt on same line read -p "Enter your age: " age echo "You are $age years old" # Silent input (for passwords) read -sp "Enter password: " password echo # New line after password echo "Password saved (not really!)" # Reading multiple values read -p "Enter first and last name: " first last echo "First: $first, Last: $last" # Default value with timeout read -t 5 -p "Enter choice (default: yes): " choice choice=${choice:-yes} # Use 'yes' if empty echo "You chose: $choice" # Reading from file while read line; do echo "Line: $line" done < input.txt

Environment Variables

üìñ Environment Variables

Environment variables are variables that are available to all processes started from the current shell. They're typically set by the system or user configuration files.

Variable Description Example
$HOME User's home directory /home/username
$USER Current username username
$PATH Executable search path /usr/bin:/bin:/usr/local/bin
$PWD Current working directory /home/username/projects
$SHELL Current shell /bin/bash
$HOSTNAME System hostname server01

Working with Environment Variables

#!/bin/bash # View environment variables echo "Home: $HOME" echo "User: $USER" echo "Path: $PATH" # Create environment variable (available to child processes) export MY_VAR="Hello" # Create local variable (not available to child processes) local_var="World" # Check if variable is set if [ -z "$MY_VAR" ]; then echo "MY_VAR is not set" else echo "MY_VAR is set to: $MY_VAR" fi # Unset variable unset MY_VAR # List all environment variables env # List all variables (including local) set

Variable Scope

üìñ Variable Scope

Scope determines where a variable can be accessed:

  • Global variables: Available throughout the script
  • Local variables: Available only within a function
  • Environment variables: Available to child processes

Variable Scope Example

#!/bin/bash # Global variable global_var="I'm global" function test_scope() { # Local variable (only exists in this function) local local_var="I'm local" # Can access global variable echo "Inside function:" echo " Global: $global_var" echo " Local: $local_var" # Modify global variable global_var="Modified global" } # Call function test_scope # After function echo "Outside function:" echo " Global: $global_var" echo " Local: $local_var" # Empty - local_var doesn't exist here

Output:

Inside function: Global: I'm global Local: I'm local Outside function: Global: Modified global Local:

Conditional Statements

Conditional statements allow your scripts to make decisions and execute different code based on conditions. This is what transforms simple command sequences into intelligent programs.

The if Statement

üìñ if/then/else Structure

The if statement executes commands based on whether a condition is true or false.

Basic syntax:

if [ condition ]; then # commands if condition is true fi

With else:

if [ condition ]; then # commands if condition is true else # commands if condition is false fi

With elif (else if):

if [ condition1 ]; then # commands if condition1 is true elif [ condition2 ]; then # commands if condition2 is true else # commands if all conditions are false fi

‚ö†Ô∏è Important Syntax Notes

  • Spaces are required after [ and before ]
  • Semicolon before then is required (or then on next line)
  • Always close with fi (if backwards)
  • Use [[ ]] for advanced conditions (recommended in Bash)

Test Conditions and Operators

File Test Operators

Operator Description Example
-e file File exists [ -e /etc/passwd ]
-f file File exists and is regular file [ -f script.sh ]
-d file File exists and is directory [ -d /home ]
-r file File exists and is readable [ -r config.txt ]
-w file File exists and is writable [ -w log.txt ]
-x file File exists and is executable [ -x script.sh ]
-s file File exists and is not empty [ -s data.txt ]
-L file File exists and is symbolic link [ -L link ]

File Test Examples

#!/bin/bash file="/etc/passwd" # Check if file exists if [ -e "$file" ]; then echo "File exists" fi # Check if it's a regular file if [ -f "$file" ]; then echo "It's a regular file" fi # Check if readable if [ -r "$file" ]; then echo "File is readable" fi # Multiple conditions if [ -f "$file" ] && [ -r "$file" ]; then echo "File exists and is readable" cat "$file" fi # Check if directory exists, create if not backup_dir="/backup" if [ ! -d "$backup_dir" ]; then echo "Creating backup directory..." mkdir -p "$backup_dir" fi

String Comparison Operators

Operator Description Example
= or == Strings are equal [ "$a" = "$b" ]
!= Strings are not equal [ "$a" != "$b" ]
-z string String is empty (zero length) [ -z "$var" ]
-n string String is not empty [ -n "$var" ]
< String is less than (alphabetically) [[ "$a" < "$b" ]]
> String is greater than [[ "$a" > "$b" ]]

String Comparison Examples

#!/bin/bash name="Alice" # Check if string equals value if [ "$name" = "Alice" ]; then echo "Hello, Alice!" fi # Check if variable is empty read -p "Enter your name: " input if [ -z "$input" ]; then echo "You didn't enter a name" input="Guest" fi echo "Hello, $input!" # Check if variable is not empty if [ -n "$name" ]; then echo "Name is set to: $name" fi # Case-insensitive comparison (convert to lowercase) read -p "Continue? (yes/no): " answer answer=$(echo "$answer" | tr '[:upper:]' '[:lower:]') if [ "$answer" = "yes" ]; then echo "Continuing..." elif [ "$answer" = "no" ]; then echo "Exiting..." exit 0 else echo "Invalid response" fi

Numeric Comparison Operators

Operator Description Example
-eq Equal to [ "$a" -eq "$b" ]
-ne Not equal to [ "$a" -ne "$b" ]
-lt Less than [ "$a" -lt "$b" ]
-le Less than or equal to [ "$a" -le "$b" ]
-gt Greater than [ "$a" -gt "$b" ]
-ge Greater than or equal to [ "$a" -ge "$b" ]

Numeric Comparison Examples

#!/bin/bash age=25 # Check if age is valid if [ "$age" -ge 18 ]; then echo "You are an adult" else echo "You are a minor" fi # Check disk usage disk_usage=$(df / | tail -1 | awk '{print $5}' | sed 's/%//') if [ "$disk_usage" -gt 90 ]; then echo "WARNING: Disk usage is at ${disk_usage}%" elif [ "$disk_usage" -gt 75 ]; then echo "CAUTION: Disk usage is at ${disk_usage}%" else echo "Disk usage is normal: ${disk_usage}%" fi # Check number of arguments if [ $# -eq 0 ]; then echo "Error: No arguments provided" echo "Usage: $0 " exit 1 elif [ $# -gt 1 ]; then echo "Warning: Only first argument will be used" fi

Logical Operators

Operator Description Example
&& Logical AND [ cond1 ] && [ cond2 ]
|| Logical OR [ cond1 ] || [ cond2 ]
! Logical NOT [ ! cond ]
-a AND (inside [ ]) [ cond1 -a cond2 ]
-o OR (inside [ ]) [ cond1 -o cond2 ]

Logical Operator Examples

#!/bin/bash file="data.txt" min_size=1000 # AND: File must exist AND be larger than min_size if [ -f "$file" ] && [ $(stat -f%z "$file") -gt $min_size ]; then echo "File exists and is large enough" fi # OR: Accept either .txt or .log files if [[ "$file" == *.txt ]] || [[ "$file" == *.log ]]; then echo "Valid file type" fi # NOT: Check if file does NOT exist if [ ! -f "$file" ]; then echo "File does not exist, creating..." touch "$file" fi # Complex condition age=25 has_license=true if [ "$age" -ge 18 ] && [ "$has_license" = true ]; then echo "You can drive" else echo "You cannot drive" fi

The case Statement

üìñ case Statement

The case statement is useful when you need to compare a variable against multiple patterns. It's cleaner than multiple if/elif statements.

Syntax:

case $variable in pattern1) commands ;; pattern2) commands ;; *) default commands ;; esac

case Statement Examples

#!/bin/bash read -p "Enter your choice (start/stop/restart): " action case $action in start) echo "Starting service..." # systemctl start myservice ;; stop) echo "Stopping service..." # systemctl stop myservice ;; restart) echo "Restarting service..." # systemctl restart myservice ;; *) echo "Invalid option: $action" echo "Valid options: start, stop, restart" exit 1 ;; esac # Pattern matching with wildcards filename="document.pdf" case $filename in *.txt) echo "Text file" ;; *.pdf) echo "PDF file" ;; *.jpg|*.png|*.gif) echo "Image file" ;; *) echo "Unknown file type" ;; esac # Multiple patterns read -p "Continue? (y/n): " answer case $answer in y|Y|yes|Yes|YES) echo "Continuing..." ;; n|N|no|No|NO) echo "Exiting..." exit 0 ;; *) echo "Please answer yes or no" ;; esac
