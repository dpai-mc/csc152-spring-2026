Week 10 of 17

Storage Technologies & Management

Welcome to Week 10! This week you move from managing users and permissions into the physical and logical foundations that keep server data safe, organized, and accessible — skills that are central to every systems and network administration role.

🎯 Why This Week Matters

Storage is the backbone of every production server environment. Whether you are deploying a file server, a domain controller, or a cloud-hosted workload, understanding how disks are partitioned, how RAID protects against drive failure, and how Logical Volume Manager (LVM) gives you flexible, live-resizable storage is a day-one expectation for junior sysadmins and cloud engineers.

This week also reinforces your monitoring and log-management skills by connecting storage health to centralized logging with Splunk — bridging the gap between infrastructure management and security operations. By the end of the week you will have a solid conceptual and practical foundation to carry into the domain-controller work coming in Week 11.

📋 Week 10 Learning Goals

By the end of this week you should be able to:

  • Identify and compare common storage technologies used in server environments, including HDDs, SSDs, and NVMe drives.
  • Explain the purpose and structure of MBR and GPT partition tables and create partitions using command-line tools.
  • Describe the major RAID levels (0, 1, 5, 6, 10) and select an appropriate level given a scenario's performance and redundancy requirements.
  • Configure and manage software RAID arrays on a Linux server using mdadm.
  • Explain the three-layer LVM architecture (Physical Volumes, Volume Groups, Logical Volumes) and perform basic LVM operations including creation and live extension.
  • Apply appropriate Linux filesystem types (ext4, XFS, Btrfs) and mount filesystems persistently via /etc/fstab.
  • Use core Linux storage management commands (fdisk, parted, lsblk, df, du, mount) to inspect and manage storage.
  • Configure Splunk to monitor storage-related log sources and apply retention policies aligned with organizational requirements.
  • Recognize storage-related security considerations, including encryption at rest and access control for storage resources.

🔑 Key Concepts to Watch For

As you work through the modules this week, pay particular attention to these terms and ideas. Understanding them deeply will prepare you for Quiz 10, Lab 10, and future project work.

GPT Partition Table RAID 5 Striping with Parity LVM Volume Group Logical Volume Extension mdadm Software RAID /etc/fstab Persistent Mounts Splunk Log Retention Policy Encryption at Rest
Quick concept hints
  • GPT vs. MBR: Know the partition count and disk-size limits that make GPT the modern standard.
  • RAID 5: Requires a minimum of 3 drives; distributes parity across all drives so any single drive can fail without data loss.
  • LVM Volume Group: The pool of storage made up of one or more Physical Volumes — Logical Volumes are carved out of this pool.
  • Live Extension: LVM allows you to grow a Logical Volume and its filesystem while it is still mounted and in use.
  • mdadm: The Linux tool for creating, assembling, and monitoring software RAID arrays.
  • /etc/fstab: The configuration file that tells Linux which filesystems to mount automatically at boot and with what options.
  • Splunk Retention: Retention policies control how long indexed data is kept — critical for compliance and storage capacity planning.
  • Encryption at Rest: Protects data on disk from physical theft; common implementations include LUKS on Linux.
