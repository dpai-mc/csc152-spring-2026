Protecting the infrastructure you built

Welcome to Week 14

The shift from building to defending. Over Weeks 1–13 you assembled a comprehensive server infrastructure — hardware fundamentals, Linux and Windows administration, user and group management, file permissions, networking, storage, directory services (Active Directory and OpenLDAP), virtualization, and a Splunk monitoring stack. Now we secure it.

Week 14 is the primary week for CSLO 2: Apply physical and network data security techniques. It brings together every security concept introduced incrementally across the semester — password policies, file permissions, firewalls, Group Policy — into a unified, systematic hardening approach.

You will work through four learning modules that move from the outermost layer of defense (physical security) inward through host-level hardening, authentication and encryption, and finally to security validation with Splunk. Everything you learn this week directly supports Project 3, which is your primary graded deliverable for the rest of the semester.

Course structure reminder (Weeks 14–17): There is no graded lab, quiz, or discussion this week. Project 4 has been removed from the course. Your effort should focus on the learning modules below and on making steady progress on Project 3.

Key Concepts & Themes

As you work through the modules, pay special attention to these ideas and terms:

🛡️ Defense in Depth

A layered security model — physical, network, host, application, data — where each layer provides protection even if another layer fails. This framework organizes the entire week.

🔧 Server Hardening & Attack Surface

Systematically reducing the attack surface by disabling unnecessary services, applying CIS Benchmarks or DISA STIGs, and following the principle of least functionality.

🐧 Linux Hardening

Key techniques: systemctl disable/mask, SSH hardening in sshd_config, UFW/iptables rules, sysctl kernel parameters, auditd logging, and file-system mount options (noexec, nosuid).

🪟 Windows Hardening

Key techniques: disabling SMBv1, Windows Firewall with Advanced Security, Local Security Policy / GPO hardening, audit policies, restricting Remote Desktop, and Windows Defender configuration.

🔑 Secure Authentication

SSH key-based authentication (public/private key pairs), multi-factor authentication concepts, and centralized authentication review (Active Directory Kerberos, LDAP).

🔒 Encryption

Data in transit: SSL/TLS certificates, HTTPS, SFTP, LDAPS, IPSec. Data at rest: LUKS/dm-crypt (Linux), BitLocker (Windows). Secure storage disposal (NIST SP 800-88).

📦 Patch Management

The CVE system, Linux and Windows update workflows, staged rollouts, maintenance windows, rollback procedures, and balancing security urgency with operational stability.

📊 Splunk as a SIEM

The hardening → monitoring → evidence cycle. Writing SPL searches for failed logins (Event ID 4625), firewall blocks, privilege escalation, and service changes. Building dashboards and configuring alerts.
