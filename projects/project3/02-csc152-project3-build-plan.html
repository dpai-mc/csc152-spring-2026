Build pfSense First — Everything Else Depends On It

pfSense is the first VM to build because all other VMs depend on it for routing and internet access. Build and perform the basic installation before touching any other VMs. Full interface configuration (DHCP, DNS, firewall rules) happens later in Integration Plan 08 once all VMs are in place.

Download pfSense ISO

+
Download the pfSense Community Edition ISO and upload it to your Proxmox node's local storage before creating the VM.

Option A — Download via Browser and Upload to Proxmox (Recommended)

1

Go to the pfSense Download Page

Navigate to pfsense.org/download and select Community Edition (CE).

2

Select Download Options

  • Architecture: AMD64
  • Installer: DVD Image (ISO) Installer
  • Mirror: select the closest server to you

Click Download and save the .iso (or .iso.gz) file locally.

3

Upload ISO to Proxmox

In the Proxmox web UI:

  • Expand Datacenter → pve in the left panel
  • Click local storage
  • Select the ISO Images tab
  • Click Upload and select your downloaded ISO file

Option B — CLI Upload via SSH (Alternative)

If you prefer to download directly to the Proxmox host over SSH:

Bash — Proxmox SSH 
# SSH to your Proxmox host first, then:

# Download the compressed ISO directly to Proxmox ISO storage
wget -P /var/lib/vz/template/iso/ \
  "https://atxfiles.netgate.com/mirror/downloads/pfSense-CE-2.7.2-RELEASE-amd64.iso.gz"

# Decompress the .gz archive (removes .gz, leaves .iso)
gunzip /var/lib/vz/template/iso/pfSense-CE-2.7.2-RELEASE-amd64.iso.gz

# Confirm the ISO is present
ls -lh /var/lib/vz/template/iso/pfSense*.iso
ℹ️
After CLI Upload

After using the CLI method, refresh the Proxmox web UI storage view (local → ISO Images) — the ISO should appear automatically. No further steps needed; Proxmox discovers ISOs in that directory.

Create the VM in Proxmox (Web UI Wizard)

+
Click Create VM in the top-right of the Proxmox web UI. Walk through the wizard tabs in order. After creating the VM with one NIC, you will add the remaining three NICs in Section 4.

VM Creation Wizard — Tab by Tab

Step 1
General Tab
  • Node: pve
  • VM ID: 1001
  • Name: pfsense

Click Next when done.

Step 2
OS Tab
  • ISO image: select the pfSense ISO you uploaded
  • Guest OS Type: Other

Click Next when done.

Step 3
System Tab
  • Machine: q35
  • BIOS: SeaBIOS — use SeaBIOS for pfSense compatibility
  • Leave all other defaults as-is

Do not select OVMF (UEFI) — pfSense CE 2.7.x installs more reliably with SeaBIOS. Click Next.

Step 4
Disk Tab
  • Bus/Device: SCSI with VirtIO SCSI controller (or VirtIO Block)
  • Disk size: 32 GB
  • Storage: local-lvm (or your primary storage pool)

Click Next when done.

Step 5
CPU Tab
  • Sockets: 1
  • Cores: 2
  • Type: host (best performance for home lab)

Click Next when done.

Step 6
Memory Tab
  • Memory: 2048 MB (2 GB)

Click Next when done.

Step 7
Network Tab
  • Bridge: vmbr0
  • Model: VirtIO (paravirtualized)

This adds the first NIC (vtnet0 = WAN). You will add the remaining 3 NICs after the VM is created — see Section 4. Click Next.

Step 8
Confirm Tab
  • Review all settings against the spec table in Section 1
  • Uncheck "Start after created" — you need to add NICs first
  • Click Finish

VM 1001 will appear in the left panel. Do not start it yet.

§4

Add Additional Network Interfaces

+
The VM wizard only allows one NIC during creation. Add the remaining three network adapters now, before starting the VM.
1

Open VM Hardware Settings

In the Proxmox left panel, click VM 1001 (pfsense), then select the Hardware tab in the main panel.

2

Add vmbr1 (Management / LAN)

Click Add → Network Device:

  • Bridge: vmbr1
  • Model: VirtIO

Click Add.

3

Add vmbr2 (Servers / OPT1)

Click Add → Network Device again:

  • Bridge: vmbr2
  • Model: VirtIO

Click Add.

4

Add vmbr3 (Clients / OPT2)

Click Add → Network Device one final time:

  • Bridge: vmbr3
  • Model: VirtIO

Click Add.

5

Verify All 4 NICs Are Listed

The Hardware tab should now show four network devices: net0 (vmbr0), net1 (vmbr1), net2 (vmbr2), net3 (vmbr3).

ℹ️
NIC Order Matters

Proxmox assigns NICs to pfSense interfaces in the order they appear (net0 = vtnet0, net1 = vtnet1, etc.). Adding them in bridge order (vmbr0 → vmbr3) ensures vtnet0 is WAN, which is what pfSense expects when you assign interfaces in Section 6.

§5

Install pfSense from ISO

+
Start the VM, boot from the ISO, and run the pfSense installer. This permanently installs pfSense to the 32 GB virtual disk.
1

Start the VM and Open Console

Select VM 1001 in the Proxmox panel. Click Start, then immediately click Console (noVNC) to watch the boot process.

2

Boot from ISO

At the pfSense boot menu, press Enter (or wait — it boots automatically after a short countdown). The installer will load.

3

Accept the Copyright Notice

Read and accept the "Copyright and distribution notice" screen by pressing Enter (or choosing Accept).

4

Select Install pfSense

Choose Install from the menu (not "Rescue Shell" or "Live CD / No Changes"). This begins the installation wizard.

5

Keymap Selection

Select your keyboard layout. The default US is correct for most users. Press Enter to continue.

6

Partitioning — Select Auto (ZFS)

Choose Auto (ZFS) from the partitioning menu. A ZFS Configuration screen appears — ZFS requires you to explicitly choose a pool type and select a disk before proceeding.

  1. Highlight >>> Pool Type/Disks: and press Enter
  2. Pool type: select stripe (single disk — no redundancy, correct for a single-VM lab)
  3. Disk list appears: press Space to select da0 (the 32 GB VirtIO disk), then press Enter
  4. Back on the ZFS Configuration screen, highlight >>> Install and press Enter
  5. Confirm the data-destruction warning to begin installation

The disk appears as da0 in the pfSense installer. If you see no disk listed under Pool Type/Disks, you have not yet pressed Space to select it — press Space on da0, then Enter.

7

Wait for Installation to Complete

Installation will copy pfSense to disk — this typically takes 2–5 minutes. When it finishes, you will be prompted to reboot.

8

Remove ISO Before Rebooting

Before selecting Reboot — in the Proxmox Hardware tab:

  • Select the CD/DVD Drive entry
  • Click Edit
  • Select "Do not use any media"
  • Click OK

Now go back to the console and select Reboot. The VM will boot from the installed disk.

⚠️
Remove the ISO Before Rebooting

If you forget to remove the ISO and the VM reboots back into the pfSense installer, power off the VM, remove the ISO from Hardware, then start the VM again. The installed system on disk is not affected.

§6

Initial Interface Assignment (Console)

+
On first boot, pfSense displays a console menu asking you to assign network interfaces. This maps each physical virtual NIC to a pfSense role (WAN, LAN, OPT1, OPT2).

The console will list the detected interfaces with their MAC addresses. Respond to the prompts as shown below:

pfSense Console — Interface Assignment
# pfSense detects available interfaces on first boot Valid interfaces are: vtnet0 xx:xx:xx:xx:xx:xx (vmbr0 = WAN) vtnet1 xx:xx:xx:xx:xx:xx (vmbr1 = Management/LAN) vtnet2 xx:xx:xx:xx:xx:xx (vmbr2 = Servers/OPT1) vtnet3 xx:xx:xx:xx:xx:xx (vmbr3 = Clients/OPT2) Do you want to set up VLANs now? [y|n]: n Enter the WAN interface name or 'a' for auto-detection: vtnet0 Enter the LAN interface name or 'a' for auto-detection (or nothing if finished): vtnet1 Enter the Optional 1 interface name or 'a' for auto-detection (or nothing if finished): vtnet2 Enter the Optional 2 interface name or 'a' for auto-detection (or nothing if finished): vtnet3 Enter the Optional 3 interface name or 'a' for auto-detection (or nothing if finished): [press Enter — no more interfaces] The interfaces will be assigned as follows: WAN -> vtnet0 LAN -> vtnet1 OPT1 -> vtnet2 OPT2 -> vtnet3 Do you want to proceed? [y|n]: y
ℹ️
Default LAN Address After Assignment

After interface assignment, pfSense assigns LAN (vtnet1) the default address 192.168.1.1/24. You will see this displayed in the console menu. This address is temporary — full interface reconfiguration with your lab's subnet plan happens in Integration Plan 08. Do not try to access the pfSense web UI yet.

§7

Verify WAN Connectivity

+
Confirm that pfSense received a DHCP address on WAN from your home router and can reach the internet. Use the pfSense console menu to run pings.
1

Check WAN IP in Console Menu

The pfSense console main menu shows the current addresses. Verify that WAN (vtnet0) shows an IP address in the 10.255.254.x range (assigned by your classroom router). If it shows no IP, see the warning below.

2

Ping the Home Router (Option 7)

From the pfSense console menu, enter 7 (Ping host) and ping your home gateway:

  • Host to ping: 10.255.254.1
  • Expected result: replies received — WAN connectivity confirmed
3

Ping a Public DNS Server

From the same ping menu, ping a public address:

  • Host to ping: 8.8.8.8
  • Expected result: replies received — internet access confirmed
⚠️
WAN Not Getting a DHCP Address?

If WAN does not receive an IP from your home router (shown as 0.0.0.0 or blank), the most likely cause is a misconfigured bridge:

  • In the Proxmox web UI, go to your node (pve) → Network
  • Select vmbr0 and verify that your physical NIC (e.g., enp3s0 or similar) is listed as a bridge port
  • If the physical NIC is missing, edit vmbr0, add the NIC, then click Apply Configuration and recheck
