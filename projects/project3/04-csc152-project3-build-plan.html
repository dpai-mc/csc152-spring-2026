Build 04: Windows Server 2022 VM (dc01)

Create and install Windows Server 2022 Evaluation — the future Active Directory Domain Controller

🖌 VM ID: 1003 🔄 vCPU: 2 🪃 RAM: 8 GB 💾 Disk: 80 GB 🌐 IP: 10.0.2.11

ℹ️

This VM is a build-only install for now. Active Directory Domain Services (AD DS) will be installed and the server promoted to a Domain Controller in Build 09 · Active Directory & DNS. Complete this guide first and take your snapshot before moving on.

2 Download Windows Server 2022 Evaluation ISO

You need two ISO images uploaded to Proxmox before creating the VM.

ISO 1 — Windows Server 2022 Evaluation

  1. Navigate to: microsoft.com/en-us/evalcenter/evaluate-windows-server-2022
  2. Fill in the registration form and select the ISO download (approximately 4.7 GB).
  3. In Proxmox: Datacenter → local → ISO Images → Upload
ℹ️

The Evaluation edition is valid for 180 days with no product key required. Students who have access to MSDN or Azure Dev Tools for Teaching may use a licensed product key instead.

ISO 2 — VirtIO Drivers for Windows

VirtIO drivers are required so Windows can see the high-performance VirtIO disk and network adapters. Without these drivers, Windows Setup will not find any disk to install to.

  1. Reference page: pve.proxmox.com/wiki/Windows_VirtIO_Drivers
  2. Direct ISO download: fedorapeople.org/.../stable-virtio/virtio-win.iso
  3. Upload virtio-win.iso to Proxmox the same way: local → ISO Images → Upload
⚠️

Do not skip the VirtIO ISO. Without it, you will not be able to load the SCSI driver during Windows Setup, and the installer will show an empty disk list with nowhere to install Windows.

4 Install Windows Server 2022

  1. 1
    Start VM 1003 and open the console

    Click Start in the Proxmox toolbar, then click Console → noVNC to open the graphical console in your browser.

  2. 2
    Boot from CD

    When the console shows "Press any key to boot from CD or DVD…" press a key immediately. You have only a couple of seconds before it times out.

  3. 3
    Windows Setup — Language selection

    Language: English (United States). Leave Time/Currency and Keyboard at defaults. Click Next.

  4. 4
    Click "Install now"
  5. 5
    Product key

    Click "I don't have a product key" to proceed with the 180-day Evaluation. If you have an MSDN key, enter it here instead.

  6. 6
    Select OS edition

    Choose: Windows Server 2022 Standard Evaluation (Desktop Experience)

    ⚠️

    Do not select the "Server Core" option. You need the Desktop Experience (full GUI) version. Server Core has no graphical interface and is much harder to work with for this lab.

  7. 7
    Accept license terms

    Check "I accept the Microsoft Software License Terms" and click Next.

  8. 8
    Installation type: Custom (advanced)

    Select "Custom: Install Windows only (advanced)" — not the Upgrade option.

  9. 9
    Load the VirtIO SCSI driver

    The disk list will be empty because Windows does not include VirtIO drivers. Load the driver now:

    • Click Load driver
    • Click Browse, then navigate to:
      CD Drive (virtio-win) → vioscsi → 2k22 → amd64
    • Click OK
    • Select "Red Hat VirtIO SCSI pass-through controller" → click Next
    • The 80 GB virtual disk now appears — select it → click Next
  10. 10
    Installation runs automatically

    Windows copies files and expands the installation. This takes approximately 15–20 minutes. The VM will reboot automatically one or more times. Watch the console and do not interrupt the process.

  11. 11
    Set the Administrator password

    After the final reboot, Windows prompts you to set the built-in Administrator password. Choose a strong password of at least 14 characters (uppercase, lowercase, numbers, symbol). Record it in your lab notebook.

5 Install VirtIO Network Driver & Guest Agent

After Windows boots to the desktop, the VM has no working network adapter because the VirtIO network driver is also missing. Install it now.

Network Driver (NetKVM)

  1. Right-click the Start button → select Device Manager
  2. Find Ethernet Controller with a yellow warning triangle
  3. Right-click it → Update driver
  4. Choose "Browse my computer for drivers"
  5. Browse to: CD Drive (virtio-win) → NetKVM → 2k22 → amd64
  6. Click OK → Windows installs the driver
  7. The adapter will appear as "Red Hat VirtIO Ethernet Adapter"

VirtIO Guest Agent (Proxmox integration)

  1. Open File Explorer and navigate to the CD Drive containing virtio-win
  2. Run virtio-win-guest-tools.exe
  3. Accept defaults and install all components
  4. Reboot if prompted
ℹ️

The guest agent enables Proxmox to display the VM's IP address in the Summary tab, allows clean shutdowns from the Proxmox UI, and supports file system freeze/thaw for consistent snapshots. Always install it.

6 Initial Windows Configuration

Open PowerShell as Administrator (right-click Start → Windows PowerShell (Admin) or Terminal (Admin)).

Step 1 — Rename the computer

PowerShell 
# Set computer name
Rename-Computer -NewName "dc01" -Restart

The VM will reboot. Log back in as Administrator, then open PowerShell as Administrator again.

Step 2 — Configure static IP and DNS

PowerShell 
# Set static IP address
New-NetIPAddress -InterfaceAlias "Ethernet" `
  -IPAddress 10.0.2.11 `
  -PrefixLength 24 `
  -DefaultGateway 10.0.2.1

# Set DNS (point to itself for now; updated after AD DS promotion)
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" `
  -ServerAddresses 127.0.0.1, 8.8.8.8

# Verify the configuration
Get-NetIPAddress -InterfaceAlias "Ethernet"
ipconfig /all
ℹ️

If your adapter is not named "Ethernet", run Get-NetAdapter first to find the correct alias. Substitute the name you see in the -InterfaceAlias parameter above.

7 Windows Update and Baseline

⚠️

Allow Windows to install all available updates before proceeding. This can take 30–60 minutes and may require multiple reboots. Building on a fully updated baseline avoids issues later in the lab — especially during AD DS promotion and domain joins.

Option A — GUI (recommended for most students):

Start → Settings → Windows Update → Check for updates. Install all available updates, reboot when prompted, and repeat until no updates remain.

Option B — PowerShell (PSWindowsUpdate module):

PowerShell 
# Install the PSWindowsUpdate module (requires internet access)
Install-Module PSWindowsUpdate -Force

# Import and run updates
Import-Module PSWindowsUpdate
Get-WindowsUpdate
Install-WindowsUpdate -AcceptAll -AutoReboot
ℹ️

The -AutoReboot flag will reboot the VM without warning when updates are ready. Save any open work first. After reboots, re-run Install-WindowsUpdate -AcceptAll -AutoReboot until no further updates are found.

8 Enable Remote Desktop

Enabling RDP lets you connect to dc01 from the management VM (mgmt-vm) without going through the Proxmox noVNC console every time — much more comfortable for longer sessions.

PowerShell 
# Enable RDP connections
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
  -Name "fDenyTSConnections" -Value 0

# Open the Windows Firewall rule for Remote Desktop
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

# Verify the setting (should return 0 = connections allowed)
Get-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
  -Name "fDenyTSConnections"
ℹ️

You can now connect with any RDP client using 10.0.2.11 and the Administrator credentials. For lab use this is acceptable; in production you would restrict RDP access further and disable it on Domain Controllers. That hardening is covered in Build 11 · Windows Server Hardening.

9 Take a Snapshot Before AD Promotion

Before making any role-specific changes (like installing Active Directory Domain Services), capture a clean snapshot of the fully updated, configured baseline.

  1. In the Proxmox web UI, select VM 1003 (dc01) in the left panel
  2. Click the Snapshots tab
  3. Click Take Snapshot
  4. Name: pre-AD-baseline
  5. Description: Windows Server 2022 base install, before AD DS — fully updated, RDP enabled, static IP set
  6. Click Take Snapshot — this completes in a few seconds
ℹ️

This snapshot is your safety net. If the AD DS promotion in Build 09 goes wrong (misconfigured domain name, network issue, etc.), you can roll back to this clean baseline instantly with a single click. Snapshots are fast on Proxmox — always take one before major configuration changes. Disk space cost is minimal unless you make large data changes after the snapshot.
