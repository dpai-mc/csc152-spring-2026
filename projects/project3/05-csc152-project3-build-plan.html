1 Why Xubuntu Minimal?

Xubuntu is Ubuntu with the XFCE desktop environment — lightweight, fast, and perfectly suited for a management workstation VM. The minimal install omits office applications, games, and other bloat, leaving a clean GUI environment with a browser and terminal.

At roughly 1 GB RAM usage at idle, it is ideal for low-resource VMs on a Proxmox host that is already running pfSense, Windows Server, and other demanding guests.

The mgmt-vm serves as the primary workstation for all administrative tasks in this project: accessing the pfSense web GUI, connecting via RDP to the Windows domain controller (dc01), monitoring Splunk, and managing Linux servers.

2 Download Xubuntu ISO

  • Visit xubuntu.org/download/ and download Xubuntu 24.04 LTS (Noble Numbat)
  • Select: 24.04 LTS, architecture 64-bit PC (AMD64)
  • Download the full desktop ISO (not netboot) — approximately 2.6 GB
  • In Proxmox: navigate to local → ISO Images → Upload and upload the downloaded file
Tip: You can also use the Proxmox "Download from URL" feature in the ISO Images section to download directly to the Proxmox host, skipping a local download.

4 Install Xubuntu with Minimal Selection

Start VM 9002 and open its console in Proxmox. Follow these steps:

  1. Start VM 9002 and open its console (click the VM name → Console tab or the noVNC button).
  2. At the Xubuntu GRUB menu, select "Try or Install Xubuntu" and press Enter.
  3. Language → English → click Install Xubuntu.
  4. Keyboard Layout → English (US) → Continue.
  5. Installation type → select Minimal installation. Uncheck "Download updates during installation" (saves time).
    Uncheck "Install third-party software for graphics and Wi-Fi hardware."
  6. Disk setup → select "Erase disk and install Xubuntu" → Install Now → Continue to confirm.
  7. Time zone → select your region → Continue.
  8. User account setup: Name: sysadmin
    Computer name: xubuntu-template
    Username: sysadmin
    Password: choose a strong password (students will change this after cloning)
    Select "Require my password to log in"
  9. Wait for installation to complete (~10–15 minutes), then click Restart Now.
  10. Before the reboot completes: in Proxmox, go to VM 9002 → Hardware → CD/DVD Drive → set it to "Do not use any media" to eject the ISO.

5 Post-Install Template Preparation

After Xubuntu boots to the desktop, right-click the desktop and select Terminal Emulator. Run the following commands to prepare the VM for use as a reusable template:

Terminal — VM 9002 (xubuntu-template)
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install qemu-guest-agent (critical for Proxmox IP display and snapshots)
sudo apt install -y qemu-guest-agent
sudo systemctl enable --now qemu-guest-agent

# Install useful administration tools
sudo apt install -y curl wget vim net-tools

# Install Remmina RDP client (for connecting to Windows Server dc01)
sudo apt install -y remmina

# Clean up downloaded package files
sudo apt autoremove -y
sudo apt clean

# Clear machine-id so each clone receives a unique ID
sudo truncate -s 0 /etc/machine-id
sudo truncate -s 0 /var/lib/dbus/machine-id

# Clear bash history before shutting down
history -c
cat /dev/null > ~/.bash_history

# Shut down — do NOT reboot (avoids regenerating machine-id)
sudo poweroff
⚠
Important — Clear machine-id before converting: The truncate commands above zero out the machine identity file. Proxmox will regenerate a unique machine-id in each clone. If you skip this step, all clones share the same DHCP and systemd identity, causing IP and networking conflicts.

8 Configure mgmt-vm

Start VM 1004 and log in as sysadmin. Open a terminal (right-click desktop → Terminal Emulator) and run the following:

Step 1 — Set hostname
sudo hostnamectl set-hostname mgmt-vm
Step 2 — Identify network interface name
# List network interfaces to find the correct name
ip link show
# Look for an interface starting with 'ens' or 'eth' (commonly ens18)
Step 3 — Open Netplan configuration file
sudo nano /etc/netplan/01-netcfg.yaml

Replace the file contents with the following Netplan configuration. If your interface name is not ens18, substitute the correct name found in the previous step:

Netplan config — /etc/netplan/01-netcfg.yaml
network:
  version: 2
  ethernets:
    ens18:
      dhcp4: false
      addresses:
        - 10.0.1.10/24
      routes:
        - to: default
          via: 10.0.1.1
      nameservers:
        addresses:
          - 10.0.2.11
          - 8.8.8.8

Save and exit nano (Ctrl+O, Enter, Ctrl+X), then apply and verify:

Step 4 — Apply and verify
sudo netplan apply
ip addr show ens18
Gateway note: The default gateway 10.0.1.1 is the pfSense LAN interface. This gateway will not be reachable until pfSense is configured in Build 08 (pfSense Full Configuration). Internet access from mgmt-vm is expected to fail until then.

9 Verify and Set Up Browser

Firefox is installed by default in Xubuntu and is available in the applications menu. At this stage, you can verify the VM is working:

  • Open Firefox from the taskbar or application menu
  • The browser will load, confirming the GUI environment is functional
  • Internet browsing will not work yet (pfSense gateway not configured) — this is expected
  • After Build 08, navigate to https://10.0.1.1 to access the pfSense web GUI
  • After Build 09, use Remmina to RDP to dc01 at 10.0.2.10
