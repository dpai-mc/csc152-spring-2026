🏆 Project Overview

Project 4 is the capstone project for CSC 152, requiring you to demonstrate mastery of all course concepts by integrating your existing infrastructure into a comprehensive, production-ready server environment. Building upon the foundation established in Projects 1–3, you will:

  • Optimize existing infrastructure configurations for performance, security, and reliability
  • Implement file sharing services with appropriate access controls
  • Validate security controls and monitoring systems
  • Execute and document disaster recovery procedures
  • Create professional-quality technical documentation
  • Present your complete solution to stakeholders
🎓
Real-World Simulation

This project simulates a real-world scenario where a system administrator must deliver a fully functional, secure, documented, and maintainable IT infrastructure to support business operations. You will demonstrate not only technical proficiency but also professional documentation and communication skills essential for IT careers.

🎯 Learning Objectives

Upon successful completion of this project, you will be able to:

  1. Integrate multiple server technologies into a cohesive infrastructure solution
  2. Optimize existing configurations for performance, security, and reliability
  3. Implement file sharing services with appropriate access controls
  4. Demonstrate working directory services with proper user management
  5. Verify network services functionality across segmented networks
  6. Validate security controls and monitoring systems
  7. Execute and document disaster recovery procedures
  8. Create professional-quality technical documentation
  9. Present technical solutions to stakeholders effectively
  10. Reflect on learning and identify areas for professional growth

New-SmbShare -Name "Company" -Path "C:\Shares\Company" -FullAccess "IT-Staff" -ReadAccess "Domain Users"
New-SmbShare -Name "IT" -Path "C:\Shares\IT" -FullAccess "IT-Staff" -NoAccess "Domain Users"
New-SmbShare -Name "Sales" -Path "C:\Shares\Sales" -FullAccess "IT-Staff" -ChangeAccess "Sales-Staff"
New-SmbShare -Name "Engineering" -Path "C:\Shares\Engineering" -FullAccess "IT-Staff" -ChangeAccess "Engineering-Staff"

# Set NTFS permissions (example for IT share)
$acl = Get-Acl "C:\Shares\IT"
$acl.SetAccessRuleProtection($true, $false)  # Disable inheritance
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule("IT-Staff","FullControl","ContainerInherit,ObjectInherit","None","Allow")
$acl.AddAccessRule($rule)
$rule2 = New-Object System.Security.AccessControl.FileSystemAccessRule("BUILTIN\Administrators","FullControl","ContainerInherit,ObjectInherit","None","Allow")
$acl.AddAccessRule($rule2)
Set-Acl "C:\Shares\IT" $acl

# Verify shares
Get-SmbShare | Format-Table Name, Path, Description
Get-SmbShareAccess -Name "Company"

# Install Samba
sudo apt install samba samba-common -y    # Ubuntu

sudo dnf install samba samba-common -y    # Rocky Linux

sudo mkdir -p /srv/samba/public
sudo mkdir -p /srv/samba/projects

# Set ownership and permissions
sudo chown -R nobody:nogroup /srv/samba/public
sudo chmod 0775 /srv/samba/public
sudo chown -R root:engineering /srv/samba/projects
sudo chmod 0770 /srv/samba/projects

Sample smb.conf

[global]
    workgroup = TECHSTART
    realm = TECHSTART.LOCAL
    server string = TechStart Linux File Server
    security = user
    map to guest = Bad User
    log file = /var/log/samba/log.%m
    max log size = 50

[Public]
    path = /srv/samba/public
    browseable = yes
    read only = no
    guest ok = yes
    create mask = 0664
    directory mask = 0775

[Projects]
    path = /srv/samba/projects
    browseable = yes
    read only = no
    guest ok = no
    valid users = @engineering @it-staff
    write list = @engineering @it-staff
    create mask = 0660
    directory mask = 0770

Apply Configuration

# Test configuration
testparm

# Restart Samba
sudo systemctl restart smbd nmbd
sudo systemctl enable smbd nmbd

# Add Samba users
sudo smbpasswd -a username

# Allow through firewall
sudo ufw allow samba

⚠️
Take a Snapshot First!

Before simulating any failure, take a Proxmox snapshot of the VM. This ensures you can recover even if the restore process fails.

Suggested Recovery Scenario

  1. Take Proxmox snapshot of the target VM
  2. Simulate failure: Rename or delete a critical config file (e.g., /etc/ssh/sshd_config or DNS zone file)
  3. Document the failure: Show the service is broken
  4. Restore from backup: Extract the file from your backup archive
  5. Verify restoration: Show the service is working again

Example Recovery (Linux)

# Step 1: Take snapshot in Proxmox first!

# Step 2: Simulate failure
sudo mv /etc/ssh/sshd_config /etc/ssh/sshd_config.broken
sudo systemctl restart sshd  # This will fail

# Step 3: Document failure
sudo systemctl status sshd   # Show failed status

# Step 4: Restore from backup
sudo tar -xzf /backup/etc_YYYYMMDD_HHMMSS.tar.gz -C / etc/ssh/sshd_config

# Step 5: Verify recovery
sudo systemctl restart sshd
sudo systemctl status sshd   # Should show active

User Management Procedure Example

PROCEDURE: Adding a New Employee
=================================
1. Create AD user account:
   - Open Active Directory Users and Computers
   - Navigate to appropriate OU (e.g., TechStart → Users → Sales)
   - Right-click → New → User
   - Fill in: First Name, Last Name, User Logon Name
   - Set initial password, require change at next logon

2. Add to security groups:
   - Right-click user → Properties → Member Of
   - Add department group (e.g., Sales-Staff)
   - Add File-Share-Users group

3. Verify access:
   - Test login with new credentials
   - Verify file share access
   - Verify group policy application

4. Document:
   - Record in user inventory spreadsheet
   - Note creation date and department
