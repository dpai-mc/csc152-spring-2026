Project 2: Network Infrastructure & Firewall Implementation

Design and implement a segmented network infrastructure for TechStart Solutions

Course: CSC 152
Project: 2 of 4
Points: 100
Institution: Mendocino College

📋 Project Overview

Project 2 challenges students to design and implement a segmented network infrastructure for a fictional small business. Building upon the server administration skills developed in Weeks 1-7 and Project 1, you will:

  • Deploy a pfSense 2.7.2 virtual firewall appliance in Proxmox
  • Create multiple network segments (Management, Servers, Clients)
  • Configure essential network services (DHCP, DNS)
  • Implement directory services (Active Directory or OpenLDAP)
  • Establish security controls through firewall rules
💡
Real-World Simulation

This project simulates a real-world scenario where a system administrator must design and implement a secure, functional network infrastructure that separates different types of network traffic while enabling necessary communication between systems.

🎯 Learning Objectives

Upon successful completion of this project, you will be able to:

  1. Deploy and configure pfSense 2.7.2 as a virtual firewall appliance in Proxmox
  2. Design and implement a multi-subnet network architecture
  3. Configure DHCP services for dynamic IP address assignment
  4. Configure DNS services for internal name resolution
  5. Implement firewall rules to control traffic between network segments
  6. Integrate directory services (Active Directory or OpenLDAP) with network infrastructure
  7. Document network infrastructure using professional standards
  8. Test and verify network connectivity and security controls

🏢 Scenario: TechStart Solutions

🏗️
TechStart Solutions Network Infrastructure Project

TechStart Solutions is a growing startup with 25 employees. As their new IT administrator, you have been tasked with implementing a secure network infrastructure.

Company Requirements

  • A dedicated management network for IT administration
  • A server network for production services
  • A client network for employee workstations
  • Centralized user authentication through directory services
  • Network services (DHCP, DNS) for automatic device configuration
  • Security controls to limit unauthorized access between network segments

Your infrastructure must be documented thoroughly for future administrators and comply with security best practices.

💡 Tips for Success

Planning

  1. Read the entire specification before starting
  2. Plan your network design on paper first
  3. Create a checklist of all deliverables
  4. Take snapshots of VMs before major changes

Implementation

  1. Test incrementally - verify each component before moving to the next
  2. Document as you go - don't leave documentation until the end
  3. Use meaningful names for VMs, users, and groups
  4. Follow the IP addressing scheme exactly as specified

Troubleshooting

  1. Check interface assignments if network connectivity fails
  2. Verify firewall rules order - first match wins
  3. Review pfSense logs for blocked connections
  4. Test from multiple locations (client, server, management)

Documentation

  1. Use professional formatting - headers, tables, captions
  2. Include all required screenshots with clear labels
  3. Explain your decisions - don't just show configurations
  4. Proofread before submission

🔧 Common Issues and Solutions

Issue Possible Cause Solution
pfSense web GUI inaccessible Wrong interface, firewall blocking Verify LAN IP, check from management network
DHCP not assigning addresses Service not enabled, wrong interface Check DHCP service status, verify interface selection
DNS not resolving Wrong DNS server in DHCP, zone issues Verify DHCP options, check zone configuration
Firewall blocking legitimate traffic Rule order, missing rules Review rule order, add specific allow rules
VMs can't reach internet WAN configuration, NAT issues Check pfSense WAN IP, verify NAT is enabled
Inter-VLAN routing not working Missing firewall rules Add rules to allow traffic between segments
