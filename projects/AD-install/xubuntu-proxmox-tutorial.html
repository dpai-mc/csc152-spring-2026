Installing Xubuntu Minimal on Proxmox

Create and configure your Project 2 Linux Client VM — a lightweight Xubuntu desktop on Proxmox connected to the CLIENTS network segment for testing and verification.

📋 Project 2 – Linux Client VM ⏱️ Est. 30–45 minutes 🐧 Xubuntu 25.10 Minimal 🖥️ Proxmox Environment
★

Introduction & Role in Project 2

+
What this tutorial builds: A Xubuntu Minimal virtual machine connected to the CLIENTS network (vmbr3 / 10.0.3.0/24) in your Project 2 lab. This VM serves as your test workstation for verifying DHCP assignment, DNS resolution, firewall rules, internet access, and directory service connectivity through pfSense.

Why Xubuntu Minimal?

Xubuntu Minimal uses the lightweight XFCE desktop environment with the minimum set of pre-installed applications. This makes it ideal for a VM that needs a desktop for browser testing but should consume minimal resources alongside pfSense and your server VMs.

Table 1: Desktop Linux ISO Comparison

Lubuntu 24.04~3 GBLXQt (minimal)~380 MBAlso acceptable

What This VM Will Do in Project 2

  • Receive a DHCP address from pfSense in the 10.0.3.100–10.0.3.200 range
  • Use pfSense DHCP options to configure DNS server and default gateway automatically
  • Test DNS resolution of internal names (techstart.local zone records)
  • Test internet access through pfSense NAT
  • Test firewall rule enforcement — SSH to servers should be blocked, web access to servers should work
  • Optionally join the techstart.local Active Directory domain (Part C bonus)

Prerequisites

  • pfSense is installed and fully configured with vmbr3 (CLIENTS) interface active at 10.0.3.1
  • DHCP server is enabled on the CLIENTS interface in pfSense with range 10.0.3.100–200
  • Firewall rules for the CLIENTS interface are entered in pfSense
  • You have access to the Xubuntu 25.10 Minimal ISO in Proxmox ISO storage
  • Your Proxmox server has sufficient resources: at least 1 free vCPU and 2 GB free RAM

Get the Xubuntu Minimal ISO

+
Goal: Ensure the Xubuntu 25.10 Minimal ISO is available in your Proxmox ISO storage. Unlike VirtualBox where you download to your laptop, Proxmox stores ISOs centrally on the server.

Check Existing ISO Storage

1

Check if the ISO is Already Available

In Proxmox, navigate to: Datacenter → your node → local (storage) → ISO Images. Look for a file named xubuntu-25.10-minimal-amd64.iso or similar. If it's listed, skip to Step 2.

Upload or Download the ISO

If the ISO is not already in Proxmox storage, you have three options listed below in recommended order.

ℹ️
Proxmox vs. VirtualBox ISO Handling

In VirtualBox, you download the ISO to your laptop and point the VM at your local file. In Proxmox, ISOs are stored on the server in a central ISO library. All VMs on the Proxmox node share access to this library — so the instructor may have already placed the ISO there for you.

2

Option A: Download Directly from Proxmox via URL (Fastest)

In Proxmox, go to local (storage) → ISO Images → click Download from URL. Paste the direct download URL below, click Query URL, then Download. Proxmox fetches the file directly from the internet at server speed — nothing is transferred from your laptop.

💡
If the URL doesn't resolve

Mirror availability can change. Visit xubuntu.org/download to get a fresh direct-download link for the current release. Select Minimal and amd64 architecture, then copy a mirror URL and paste it into the Proxmox Download from URL dialog.

3

Option B: Download from the Class OneDrive Folder, Then Upload

Your instructor has placed the Xubuntu 25.10 Minimal ISO in the class OneDrive folder alongside other course software. This is the best fallback if the direct Proxmox download fails.

Install Xubuntu

+
Goal: Boot the VM from the Xubuntu ISO and complete the graphical installer. The Xubuntu installer is the same Ubiquity/Calamares installer used by Ubuntu — friendly and straightforward.

Boot and Open the Console

1

Start the VM

Select client01 in Proxmox → click Start. Then immediately click Console (or the >_ Console button) to open the VM's display in a new browser tab or window.

2

Xubuntu Boot Menu Appears

A purple/blue GNU GRUB boot menu will appear with options. The first option "Try or Install Xubuntu" will be highlighted. Press Enter to select it, or wait a few seconds for it to auto-start.

ℹ️
If the screen goes black or nothing happens

Click inside the console window first to capture keyboard input (Proxmox console may need a click before it captures keyboard events). Then press Enter or a key to wake the display.

3

Xubuntu Live Environment Loads

Xubuntu loads a live desktop first. After a minute, the installer dialog appears automatically. If it doesn't appear automatically, double-click the "Install Xubuntu" icon on the desktop.

Installer Screens — Walk Through in Order

The Xubuntu installer is sequential. Complete each screen and click Next (or Continue) to advance.

Screen 1

Language — Select English. Click Next.

Screen 2

Accessibility — Leave defaults. Click Next.

Screen 3

Keyboard Layout — Select English (US). Click Next.

Screen 4

Internet Connection — Select "Use wired connection" or "Connect to the internet" if prompted. Xubuntu may not be able to reach the internet at this stage (since pfSense must be running and DHCP must be active). If the connection fails, select "I don't want to connect to the internet right now" — you can update after installation.

Screen 5

What would you like to install? — Select Interactive installation. Click Next.

Screen 6

Application selection — Select Minimal installation. Do not check the boxes for NVIDIA drivers or media codecs — these are not needed for a test client VM. Click Next.

Disk Setup & User Account

Screen 7

Disk Setup — Choose "Erase disk and install Xubuntu." This erases only the virtual 20 GB disk assigned to this VM — your physical computer and other VMs are not affected. Do not enable encryption — leave that toggle off. Click Next.

💡
Why No Encryption?

Full disk encryption on a VM adds overhead and complicates snapshots and automated recovery. For a lab test client, the security benefit is minimal — the real security comes from pfSense's network segmentation, not disk encryption on the client.

Wait for Installation to Complete

4

Monitor Installation Progress

Installation takes approximately 6–15 minutes depending on your Proxmox storage speed. A progress bar is shown. Files are being copied from the ISO to the virtual disk.

⚠️
If progress appears frozen

In Proxmox, check the VM's Summary tab — if CPU and disk I/O are non-zero, the install is still working even if the installer progress bar seems stuck. Wait up to 15 minutes before assuming it has frozen. Do not forcibly power off during installation.

5

Installation Complete — Restart

When finished, the installer shows a completion screen. Click Restart Now. The VM will reboot.

6

Remove the ISO After Reboot Prompt

The installer may show a message: "Please remove the installation medium and press Enter." Before pressing Enter, go to Proxmox → client01 → Hardware → double-click the CD/DVD Drive → change to "Do not use any media" → click OK. Then press Enter in the console to complete the reboot.

ℹ️
Why remove the ISO?

If the ISO is still attached when the VM reboots, it may boot from the installer again instead of the installed system. Removing it ensures the VM always boots from the hard disk going forward.

First Boot & System Updates

+
Goal: After first boot, update all packages to the latest versions, set the correct hostname, and verify the system is functioning before installing additional software.
ℹ️
Internet Access During First Boot

For this step, you need the VM to reach the internet for updates. This will work once pfSense is running and the CLIENTS firewall rules allow outbound HTTP/HTTPS traffic. If pfSense isn't running yet, start pfSense first, then boot this VM.

Open a Terminal

1

Log In and Find the Terminal

If you set "Log in automatically," you will be at the XFCE desktop. Right-click anywhere on the desktop and select "Open Terminal Here". Alternatively, click the terminal icon in the taskbar, or use the XFCE application menu (top-left): Applications → System → Terminal Emulator.

💡
Terminal Font Too Small?

If text is hard to read in the Proxmox console, open the terminal and go to Edit → Preferences → Appearance. Change the font to size 14 or larger. This only affects the terminal display — system settings are unchanged.

Set the Hostname (if not set during install)

2

Verify and Set Hostname to client01

Confirm the current hostname matches your project requirement:

BASH – Verify hostname 
hostname

If it doesn't say client01, set it now:

BASH – Set hostname 
sudo hostnamectl set-hostname client01

Verify it was applied:

BASH 
hostnamectl status

Also update /etc/hosts to reflect the new hostname:

BASH – Edit hosts file 
sudo nano /etc/hosts

Find the line with 127.0.1.1 and make sure it reads:

FILE – /etc/hosts entry 
127.0.1.1    client01

Save with Ctrl+O, then exit with Ctrl+X.

Update All Packages

3

Update Package Database

BASH 
sudo apt update

Enter your password when prompted. You will see a list of package sources being updated. This should complete without errors — successful completion confirms your internet access through pfSense is working.

⚠️
If apt update fails with "Unable to connect"

Check that pfSense is running, the CLIENTS interface (vmbr3) has the DHCP server enabled, and the CLIENTS firewall rules include a "Pass CLIENTS net to !RFC1918 on ports 80, 443" rule. Also verify the VM got a DHCP address with ip addr show.

4

Upgrade All Installed Packages

BASH 
sudo apt upgrade -y

This downloads and installs all available updates. It may take 3–10 minutes. When complete, the terminal returns to the prompt.

5

Reboot to Apply Kernel Updates

BASH 
sudo reboot

The VM restarts. Log back in and open a terminal to continue.

5

Install QEMU Guest Agent

+
Goal: Install the QEMU Guest Agent inside the VM. This is the Proxmox equivalent of VirtualBox Guest Additions — it enables better integration between the VM and Proxmox: proper IP address reporting, clean shutdowns from the Proxmox UI, and consistent snapshots.
💡
Proxmox vs. VirtualBox: No Compilation Required

In VirtualBox, Guest Additions must be compiled against your kernel — requiring build-essential, dkms, and kernel headers. The Proxmox QEMU Guest Agent is a pre-compiled package from the Ubuntu repositories. Installation is a single apt command with no compilation step.

1

Install the QEMU Guest Agent Package

BASH 
sudo apt install qemu-guest-agent -y

This installs the agent service. It's a small package and installs quickly.

2

Enable and Start the Agent Service

BASH 
sudo systemctl enable qemu-guest-agent
sudo systemctl start qemu-guest-agent
3

Verify the Agent is Running

BASH 
sudo systemctl status qemu-guest-agent

You should see active (running) in green. Press q to exit the status view.

4

Reboot to Fully Activate

BASH 
sudo reboot

After rebooting, the Proxmox Summary tab for your VM should now display the VM's IP address — confirming the Guest Agent is communicating with Proxmox. Previously it would have shown "Guest Agent not running."

Network Configuration & DHCP Reservation

+
Goal: Verify that the VM is getting a DHCP address from pfSense on the correct network, then create a static DHCP reservation in pfSense so this client always gets the same IP address.

Verify DHCP Address Assignment

1

Check Network Interface Configuration

BASH – Check IP and network config 
ip addr show

Look for your network interface (likely ens18, eth0, or enp6s18). The inet line should show an address in the range 10.0.3.100 to 10.0.3.200.

Expected output example:

Expected Output
2: ens18: <BROADCAST,MULTICAST,UP,LOWER_UP> ...
    link/ether aa:bb:cc:dd:ee:ff brd ff:ff:ff:ff:ff:ff
    inet 10.0.3.101/24 brd 10.0.3.255 scope global dynamic ens18
2

Check Default Gateway

BASH 
ip route show

Expected: The default route should go through 10.0.3.1 (pfSense CLIENTS interface):

Expected Output
default via 10.0.3.1 dev ens18 proto dhcp src 10.0.3.101 ...
3

Check DNS Configuration

BASH 
resolvectl status
# or
cat /etc/resolv.conf

Expected: DNS server should be 10.0.2.10 or 10.0.2.11 (the DNS server you configured in pfSense DHCP options).

4

Test Basic Connectivity

BASH – Quick connectivity tests 
# Ping pfSense CLIENTS gateway
ping -c3 10.0.3.1

# Ping a server
ping -c3 10.0.2.10

# Ping internet
ping -c3 8.8.8.8

All three should succeed if pfSense and the firewall rules are correctly configured.

Troubleshoot: No DHCP Address

If you don't get a DHCP address (no 10.0.3.x address shown), try requesting one manually:

BASH – Request DHCP manually 
# Replace ens18 with your actual interface name
sudo dhclient -v ens18

If DHCP still fails, verify pfSense is running, the CLIENTS DHCP server is enabled, and the VM is connected to vmbr3 (check in Proxmox Hardware settings).

Get the MAC Address for pfSense Reservation

5

Find the VM's MAC Address

BASH – Get MAC address 
ip link show ens18
# Look for: link/ether aa:bb:cc:dd:ee:ff

Note the MAC address (the link/ether value). You will use this in pfSense to create a static DHCP reservation. Alternatively, in Proxmox, select the VM → Hardware → the Network Device shows the MAC address.

Create a Static DHCP Reservation in pfSense

A static DHCP reservation (also called DHCP reservation or static mapping) ensures this client always gets the same IP address — even though it's technically using DHCP.

6

Navigate to pfSense DHCP Leases

In pfSense web GUI: Status → DHCP Leases. Find your client01 in the leases list. Click the + icon next to it to quickly create a static mapping from the current lease.

7

Set the Reservation Details

FieldValue

Click Save → Apply Changes. For the new IP to take effect, release and renew the DHCP lease on the client:

BASH – Renew DHCP lease 
sudo dhclient -r ens18 && sudo dhclient ens18
ip addr show ens18

The IP should now be 10.0.3.101 (or whichever address you reserved).

Verify Project 2 Readiness

+
Goal: Run through the complete Project 2 verification test suite from the Linux Client. Document all results with screenshots for your Part E.1 submission.

Test 1: DHCP and Network Configuration

BASH – DHCP verification commands 
# Show IP address — should be 10.0.3.x
ip addr show

# Show routing table — default via 10.0.3.1
ip route show

# Show DNS configuration — should point to 10.0.2.10 or .11
resolvectl status | grep -A5 "DNS Servers"

Test 2: DNS Resolution

BASH – DNS tests 
# Internal forward lookups
nslookup linux.techstart.local
nslookup dc01.techstart.local
nslookup pfsense.techstart.local

# Alternative with dig (more detail)
dig linux.techstart.local
dig dc01.techstart.local

# External resolution
nslookup google.com

Expected: Internal names resolve to their correct IPs (10.0.2.10, .11, .1). External name resolves to a Google IP.

Test 3: Internet Connectivity

BASH – Internet access tests 
# Ping internet IP
ping -c4 8.8.8.8

# Ping internet hostname
ping -c4 google.com

# HTTP test
curl -I https://www.google.com

Expected: All succeed. Open the Firefox browser (from the XFCE menu) and navigate to any website to capture a browser screenshot.

Test 4: Server Connectivity (Allowed Traffic)

BASH – Server access tests 
# Ping Linux server (should succeed — ICMP is allowed)
ping -c3 10.0.2.10

# Ping Windows server
ping -c3 10.0.2.11

# Test DNS port to servers
nmap -p 53 10.0.2.10

# Test HTTP/HTTPS to servers
nmap -p 80,443 10.0.2.11

Test 5: Firewall Block Tests (Should FAIL)

BASH – Tests that should be BLOCKED 
# SSH to servers — should be BLOCKED/REFUSED
ssh 10.0.2.10
# Expected: Connection refused, Connection timed out, or Permission denied
# NOT expected: ssh prompt appearing

# Ping management network — should be BLOCKED
ping -c3 10.0.1.1
# Expected: 100% packet loss

# Nmap SSH port — should show filtered or closed
nmap -p 22 10.0.2.10
✅
After blocked tests: Check pfSense Logs

In pfSense web GUI: Status → System Logs → Firewall tab. You should see entries for the blocked SSH attempt and the blocked ping to management. Take a screenshot of these log entries — they are required evidence for Part D.2 of your project.

Test 6: Traceroute Through pfSense

BASH – Traceroute to verify routing path 
traceroute 8.8.8.8

Expected first hop: 10.0.3.1 (pfSense CLIENTS interface). Traffic passes through pfSense before reaching the internet. This confirms your client is properly routed through the firewall.

Project 2 Readiness Checklist

  • IP address in range 10.0.3.100–10.0.3.200 confirmed with ip addr show
  • Default gateway is 10.0.3.1 (pfSense)
  • DNS server is 10.0.2.10 or 10.0.2.11
  • Internal name resolution works: nslookup linux.techstart.local returns 10.0.2.10
  • External name resolution works: nslookup google.com returns an IP
  • Internet is accessible: ping -c3 8.8.8.8 shows 0% loss
  • Web browsing works in Firefox browser
  • SSH to servers is blocked (connection refused or timeout)
  • Ping to management (10.0.1.1) is blocked (100% packet loss)
  • Blocked connections appear in pfSense firewall log
  • pfSense DHCP Leases shows client01 entry
  • Static DHCP reservation created for client01
