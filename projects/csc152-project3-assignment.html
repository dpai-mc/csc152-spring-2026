Project 3: Security Hardening & Monitoring with Splunk

Implement security controls and deploy centralized log monitoring for TechStart Solutions

Course: CSC 152
Project: 3 of 4
Points: 75
Institution: Mendocino College

🔒 Project Overview

Project 3 focuses on implementing security hardening measures and deploying centralized log monitoring using Splunk Free. Building upon the network infrastructure established in Projects 1 and 2, you will:

  • Apply industry-standard security controls to Linux and Windows servers
  • Deploy Splunk Free as a Security Information and Event Management (SIEM) solution
  • Configure log forwarding from multiple sources
  • Create meaningful security dashboards for monitoring
  • Implement a backup strategy for disaster recovery
🛡️
Security Focus

This project emphasizes the principle of "defense in depth" - implementing multiple layers of security controls to protect systems and data. You will learn to think like a security professional while maintaining system functionality.

🎯 Learning Objectives

Upon successful completion of this project, you will be able to:

  1. Apply security hardening best practices to Linux server systems
  2. Apply security hardening best practices to Windows Server systems
  3. Deploy and configure Splunk Free for log aggregation and analysis
  4. Configure log forwarding from Linux systems using Splunk Universal Forwarder
  5. Configure Windows Event Log forwarding to Splunk
  6. Create Splunk dashboards for security monitoring
  7. Implement basic Splunk searches and alerts
  8. Design and implement a backup strategy for critical data
  9. Document security configurations and monitoring procedures

Services to Consider Disabling

# Check if services are running and disable if not needed
sudo systemctl disable --now cups.service        # Print service (if no printing needed)
sudo systemctl disable --now avahi-daemon.service # mDNS (if not needed)
sudo systemctl disable --now bluetooth.service   # Bluetooth (servers rarely need this)
sudo systemctl disable --now ModemManager.service # Modem support

# Verify disabled services
systemctl list-unit-files --state=disabled

Required SSH Configuration Changes

Edit /etc/ssh/sshd_config:

# Disable root login
PermitRootLogin no

# Use SSH Protocol 2 only (default in modern versions)
Protocol 2

# Disable password authentication (after setting up key-based auth)
# PasswordAuthentication no  # Only after keys are configured!

# Limit authentication attempts
MaxAuthTries 3

# Set idle timeout (5 minutes)
ClientAliveInterval 300
ClientAliveCountMax 0

# Disable empty passwords
PermitEmptyPasswords no

# Disable X11 forwarding (if not needed)
X11Forwarding no

# Limit users who can SSH (replace with actual usernames)
AllowUsers admin sysadmin

# Use strong ciphers
Ciphers aes256-gcm@openssh.com,aes128-gcm@openssh.com,aes256-ctr,aes192-ctr,aes128-ctr

# Use strong MACs
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com,hmac-sha2-512,hmac-sha2-256

# Use strong key exchange algorithms
KexAlgorithms curve25519-sha256@libssh.org,diffie-hellman-group-exchange-sha256

Apply Changes

# Validate configuration
sudo sshd -t

# Restart SSH service
sudo systemctl restart sshd

# Verify SSH is still accessible (test from another terminal before closing current session!)
⚠️
Warning

Always test SSH access from a new terminal before closing your current session. Incorrect SSH configuration can lock you out of the server!

UFW Configuration (Ubuntu)

# Enable UFW
sudo ufw enable

# Set default policies
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow SSH
sudo ufw allow 22/tcp comment 'SSH'

# Allow DNS (if DNS server)
sudo ufw allow 53/tcp comment 'DNS TCP'
sudo ufw allow 53/udp comment 'DNS UDP'

# Allow Splunk forwarder (receiving)
sudo ufw allow 9997/tcp comment 'Splunk Forwarder'

# Allow Splunk web (if Splunk server)
sudo ufw allow 8000/tcp comment 'Splunk Web'

# Check status
sudo ufw status verbose

firewalld Configuration (Rocky Linux)

# Start and enable firewalld
sudo systemctl enable --now firewalld

# Set default zone
sudo firewall-cmd --set-default-zone=drop

# Add allowed services
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=dns  # if DNS server

# Add Splunk ports
sudo firewall-cmd --permanent --add-port=9997/tcp  # Forwarder
sudo firewall-cmd --permanent --add-port=8000/tcp  # Web UI

# Reload and verify
sudo firewall-cmd --reload
sudo firewall-cmd --list-all

Install and Configure auditd

# Install auditd
sudo apt install auditd audispd-plugins  # Ubuntu
# OR
sudo dnf install audit                    # Rocky Linux

# Enable and start
sudo systemctl enable --now auditd

Add Security Audit Rules

Create /etc/audit/rules.d/security.rules:

# Monitor authentication files
-w /etc/passwd -p wa -k identity
-w /etc/group -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/gshadow -p wa -k identity

# Monitor sudoers
-w /etc/sudoers -p wa -k sudoers
-w /etc/sudoers.d/ -p wa -k sudoers

# Monitor SSH configuration
-w /etc/ssh/sshd_config -p wa -k sshd_config

# Monitor login/logout
-w /var/log/lastlog -p wa -k logins
-w /var/log/faillog -p wa -k logins

# Monitor cron
-w /etc/crontab -p wa -k cron
-w /etc/cron.d/ -p wa -k cron

# Monitor user/group modifications
-w /usr/sbin/useradd -p x -k user_modification
-w /usr/sbin/usermod -p x -k user_modification
-w /usr/sbin/userdel -p x -k user_modification
-w /usr/sbin/groupadd -p x -k group_modification
-w /usr/sbin/groupmod -p x -k group_modification
-w /usr/sbin/groupdel -p x -k group_modification

Apply Rules

# Load new rules
sudo augenrules --load

# Verify rules are loaded
sudo auditctl -l

# Test by viewing audit log
sudo ausearch -k identity

Linux Backup Script

#!/bin/bash
# backup.sh - Automated backup script for Linux server

# Configuration
BACKUP_DIR="/backup"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=7

# Create backup directory if it doesn't exist
mkdir -p ${BACKUP_DIR}

# Backup /etc
tar -czf ${BACKUP_DIR}/etc_${DATE}.tar.gz /etc 2>/dev/null

# Backup /home
tar -czf ${BACKUP_DIR}/home_${DATE}.tar.gz /home 2>/dev/null

# Backup Splunk configuration (if installed)
if [ -d "/opt/splunk/etc" ]; then
    tar -czf ${BACKUP_DIR}/splunk_etc_${DATE}.tar.gz /opt/splunk/etc 2>/dev/null
fi

# Backup DNS zone files (adjust path as needed)
if [ -d "/etc/bind" ]; then
    tar -czf ${BACKUP_DIR}/bind_${DATE}.tar.gz /etc/bind 2>/dev/null
fi

# Remove backups older than retention period
find ${BACKUP_DIR} -name "*.tar.gz" -mtime +${RETENTION_DAYS} -delete

# Log backup completion
echo "${DATE}: Backup completed" >> ${BACKUP_DIR}/backup.log

# List backup files
ls -la ${BACKUP_DIR}/*.tar.gz

Schedule with Cron

# Edit crontab
sudo crontab -e

# Add daily backup at 2 AM
0 2 * * * /usr/local/bin/backup.sh >> /var/log/backup.log 2>&1

Windows Server Backup

# Install Windows Server Backup feature
Install-WindowsFeature Windows-Server-Backup

# Create backup policy
$policy = New-WBPolicy
$volume = Get-WBVolume -AllVolumes | Where-Object {$_.MountPath -eq "C:"}
Add-WBVolume -Policy $policy -Volume $volume
Add-WBSystemState -Policy $policy

# Set backup target (adjust path)
$backupTarget = New-WBBackupTarget -VolumePath "D:"
Add-WBBackupTarget -Policy $policy -Target $backupTarget

# Set schedule (daily at 3 AM)
Set-WBSchedule -Policy $policy -Schedule 03:00

# Save policy
Set-WBPolicy -Policy $policy

💡 Tips for Success

Security Hardening

  1. Take snapshots before making any hardening changes
  2. Test incrementally - verify services still work after each change
  3. Document everything - record what you changed and why
  4. Don't lock yourself out - always test SSH from a new session before closing the current one

Splunk

  1. Start simple - get basic forwarding working before creating complex dashboards
  2. Use the Search & Reporting app to test queries before adding to dashboards
  3. Monitor your data volume - stay under the 500 MB/day limit
  4. Check forwarder connectivity - use splunk list forward-server

Backups

  1. Test your backups - a backup that can't be restored is worthless
  2. Document the restore process - you'll need this in an emergency
  3. Store backups off-system - a backup on the same disk as the data isn't safe
